## 문제
오늘도 서준이는 점근적 표기 수업 조교를 하고 있다. 아빠가 수업한 내용을 학생들이 잘 이해했는지 문제를 통해서 확인해보자.

알고리즘의 소요 시간을 나타내는 O-표기법(빅-오)을 다음과 같이 정의하자.

O(g(n)) = {f(n) | 모든 n ≥ n0에 대하여 f(n) ≤ c × g(n)인 양의 상수 c와 n0가 존재한다}

이 정의는 실제 O-표기법(https://en.wikipedia.org/wiki/Big_O_notation)과 다를 수 있다.

함수 f(n) = a1n + a0, 양의 정수 c, n0가 주어질 경우 O(n) 정의를 만족하는지 알아보자.

---

## 입력
첫째 줄에 함수 f(n)을 나타내는 정수 a1, a0가 주어진다. (0 ≤ |ai| ≤ 100)

다음 줄에 양의 정수 c가 주어진다. (1 ≤ c ≤ 100)

다음 줄에 양의 정수 n0가 주어진다. (1 ≤ n0 ≤ 100)

---

## 출력
f(n), c, n0가 O(n) 정의를 만족하면 1, 아니면 0을 출력한다.

---

## 풀이

### 풀이 핵심 로직
주어진 함수 \( f(n) = a1 \cdot n + a0 \)가 O(n) 정의를 만족하는지 확인하기 위해, 모든 \( n \geq n0 \)에 대해 \( f(n) \leq c \cdot n \)이 성립하는지를 검사한다. 또한, \( a1 \)이 \( c \)보다 작거나 같아야 \( f(n) \)이 선형적으로 증가한다는 조건도 만족해야 한다.

### 동작 과정
1. **입력 예시**: 
   - \( a1 = 2, a0 = 3 \)
   - \( c = 3 \)
   - \( n0 = 5 \)

2. **계산**:
   - \( n = 5 \)에서 \( f(n) = 2 \cdot 5 + 3 = 10 + 3 = 13 \)
   - \( g(n) = c \cdot n = 3 \cdot 5 = 15 \)

3. **비교**:
   - \( f(n) \leq g(n) \) 즉, \( 13 \leq 15 \)가 참이므로 조건을 만족한다.
   - \( a1 \leq c \) 즉, \( 2 \leq 3 \)도 참이므로 조건을 만족한다.

4. **결과**: 두 조건을 모두 만족하므로 출력은 1이 된다.

### 시간 복잡도
이 알고리즘은 상수 시간 내에 결과를 도출하므로, 시간 복잡도는 \( O(1) \)이다.

---

## 코드
```python
a1, a0 = map(int, input().split())
c = int(input())
n = int(input())

f = a1 * n + a0
g = c * n

if f <= g and  a1 <= c:
    print(1)

else:
    print(0)
```
