## 문제
은민이는 4와 7을 좋아하고, 나머지 숫자는 싫어한다. 금민수는 어떤 수가 4와 7로만 이루어진 수를 말한다.

A와 B가 주어졌을 때, A보다 크거나 같고, B보다 작거나 같은 자연수 중에 금민수인 것의 개수를 출력하는 프로그램을 작성하시오.

---

## 입력
첫째 줄에 A와 B가 주어진다. A는 1보다 크거나 같고, 1,000,000,000보다 작거나 같은 자연수이다. B는 A보다 크거나 같고, 1,000,000,000보다 작거나 같은 자연수이다.

---

## 출력
첫째 줄에 A보다 크거나 같고, B보다 작거나 같은 자연수 중에 금민수인 것의 개수를 출력한다.

## 풀이

### 풀이 핵심 로직
금민수는 4와 7로만 이루어진 숫자이므로, 모든 가능한 금민수를 생성하여 주어진 범위 [A, B] 내에 있는지를 체크하는 방식으로 문제를 해결할 수 있다. 이 문제는 브루트포스 알고리즘을 사용하여 가능한 모든 금민수를 생성한 후, 두 수 사이에 있는 금민수의 개수를 세는 방식으로 접근한다.

### 동작 과정
1. **입력 받기**: 먼저 A와 B를 입력받는다. 예를 들어 A=4, B=77이라고 가정하자.
2. **금민수 생성**: 4와 7로 이루어진 숫자를 재귀적으로 생성한다. 숫자의 자리수에 따라 최대 10자리까지 생성할 수 있다.
   - 첫 번째 자리: '4', '7'를 시작으로,
   - 두 번째 자리: 각각 앞에 '4'와 '7'을 붙여서 '44', '47', '74', '77'을 만든다.
   - 이런 식으로 계속 진행하면, 모든 가능한 금민수를 생성할 수 있다.
3. **범위 체크**: 생성된 금민수를 리스트에 저장한 후, 각 금민수가 A와 B 사이에 있는지를 체크한다. 예를 들어, 생성된 금민수가 [4, 7, 44, 47, 74, 77]일 경우,
   - A=4와 B=77 사이에 있는 금민수는 모두 포함되어 있으므로, 개수를 세면 6이 된다.
4. **결과 출력**: 최종적으로 범위에 있는 금민수의 개수를 출력한다.

### 시간 복잡도
이 알고리즘의 시간 복잡도는 O(2^n)이며, 여기서 n은 최대 자리수(최대 10자리수)에 해당한다. 이는 모든 가능한 금민수를 생성하기 위해 2의 n 제곱 만큼의 조합을 생성하기 때문이다. 하지만, 실제로는 10자리 이하의 금민수는 제한적이기 때문에 실제 수행 시간은 더 짧다.

## 코드
```python
import sys
input = sys.stdin.readline

a, b = map(int, input().split())
arr = [[] for _ in range(10)]
brr = [[] for _ in range(10)]
arr[0].append('4')
brr[0].append('7')

for i in range(1,10):
    for j in arr[i - 1]:
        arr[i].append('4' + j)
        arr[i].append('7' + j)
    for k in brr[i - 1]:
        brr[i].append('4' + k)
        brr[i].append('7' + k)

aa = len(str(a))
bb = len(str(b))
cnt = 0
for i in range(10):
    for j in arr[i]:
        if a <= int(j) <= b:
            cnt += 1
    
    for k in brr[i]:
        if a <= int(k) <= b:
            cnt += 1
            
print(cnt)
```
