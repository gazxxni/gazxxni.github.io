## 풀이

### 풀이 핵심 로직
이 문제는 주어진 나무의 높이에서 일정한 높이로 잘라낼 때, 잘린 나무의 길이가 목표 길이 이상이 되도록 하는 최대 높이를 찾는 문제입니다. 이진 탐색을 활용하여 가능한 최대 높이를 효율적으로 찾고, 잘린 나무의 길이를 계산하여 목표 길이와 비교합니다.

### 동작 과정
1. **입력 예시**: 
   - n = 4 (나무의 수)
   - m = 7 (목표 나무 길이)
   - arr = [20, 15, 10, 17] (각 나무의 높이)

2. **초기 설정**: 
   - `start`는 1 (잘라낼 최소 높이), `end`는 20 (나무 중 가장 높은 나무의 높이).
   
3. **이진 탐색 수행**:
   - **1차 반복**:
     - `mid` = (1+20)//2 = 10
     - 잘린 나무의 총합 계산: 
       - 20-10 = 10, 15-10 = 5 (합계: 15)
     - 15 < 7 이므로 `end`를 10-1=9로 조정.
   
   - **2차 반복**:
     - `mid` = (1+9)//2 = 5
     - 잘린 나무의 총합: 
       - 20-5 = 15, 15-5 = 10, 10-5 = 5, 17-5 = 12 (합계: 42)
     - 42 >= 7 이므로 `start`를 5+1=6으로 조정.

   - **3차 반복**:
     - `mid` = (6+9)//2 = 7
     - 잘린 나무의 총합:
       - 20-7 = 13, 15-7 = 8, 10-7 = 3, 17-7 = 10 (합계: 34)
     - 34 >= 7 이므로 `start`를 7+1=8로 조정.

   - **4차 반복**:
     - `mid` = (8+9)//2 = 8
     - 잘린 나무의 총합:
       - 20-8 = 12, 15-8 = 7, 10-8 = 2, 17-8 = 9 (합계: 30)
     - 30 >= 7 이므로 `start`를 8+1=9로 조정.

   - **5차 반복**:
     - `mid` = (9+9)//2 = 9
     - 잘린 나무의 총합:
       - 20-9 = 11, 15-9 = 6, 10-9 = 1, 17-9 = 8 (합계: 26)
     - 26 >= 7 이므로 `start`를 9+1=10으로 조정.

4. 반복이 끝나면 `end` 값이 8로 남습니다. 이는 최대 높이로, 이 높이로 잘랐을 때 목표 나무 길이 7을 만족할 수 있음을 의미합니다.

### 시간 복잡도
이 알고리즘은 이진 탐색을 사용하여 O(log(max_height))의 시간 복잡도를 가지며, 각 반복마다 모든 나무의 길이를 한 번씩 확인하므로 O(n)입니다. 따라서 전체 시간 복잡도는 O(n log(max_height))입니다.

## 코드
```python
import sys

input=sys.stdin.readline

n,m=map(int,input().split())
arr=list(map(int,input().split()))

start, end=1, max(arr)   # 초기 시작과 끝 값 설정

while start <= end:
    sum=0
    mid=(start+end)//2   # 중간값 설정

    for i in arr:
        if i>mid:
            sum+= i-mid   # 중간값을 기준으로 잘랐을 때 나무 길이 합
        
    if sum<m:       # 가져갈 수 있는 나무 길이 합이 목표보다 작은 경우
        end=mid-1   # 높이를 낮춰야 함
    else:
        start=mid+1

print(end)


```
