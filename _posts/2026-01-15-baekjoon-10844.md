## 문제
45656이란 수를 보자.

이 수는 인접한 모든 자리의 차이가 1이다. 이런 수를 계단 수라고 한다.

N이 주어질 때, 길이가 N인 계단 수가 총 몇 개 있는지 구해보자. 0으로 시작하는 수는 계단수가 아니다.

---

## 입력
첫째 줄에 N이 주어진다. N은 1보다 크거나 같고, 100보다 작거나 같은 자연수이다.

---

## 출력
첫째 줄에 정답을 1,000,000,000으로 나눈 나머지를 출력한다.

## 풀이

### 풀이 핵심 로직
계단 수는 인접한 자리의 차이가 1인 수로 정의됩니다. 이를 동적 프로그래밍(DP)으로 해결할 수 있으며, 각 자리수에 대해 이전 자리수에서 올 수 있는 경우의 수를 누적하여 계산합니다. 길이가 N인 계단 수의 경우, 마지막 자리수가 0부터 9까지일 때 각각의 경우를 모두 고려하여 최종 결과를 도출합니다.

### 동작 과정
1. **초기화**: DP 테이블을 생성합니다. `dp[i][j]`는 길이가 `i`인 계단 수 중에서 마지막 자리가 `j`인 경우의 수를 저장합니다. `dp[1][2:10]`을 1로 초기화하여 1자리 계단 수의 경우를 설정합니다. (0으로 시작하는 수는 제외)

2. **DP 테이블 업데이트**: N이 1보다 클 경우, 길이가 2부터 N까지 반복하며 각 자리수에 대해 가능한 경우의 수를 업데이트합니다.
   - 마지막 자리수가 `j`일 때, 이전 자리수는 `j-1` 또는 `j+1`일 수 있습니다.
   - 따라서 `dp[i][j] = dp[i-1][j-1] + dp[i-1][j+1]`로 업데이트합니다. 단, `j`가 0일 때는 `j-1`을 고려하지 않고, `j`가 9일 때는 `j+1`을 고려하지 않습니다.

3. **결과 계산**: 마지막으로 길이가 N인 계단 수의 총 개수는 `sum(dp[n])`으로 구합니다. 이 값은 1,000,000,000으로 나눈 나머지를 출력합니다.

예를 들어, N=2일 때:
- `dp[1][1]` = 0, `dp[1][2]` = 1, `dp[1][3]` = 1, ..., `dp[1][9]` = 1 (각각 1자리 계단 수)
- 다음 단계에서 `dp[2][1]`은 `dp[1][0]` + `dp[1][2]` = 0 + 1 = 1, `dp[2][2]`는 `dp[1][1]` + `dp[1][3]` = 0 + 1 = 1, ... 이런 식으로 진행됩니다.

### 시간 복잡도
O(N) - DP 테이블을 채우기 위해 N번 반복하며 각 자리수에 대해 최대 10번의 연산이 필요합니다.

---

## 코드
```python
n = int(input())

dp = [[0] * 12 for _ in range(n + 1)]
dp[1][2:11] = [1] * 9

for i in range(2, n + 1):
    for j in range(1, 11):
        dp[i][j] = dp[i - 1][j - 1] + dp[i - 1][j + 1]
        
ans = sum(dp[n])
print(ans % 1000000000)
```
