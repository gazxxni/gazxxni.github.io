## 문제
골드바흐의 추측: 2보다 큰 짝수는 두 소수의 합으로 나타낼 수 있다.


짝수 N을 두 소수의 합으로 나타내는 표현을 골드바흐 파티션이라고 한다. 짝수 N이 주어졌을 때, 골드바흐 파티션의 개수를 구해보자. 두 소수의 순서만 다른 것은 같은 파티션이다.

---

## 입력
첫째 줄에 테스트 케이스의 개수 T (1 ≤ T ≤ 100)가 주어진다. 각 테스트 케이스는 한 줄로 이루어져 있고, 정수 N은 짝수이고, 2

---

## 출력
각각의 테스트 케이스마다 골드바흐 파티션의 수를 출력한다.

## 풀이

### 풀이 핵심 로직
골드바흐의 추측에 따르면, 주어진 짝수 \( N \)은 두 소수의 합으로 표현될 수 있습니다. 따라서, 먼저 소수를 구한 후, 짝수 \( N \)의 절반 이하의 소수와 그에 대응하는 소수가 존재하는지를 확인하여 골드바흐 파티션의 개수를 세는 방식으로 문제를 해결합니다.

### 동작 과정
1. **입력 처리**: 테스트 케이스의 수 \( T \)와 각 테스트 케이스에 대한 짝수 \( N \)을 입력받습니다.
2. **소수 생성**: 2부터 1,000,000까지의 소수를 에라토스테네스의 체를 이용하여 구합니다. 이때 소수를 리스트와 집합 형태로 저장합니다.
3. **골드바흐 파티션 계산**:
   - 각 짝수 \( N \)에 대해, \( N \)의 절반 이하의 소수 \( i \)를 순회합니다.
   - \( i \)와 \( N - i \)가 모두 소수인지 확인합니다. 만약 그렇다면, 파티션의 개수를 증가시킵니다.
   - \( i \)가 \( N/2 \)를 초과하면 더 이상 검사할 필요가 없습니다.
4. **결과 출력**: 각 테스트 케이스마다 계산된 골드바흐 파티션의 개수를 출력합니다.

**예시 입력**:
```
3
8
10
12
```
**예시 출력**:
```
1
2
2
```
**단계별 설명**:
- 첫 번째 테스트 케이스 \( N = 8 \):
  - 소수: 2, 3, 5, 7
  - 가능한 조합: (3, 5)
  - 파티션 개수: 1
- 두 번째 테스트 케이스 \( N = 10 \):
  - 가능한 조합: (3, 7), (5, 5)
  - 파티션 개수: 2
- 세 번째 테스트 케이스 \( N = 12 \):
  - 가능한 조합: (5, 7)
  - 파티션 개수: 2

### 시간 복잡도
- **소수 생성**: 에라토스테네스의 체를 이용한 소수 찾기 시간 복잡도는 \( O(n \log \log n) \)입니다.
- **골드바흐 파티션 계산**: 각 짝수 \( N \)에 대해 \( O(n) \)의 시간 복잡도를 가지므로, 전체 테스트 케이스에 대해 \( O(T \cdot n) \)입니다.
- 전체적으로, 주어진 범위 내에서 효율적으로 작동하므로, 이 문제의 최악의 경우 시간 복잡도는 \( O(n \log \log n + T \cdot n) \)로 볼 수 있습니다.

## 코드
```python
def primenum(n):
    arr = [True] * (n + 1)
    arr[0] = arr[1] = False
    
    for i in range(2, int(n ** 0.5) + 1):
        if arr[i]:
            for j in range(i * i, n + 1, i):
                arr[j] = False
                
    brr = []
    for i in range(2, n + 1):
        if arr[i]:
            brr.append(i)
            
    return brr

a = primenum(1000000)
b = set(a)
t = int(input())

for _ in range(t):
    n = int(input())

    cnt = 0

    for i in a:
        if i > n // 2:
            break
        
        if (n - i) in b:
            cnt += 1
            
    print(cnt)
```
