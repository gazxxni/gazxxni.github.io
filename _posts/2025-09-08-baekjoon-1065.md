## 문제 링크
[https://www.acmicpc.net/problem/1065](https://www.acmicpc.net/problem/1065)

---

## 풀이

### 풀이 핵심 로직
이 문제는 주어진 수 `n` 이하의 한수(한자의 수열 조건을 만족하는 수)를 세는 문제입니다. 한수는 세 자리 수의 각 자리의 차가 같아야 하는데, 주어진 수가 100 미만일 경우 모두 한수로 간주되므로, 1부터 99까지의 모든 수를 세고, 100 이상의 수에 대해서는 자리 수의 차가 같은지를 확인하여 한수를 카운트합니다.

### 동작 과정
1. 입력으로 `n`을 받습니다. 예를 들어, `n = 110`이라고 가정합니다.
2. `cnt` 변수를 0으로 초기화하여 한수의 개수를 세기 위한 변수를 준비합니다.
3. 1부터 `n`까지의 수를 순회합니다.
   - `i = 1`: `1`은 100 미만이므로 `cnt`가 1 증가하여 `cnt = 1`.
   - `i = 10`: `10`도 100 미만이므로 `cnt`가 1 증가하여 `cnt = 2`.
   - `i = 99`: `99`도 100 미만이므로 `cnt`가 1 증가하여 `cnt = 99`.
   - `i = 100`: 100 이상이므로 자리 수의 차를 확인합니다. `a = ['1', '0', '0']`이고, `1 - 0`과 `0 - 0`이 다르므로 `cnt`는 증가하지 않음.
   - `i = 101`: `a = ['1', '0', '1']`이고, `1 - 0`과 `0 - 1`이 다르므로 `cnt`는 증가하지 않음.
   - `i = 110`: `a = ['1', '1', '0']`이고, `1 - 1`과 `1 - 0`이 다르므로 `cnt`는 증가하지 않음.
4. 최종적으로 `cnt`를 출력하여 한수의 개수를 반환합니다.

### 시간 복잡도
이 알고리즘의 시간 복잡도는 O(n)입니다. 각 수에 대해 문자열 변환과 조건 체크를 수행하므로, 최악의 경우 모든 수를 체크하기 때문에 입력 크기 `n`에 비례하여 실행 시간이 증가합니다.

---

## 코드
```python
n = int(input())

cnt = 0
for i in range(1, n + 1):
    a = list(str(i))
    if i < 100:
        cnt += 1  
    elif int(a[0]) - int(a[1]) == int(a[1]) - int(a[2]):
        cnt += 1  
        
print(cnt)
```
