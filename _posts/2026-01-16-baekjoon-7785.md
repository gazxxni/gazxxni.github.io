## 문제
상근이는 세계적인 소프트웨어 회사 기글에서 일한다. 이 회사의 가장 큰 특징은 자유로운 출퇴근 시간이다. 따라서, 직원들은 반드시 9시부터 6시까지 회사에 있지 않아도 된다.

각 직원은 자기가 원할 때 출근할 수 있고, 아무때나 퇴근할 수 있다.

상근이는 모든 사람의 출입카드 시스템의 로그를 가지고 있다. 이 로그는 어떤 사람이 회사에 들어왔는지, 나갔는지가 기록되어져 있다. 로그가 주어졌을 때, 현재 회사에 있는 모든 사람을 구하는 프로그램을 작성하시오.

---

## 입력
첫째 줄에 로그에 기록된 출입 기록의 수 n이 주어진다. (2 ≤ n ≤ 106) 다음 n개의 줄에는 출입 기록이 순서대로 주어지며, 각 사람의 이름이 주어지고 "enter"나 "leave"가 주어진다. "enter"인 경우는 출근, "leave"인 경우는 퇴근이다.

회사에는 동명이인이 없으며, 대소문자가 다른 경우에는 다른 이름이다. 사람들의 이름은 알파벳 대소문자로 구성된 5글자 이하의 문자열이다.

---

## 출력
현재 회사에 있는 사람의 이름을 사전 순의 역순으로 한 줄에 한 명씩 출력한다.

## 풀이

### 풀이 핵심 로직
이 문제는 회사에 있는 사람을 추적하기 위해 출입 기록을 관리해야 합니다. `enter` 이벤트에서는 사람을 추가하고, `leave` 이벤트에서는 사람을 제거하는 방식으로 진행합니다. 결과적으로 남아 있는 사람들을 사전 순의 역순으로 정렬하여 출력합니다.

### 동작 과정
1. 입력으로 주어진 로그의 수 `n`을 읽습니다.
2. 반복문을 통해 각 출입 기록을 처리합니다.
   - `enter`인 경우 해당 이름을 딕셔너리에 추가합니다.
   - `leave`인 경우 해당 이름을 딕셔너리에서 제거합니다.
3. 모든 로그를 처리한 후, 남아 있는 사람들의 이름을 리스트에 저장합니다.
4. 리스트를 역순으로 정렬한 후 출력합니다.

예시 입력:
```
7
John enter
Jane enter
John leave
Doe enter
Jane leave
Doe leave
Jane enter
```
- 첫 번째와 두 번째 로그에서 John과 Jane이 들어옵니다.
- 세 번째 로그에서 John이 나갑니다.
- 네 번째 로그에서 Doe가 들어옵니다.
- 다섯 번째 로그에서 Jane이 나갑니다.
- 여섯 번째 로그에서 Doe가 나갑니다.
- 마지막 로그에서 Jane이 들어옵니다.

결과적으로 현재 회사에 있는 사람은 Jane이 됩니다.

### 시간 복잡도
이 알고리즘의 시간 복잡도는 O(n log n)입니다. 이는 로그를 처리하는 데 O(n) 시간이 소요되고, 최종적으로 사전 순으로 정렬하는 데 O(m log m) (m은 남아 있는 사람의 수) 시간이 소요되기 때문입니다. 최악의 경우 m은 n과 같을 수 있으므로, 전체 시간 복잡도는 O(n log n)으로 볼 수 있습니다.

## 코드
```python
from collections import defaultdict

n = int(input())
dic = defaultdict(int)

for _ in range(n):
    a, b = map(str, input().rstrip().split())
    
    if b == 'enter':
        dic[a] += 1
        
    else:
        dic[a] -= 1
        
ans = []    
for k, v in dic.items():
    if v == 1:
        ans.append(k)
        
ans.sort(reverse=True)
for i in ans:
    print(i)
```
