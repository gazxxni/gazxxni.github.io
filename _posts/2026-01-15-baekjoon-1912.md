## 문제
n개의 정수로 이루어진 임의의 수열이 주어진다. 우리는 이 중 연속된 몇 개의 수를 선택해서 구할 수 있는 합 중 가장 큰 합을 구하려고 한다. 단, 수는 한 개 이상 선택해야 한다.

예를 들어서 10, -4, 3, 1, 5, 6, -35, 12, 21, -1 이라는 수열이 주어졌다고 하자. 여기서 정답은 12+21인 33이 정답이 된다.

---

## 입력
첫째 줄에 정수 n(1 ≤ n ≤ 100,000)이 주어지고 둘째 줄에는 n개의 정수로 이루어진 수열이 주어진다. 수는 -1,000보다 크거나 같고, 1,000보다 작거나 같은 정수이다.

---

## 출력
첫째 줄에 답을 출력한다.

## 풀이

### 풀이 핵심 로직
문제의 핵심은 주어진 수열에서 연속된 부분 배열의 합 중 최대값을 찾는 것이다. 이를 위해 다이나믹 프로그래밍을 사용하여 각 단계에서 현재 원소를 포함하는 최대 부분 배열의 합을 계산하고, 이 값을 통해 전체 최대값을 업데이트한다.

### 동작 과정
1. **입력**: 예를 들어, `n = 10`이고 수열이 `[10, -4, 3, 1, 5, 6, -35, 12, 21, -1]`이라고 가정하자.
2. **초기화**: `dp` 배열을 사용하여 각 인덱스 `i`에서의 최대 연속합을 저장한다. `dp[0]`을 `arr[0]`로 초기화하고, `ans`도 `arr[0]`로 설정한다.
3. **첫 번째 반복 (i=1)**: 
   - `dp[1] = max(arr[1], dp[0] + arr[1]) = max(-4, 10 - 4) = 6`
   - `ans = max(ans, dp[1]) = max(10, 6) = 10`
4. **두 번째 반복 (i=2)**:
   - `dp[2] = max(arr[2], dp[1] + arr[2]) = max(3, 6 + 3) = 9`
   - `ans = max(ans, dp[2]) = max(10, 9) = 10`
5. **세 번째 반복 (i=3)**:
   - `dp[3] = max(arr[3], dp[2] + arr[3]) = max(1, 9 + 1) = 10`
   - `ans = max(ans, dp[3]) = max(10, 10) = 10`
6. **이 과정을 반복하여 최종적으로**:
   - `dp[8]`과 `dp[9]`에서 각각 `12`와 `21`을 더해 최종적으로 `ans`는 `33`이 된다.

결과적으로, 출력은 최종 최대값인 `33`이다.

### 시간 복잡도
이 알고리즘의 시간 복잡도는 O(n)이다. n개의 원소를 한 번만 순회하면서 최대 부분 배열의 합을 계산하기 때문에 효율적이다.

## 코드
```python
import sys
input = sys.stdin.readline

n = int(input())
arr = list(map(int, input().split()))

dp = [0] * (n + 1)
dp[0] = arr[0]
ans = arr[0]

for i in range(1, n):
    dp[i] = max(arr[i], dp[i - 1] + arr[i])
    ans = max(ans, dp[i])


print(ans)
```
