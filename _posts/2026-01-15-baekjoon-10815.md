## 문제
숫자 카드는 정수 하나가 적혀져 있는 카드이다. 상근이는 숫자 카드 N개를 가지고 있다. 정수 M개가 주어졌을 때, 이 수가 적혀있는 숫자 카드를 상근이가 가지고 있는지 아닌지를 구하는 프로그램을 작성하시오.

---

## 입력
첫째 줄에 상근이가 가지고 있는 숫자 카드의 개수 N(1 ≤ N ≤ 500,000)이 주어진다. 둘째 줄에는 숫자 카드에 적혀있는 정수가 주어진다. 숫자 카드에 적혀있는 수는 -10,000,000보다 크거나 같고, 10,000,000보다 작거나 같다. 두 숫자 카드에 같은 수가 적혀있는 경우는 없다.

셋째 줄에는 M(1 ≤ M ≤ 500,000)이 주어진다. 넷째 줄에는 상근이가 가지고 있는 숫자 카드인지 아닌지를 구해야 할 M개의 정수가 주어지며, 이 수는 공백으로 구분되어져 있다. 이 수도 -10,000,000보다 크거나 같고, 10,000,000보다 작거나 같다

---

## 출력
첫째 줄에 입력으로 주어진 M개의 수에 대해서, 각 수가 적힌 숫자 카드를 상근이가 가지고 있으면 1을, 아니면 0을 공백으로 구분해 출력한다.

## 풀이

### 풀이 핵심 로직
이 문제는 상근이가 가지고 있는 숫자 카드의 목록을 활용하여 주어진 M개의 숫자가 그 목록에 포함되어 있는지를 확인하는 문제이다. 해시 맵을 사용하여 숫자 카드의 존재 여부를 O(1) 시간에 확인할 수 있도록 하여, 전체적으로 효율적인 검색이 가능하게 한다.

### 동작 과정
1. 입력으로 숫자 카드의 개수 N을 받고, N개의 숫자 카드를 리스트 `arr`에 저장한다.
2. M의 개수를 입력받고, M개의 숫자를 리스트 `brr`에 저장한다.
3. 빈 딕셔너리 `dic`를 생성하여, 숫자 카드 리스트 `arr`의 각 숫자를 키로 추가하여 카드의 존재 여부를 기록한다.
4. 리스트 `brr`의 각 숫자에 대해, 해당 숫자가 `dic`에 존재하는지 확인하고, 존재하면 1, 존재하지 않으면 0을 `ans` 리스트에 추가한다.
5. 최종적으로 `ans` 리스트의 값을 공백으로 구분하여 출력한다.

**예시 입력**
```
5
6 3 2 10 5
8
10 9 5 2 3 4 8 6
```

**예시 동작 과정**
- N = 5, 카드: [6, 3, 2, 10, 5]
- M = 8, 확인할 카드: [10, 9, 5, 2, 3, 4, 8, 6]
- 딕셔너리 `dic` 생성: {6: 0, 3: 0, 2: 0, 10: 0, 5: 0}
- 각 M개의 숫자에 대해:
  - 10: 존재 → 1 추가
  - 9: 존재하지 않음 → 0 추가
  - 5: 존재 → 1 추가
  - 2: 존재 → 1 추가
  - 3: 존재 → 1 추가
  - 4: 존재하지 않음 → 0 추가
  - 8: 존재하지 않음 → 0 추가
  - 6: 존재 → 1 추가
- 최종 결과: [1, 0, 1, 1, 1, 0, 0, 1] → 출력: `1 0 1 1 1 0 0 1`

### 시간 복잡도
해시 맵을 사용하여 각 카드의 존재 여부를 O(1) 시간에 확인할 수 있으므로, 전체 시간 복잡도는 O(N + M)이다. 여기서 N은 상근이가 가진 카드의 개수, M은 확인해야 할 숫자의 개수이다.

---

## 코드
```python
n = int(input())
arr = list(map(int, input().split()))
m = int(input())
brr = list(map(int, input().split()))

dic = {}
for i in range(len(arr)):
    dic[arr[i]] = 0

ans = []
for i in range(m):
    if brr[i] not in dic:
        ans.append(0)
    else:
        ans.append(1)
        
print(*ans)
```
