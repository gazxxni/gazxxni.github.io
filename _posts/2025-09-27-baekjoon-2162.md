## 풀이

### 풀이 핵심 로직
백준 2162번 문제는 주어진 선분들이 몇 개의 서로 연결된 그룹으로 나뉘는지를 찾는 문제입니다. 이 문제를 해결하기 위해 선분의 교차 여부를 판단하고, 교차하는 선분들을 연결하여 그룹을 형성합니다. 이를 위해 Union-Find 자료구조를 활용하여 선분들을 그룹화하고, 교차 여부를 판단하는 함수로 각 선분을 비교합니다.

### 동작 과정
1. **입력 예시**: 
   ```
   3
   1 1 4 4
   1 3 4 2
   2 1 3 3
   ```
   여기서 3개의 선분이 주어지고, 각 선분의 좌표는 (1, 1) -> (4, 4), (1, 3) -> (4, 2), (2, 1) -> (3, 3)입니다.

2. **선분 생성**: 
   선분 객체를 생성하여 각 선분의 시작점과 끝점을 저장합니다.

3. **교차 여부 판단**: 
   모든 선분 쌍에 대해 교차 여부를 판단합니다. 두 선분 `A`와 `B`가 교차하는지 확인하기 위해, 방향을 계산하고 교차하는 경우를 체크합니다.

4. **Union-Find 사용**: 
   교차하는 선분 쌍이 발견되면, Union-Find 구조를 사용하여 두 선분이 같은 그룹에 속하도록 합니다. 즉, 두 선분의 루트를 같은 것으로 연결합니다.

5. **그룹 count**: 
   모든 선분 쌍을 비교한 후, Union-Find 구조에서 각 선분의 루트를 찾아 서로 다른 루트의 개수를 세어 최종 그룹 수를 계산합니다.

6. **출력**: 
   최종적으로 그룹의 수와 각 그룹의 크기를 출력합니다.

### 시간 복잡도
이 알고리즘의 시간 복잡도는 \(O(N^2)\)입니다. 여기서 \(N\)은 선분의 개수입니다. 각 선분에 대해 모든 다른 선분과 교차 여부를 체크하기 때문에 이중 루프가 필요합니다. Union-Find 연산은 거의 상수 시간에 수행되지만, 교차 여부를 판단하는 과정이 주된 시간 소모 요소입니다.
