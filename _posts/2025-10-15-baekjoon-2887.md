## 풀이

### 풀이 핵심 로직
이 문제는 최소 신장 트리(MST)를 찾는 문제로, 크루스칼 알고리즘을 사용하여 해결합니다. 주어진 3차원 좌표의 점들 사이의 간선 가중치는 두 점 사이의 거리로 정의되며, 이 거리를 기반으로 간선을 정렬한 후, 유니온-파인드(Union-Find) 구조를 사용하여 중복된 간선을 피하며 최소 비용의 신장 트리를 구성합니다.

### 동작 과정
1. **입력 받기**: 먼저 `n`개의 점의 좌표를 입력받습니다.
2. **점 정리**: 각 좌표의 x, y, z 값과 점의 인덱스를 튜플로 묶어 리스트에 저장합니다. 이 리스트는 각각 x, y, z 좌표에 대해 정렬됩니다.
   - 예를 들어, 입력이 `3`이고 점들이 `(1, 2, 3)`, `(3, 4, 5)`, `(6, 7, 8)`이라면, x축 기준 정렬 후 `[(1, 0), (3, 1), (6, 2)]`이 됩니다.
3. **간선 생성**: 각 축(x, y, z)에서 인접한 점들 간의 가중치를 계산하여 간선 리스트를 만듭니다. 가중치는 두 점의 좌표 차이의 절대값입니다. 
   - 예를 들어, x축에서의 간선은 `[(2, 0, 1), (3, 1, 2)]`가 됩니다.
4. **간선 정렬**: 모든 간선을 가중치 기준으로 정렬합니다.
5. **크루스칼 알고리즘**: 정렬된 간선을 순회하며 유니온-파인드 구조를 사용해 사이클이 없는 경우에만 간선을 추가합니다. 이때 비용을 누적하여 최소 신장 트리의 총 비용을 계산합니다.
6. **결과 출력**: 최종적으로 계산된 최소 비용을 출력합니다.

### 시간 복잡도
- 입력을 받고 정렬하는 과정에서 각각 O(n log n)의 시간이 소요되며, 간선 생성 및 정렬에도 O(n log n)의 시간이 소요됩니다. 따라서 전체 시간 복잡도는 O(n log n)입니다.
- 유니온-파인드의 연산은 거의 상수 시간에 수행되므로, 전체 알고리즘의 시간 복잡도는 O(n log n)입니다.

## 코드
```python
import sys
input = sys.stdin.readline

n = int(input())

def find(parent, x):
    if parent[x] != x:
        parent[x] = find(parent, parent[x])
    return parent[x]

def union(parent, a, b):
    a = find(parent, a)
    b = find(parent, b)
    
    if a < b:
        parent[b] = a
    else:
        parent[a] = b
        
xx, yy, zz = [], [], []
for i in range(n):
    x, y, z = map(int, input().split())
    xx.append((x, i))
    yy.append((y, i))
    zz.append((z, i))
    
xx.sort()
yy.sort()
zz.sort()

edge = []
for cur in xx, yy, zz:
    for i in range(1, n):
        w1, a = cur[i - 1]
        w2, b = cur[i]
        edge.append((abs(w1 - w2), a, b))
        
edge.sort()

parent = [i for i in range(n)]

def krustal():
    ans = 0
    
    for cost, a, b in edge:
        if find(parent, a) != find(parent, b):
            union(parent, a, b)
            ans += cost
    
    return ans

print(krustal())
```
