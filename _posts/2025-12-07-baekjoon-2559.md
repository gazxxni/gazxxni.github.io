## 입력

## 풀이

### 풀이 핵심 로직
이 문제는 주어진 온도 수열에서 연속적인 K일 동안의 온도 합이 최대가 되는 값을 찾는 것으로, 슬라이딩 윈도우 기법을 사용하여 효율적으로 해결할 수 있다. 초기 K일의 온도 합을 구한 후, 슬라이딩 윈도우를 이용해 다음 K일의 합을 계산할 때 이전 K일의 첫 번째 값을 빼고 다음 값을 더하는 방식으로 진행한다.

### 동작 과정
1. **입력 처리**: N과 K의 값을 입력받고, N개의 온도 값을 리스트로 저장한다.
   - 예시 입력: `10 3` (N=10, K=3)
   - 온도 리스트: `[3, -2, -4, -9, 0, 3, 7, 13, 8, -3]`

2. **초기 K일의 온도 합 계산**: 처음 K일(3일)의 온도 합을 구한다.
   - 초기 합: `ss = 3 + (-2) + (-4) = -3`

3. **슬라이딩 윈도우 적용**: 인덱스 i가 K부터 N-1까지 반복하며, 이전 K일의 첫 번째 값을 빼고 새로운 값을 더해 새로운 합을 계산한다.
   - i=3: `ss = -3 - 3 + (-9) = -15` (최대값 업데이트 없음)
   - i=4: `ss = -15 - (-2) + 0 = -13` (최대값 업데이트 없음)
   - i=5: `ss = -13 - (-4) + 3 = -10` (최대값 업데이트 없음)
   - i=6: `ss = -10 - (-9) + 7 = -12` (최대값 업데이트 없음)
   - i=7: `ss = -12 - 0 + 13 = 1` (최대값 업데이트)
   - i=8: `ss = 1 - 3 + 8 = 6` (최대값 업데이트)
   - i=9: `ss = 6 - 7 + (-3) = -4` (최대값 업데이트 없음)

4. **최종 결과 출력**: 모든 K일의 합 중 최대값을 출력한다.
   - 결과: `6` (K=3일 동안의 최대 온도 합)

### 시간 복잡도
이 알고리즘의 시간 복잡도는 O(N)이다. 초기 K일의 합을 구하는 데 O(K)의 시간이 소요되고, 이후에 N-K번의 슬라이딩 윈도우 연산이 각각 O(1) 시간이 걸리기 때문에 총 시간 복잡도는 O(N)으로 효율적이다.

## 코드
```python
import sys
input = sys.stdin.readline

n, k = map(int, input().split())
arr = list(map(int, input().split()))

ss = sum(arr[:k])
ans = ss

for i in range(k, n):
    ss = ss - arr[i - k] + arr[i]
    ans = max(ans, ss)

print(ans)
```
