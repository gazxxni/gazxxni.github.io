## 문제
은민이는 4와 7을 좋아하고, 나머지 숫자는 싫어한다. 금민수는 어떤 수가 4와 7로만 이루어진 수를 말한다.

A와 B가 주어졌을 때, A보다 크거나 같고, B보다 작거나 같은 자연수 중에 금민수인 것의 개수를 출력하는 프로그램을 작성하시오.

---

## 입력
첫째 줄에 A와 B가 주어진다. A는 1보다 크거나 같고, 1,000,000,000보다 작거나 같은 자연수이다. B는 A보다 크거나 같고, 1,000,000,000보다 작거나 같은 자연수이다.

---

## 출력
첫째 줄에 A보다 크거나 같고, B보다 작거나 같은 자연수 중에 금민수인 것의 개수를 출력한다.

## 풀이

### 풀이 핵심 로직
금민수는 숫자 4와 7로만 이루어진 숫자입니다. 이 문제에서는 주어진 범위 [A, B] 내에서 금민수를 찾기 위해, 4와 7을 조합하여 가능한 모든 금민수 후보를 생성한 후, 이 후보들이 A와 B 사이에 있는지 확인하여 개수를 세는 방식으로 해결합니다.

### 동작 과정
1. **입력**: A와 B를 입력받습니다. 예를 들어, A = 400, B = 500.
2. **금민수 생성**: 
   - 빈 배열을 만들어 금민수를 생성하기 위한 작업을 진행합니다. 각 자리수에 대해 4와 7로 이루어진 모든 조합을 생성합니다.
   - 자리수는 1부터 9까지 고려합니다 (1자리수부터 9자리수까지).
   - 각 자리수에서 이전 자리수의 숫자에 4와 7을 붙여 새로운 금민수를 생성합니다.
3. **범위 체크**:
   - 생성된 금민수 후보가 A 이상 B 이하인지 확인합니다.
   - 예를 들어, 생성된 금민수 후보 중 444, 447, 474, 477, 744, 747, 774, 777 등을 확인하여 A와 B의 범위에 포함되는지 체크합니다.
4. **결과 출력**: 유효한 금민수의 개수를 세어 출력합니다. 위의 예시에서는 400과 500 사이에 있는 금민수 개수는 1개 (444)입니다.

### 시간 복잡도
O(2^n) - n은 생성할 금민수의 최대 자리수입니다. 이 문제에서는 최대 9자리의 금민수를 생성하므로 2^9 = 512개의 금민수가 생성될 수 있습니다. 각 금민수에 대해 A와 B와의 비교를 수행하기 때문에 전체적인 시간 복잡도는 이와 같습니다.

## 코드
```python
import sys
input = sys.stdin.readline

a, b = map(int, input().split())
arr = [[] for _ in range(10)]
brr = [[] for _ in range(10)]
arr[0].append('4')
brr[0].append('7')

for i in range(1,10):
    for j in arr[i - 1]:
        arr[i].append('4' + j)
        arr[i].append('7' + j)
    for k in brr[i - 1]:
        brr[i].append('4' + k)
        brr[i].append('7' + k)

aa = len(str(a))
bb = len(str(b))
cnt = 0
for i in range(10):
    for j in arr[i]:
        if a <= int(j) <= b:
            cnt += 1
    
    for k in brr[i]:
        if a <= int(k) <= b:
            cnt += 1
            
print(cnt)
```
