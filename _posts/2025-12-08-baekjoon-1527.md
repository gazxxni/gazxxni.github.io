## 문제
은민이는 4와 7을 좋아하고, 나머지 숫자는 싫어한다. 금민수는 어떤 수가 4와 7로만 이루어진 수를 말한다.

A와 B가 주어졌을 때, A보다 크거나 같고, B보다 작거나 같은 자연수 중에 금민수인 것의 개수를 출력하는 프로그램을 작성하시오.

---

## 입력
첫째 줄에 A와 B가 주어진다. A는 1보다 크거나 같고, 1,000,000,000보다 작거나 같은 자연수이다. B는 A보다 크거나 같고, 1,000,000,000보다 작거나 같은 자연수이다.

---

## 출력
첫째 줄에 A보다 크거나 같고, B보다 작거나 같은 자연수 중에 금민수인 것의 개수를 출력한다.

## 풀이

### 풀이 핵심 로직
금민수는 4와 7로만 이루어진 수이므로, 주어진 범위 내에서 가능한 모든 금민수를 생성한 후, A와 B의 범위에 포함되는지를 확인하여 개수를 세는 방식으로 문제를 해결한다.

### 동작 과정
1. **입력 받기**: A와 B를 입력받는다. 예를 들어, A=20, B=100.
2. **금민수 생성**: 4와 7로 이루어진 모든 수를 생성하기 위해, 각 자리수에 대해 4와 7을 붙여 나가는 방식을 사용한다.
   - 1자리: 4, 7
   - 2자리: 44, 47, 74, 77
   - 3자리: 444, 447, 474, 477, 744, 747, 774, 777
   - ...
3. **범위 체크**: 생성한 금민수들이 A와 B 사이에 있는지 확인한다.
   - 예를 들어, 생성된 금민수가 44, 47, 74, 77이면 이 중에서 20 ≤ 금민수 ≤ 100인 금민수의 개수를 센다.
4. **결과 출력**: 범위 내에 있는 금민수의 총 개수를 출력한다. 

위의 예시에서 44, 47, 74, 77이 A(20)와 B(100) 사이에 있으므로 총 4개의 금민수가 존재한다.

### 시간 복잡도
이 알고리즘의 시간 복잡도는 O(2^n)로, n은 생성할 수의 자리수이다. 최악의 경우, 10자리의 금민수를 생성할 수 있으므로, 약 1024개의 금민수를 생성하게 된다. 하지만 주어진 범위(A, B)에 따라 실제로 검증하는 개수는 이보다 적을 수 있다.

## 코드
```python
import sys
input = sys.stdin.readline

a, b = map(int, input().split())
arr = [[] for _ in range(10)]
brr = [[] for _ in range(10)]
arr[0].append('4')
brr[0].append('7')

for i in range(1,10):
    for j in arr[i - 1]:
        arr[i].append('4' + j)
        arr[i].append('7' + j)
    for k in brr[i - 1]:
        brr[i].append('4' + k)
        brr[i].append('7' + k)

aa = len(str(a))
bb = len(str(b))
cnt = 0
for i in range(10):
    for j in arr[i]:
        if a <= int(j) <= b:
            cnt += 1
    
    for k in brr[i]:
        if a <= int(k) <= b:
            cnt += 1
            
print(cnt)
```
