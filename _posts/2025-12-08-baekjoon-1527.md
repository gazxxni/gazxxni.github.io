## 문제
은민이는 4와 7을 좋아하고, 나머지 숫자는 싫어한다. 금민수는 어떤 수가 4와 7로만 이루어진 수를 말한다.

A와 B가 주어졌을 때, A보다 크거나 같고, B보다 작거나 같은 자연수 중에 금민수인 것의 개수를 출력하는 프로그램을 작성하시오.

---

## 입력
첫째 줄에 A와 B가 주어진다. A는 1보다 크거나 같고, 1,000,000,000보다 작거나 같은 자연수이다. B는 A보다 크거나 같고, 1,000,000,000보다 작거나 같은 자연수이다.

---

## 출력
첫째 줄에 A보다 크거나 같고, B보다 작거나 같은 자연수 중에 금민수인 것의 개수를 출력한다.

## 풀이

### 풀이 핵심 로직
금민수는 4와 7로만 구성된 자연수입니다. 주어진 범위 [A, B] 내에서 금민수를 찾기 위해, 4와 7로만 이루어진 모든 숫자를 생성하고 이 숫자들이 A와 B 사이에 있는지를 체크합니다. 이 과정을 통해 금민수의 개수를 세어 출력합니다.

### 동작 과정
1. **입력 받기**: A와 B를 입력받습니다. 예를 들어, A=10, B=100이라고 가정합니다.
2. **금민수 생성**: 1자리부터 10자리까지의 금민수를 생성합니다.
   - 1자리: 4, 7
   - 2자리: 44, 47, 74, 77
   - 3자리: 444, 447, 474, 477, 744, 747, 774, 777
   - 이와 같은 방식으로 계속 생성합니다.
3. **범위 체크**: 생성된 금민수 중에서 A(10) 이상 B(100) 이하인 숫자를 체크합니다.
   - 예를 들어, 생성된 금민수 [44, 47, 74, 77]가 A와 B의 범위에 포함되는지를 확인합니다.
4. **카운트**: 조건에 맞는 금민수를 카운트하여 최종적으로 그 개수를 출력합니다.
   - 위 예시에서는 44, 47, 74, 77의 4개가 해당 범위에 포함되어 있으므로 결과는 4가 됩니다.

### 시간 복잡도
이 알고리즘의 시간 복잡도는 O(2^N)입니다. 여기서 N은 금민수의 최대 자리수입니다. 최대 10자리까지 생성하기 때문에, 최악의 경우 2^10 = 1024개의 금민수를 생성하고 각 금민수가 주어진 범위에 포함되는지를 체크하는 과정이 포함됩니다. 하지만 실제로는 금민수가 4와 7로만 구성되어 있어 가능한 숫자가 제한적이므로 효율적으로 동작합니다.

## 코드
```python
import sys
input = sys.stdin.readline

a, b = map(int, input().split())
arr = [[] for _ in range(10)]
brr = [[] for _ in range(10)]
arr[0].append('4')
brr[0].append('7')

for i in range(1,10):
    for j in arr[i - 1]:
        arr[i].append('4' + j)
        arr[i].append('7' + j)
    for k in brr[i - 1]:
        brr[i].append('4' + k)
        brr[i].append('7' + k)

aa = len(str(a))
bb = len(str(b))
cnt = 0
for i in range(10):
    for j in arr[i]:
        if a <= int(j) <= b:
            cnt += 1
    
    for k in brr[i]:
        if a <= int(k) <= b:
            cnt += 1
            
print(cnt)
```
