## 문제
두 자연수 A와 B에 대해서, A의 배수이면서 B의 배수인 자연수를 A와 B의 공배수라고 한다. 이런 공배수 중에서 가장 작은 수를 최소공배수라고 한다. 예를 들어, 6과 15의 공배수는 30, 60, 90등이 있으며, 최소 공배수는 30이다.

두 자연수 A와 B가 주어졌을 때, A와 B의 최소공배수를 구하는 프로그램을 작성하시오.

---

## 입력
첫째 줄에 테스트 케이스의 개수 T(1 ≤ T ≤ 1,000)가 주어진다. 둘째 줄부터 T개의 줄에 걸쳐서 A와 B가 주어진다. (1 ≤ A, B ≤ 45,000)

---

## 출력
첫째 줄부터 T개의 줄에 A와 B의 최소공배수를 입력받은 순서대로 한 줄에 하나씩 출력한다.

## 풀이

### 풀이 핵심 로직
최소공배수(LCM)는 두 수 A와 B의 곱을 그들의 최대공약수(GCD)로 나눈 값으로 계산할 수 있습니다. 즉, LCM(A, B) = (A * B) / GCD(A, B)입니다. 이 방법은 GCD를 구하기 위해 유클리드 호제법을 사용하는데, 이는 매우 효율적인 알고리즘입니다.

### 동작 과정
1. **입력 받기**: 테스트 케이스의 개수 T를 입력받고, 그 다음 T개의 줄에 걸쳐 두 자연수 A와 B를 입력받습니다.
2. **GCD 계산**: 유클리드 호제법을 이용해 A와 B의 GCD를 계산합니다.
3. **LCM 계산**: GCD를 이용하여 LCM을 계산합니다. LCM은 A와 B의 곱을 GCD로 나눈 값입니다.
4. **결과 출력**: 계산된 LCM을 출력합니다.

**예시 입력**
```
3
6 15
8 12
10 25
```

**예시 출력**
```
30
24
50
```

**단계별 설명**
- 첫 번째 테스트 케이스: A=6, B=15
  - GCD(6, 15) = 3
  - LCM(6, 15) = (6 * 15) / 3 = 30
- 두 번째 테스트 케이스: A=8, B=12
  - GCD(8, 12) = 4
  - LCM(8, 12) = (8 * 12) / 4 = 24
- 세 번째 테스트 케이스: A=10, B=25
  - GCD(10, 25) = 5
  - LCM(10, 25) = (10 * 25) / 5 = 50

### 시간 복잡도
GCD를 구하는 유클리드 호제법의 시간 복잡도는 O(log(min(A, B)))입니다. 따라서 전체 알고리즘의 시간 복잡도는 O(T * log(max(A, B)))로, 여기서 T는 테스트 케이스의 개수입니다.

## 코드
```python
t = int(input())

def gcd(a, b):
    while b > 0:
        a, b = b, a % b
    return a

def lcm(a, b):
    return a * b / gcd(a, b)

for _ in range(t):
    a, b = map(int, input().split())
    
    print(int(lcm(a, b)))
```
