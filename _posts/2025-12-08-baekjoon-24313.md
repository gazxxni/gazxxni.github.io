## 문제
오늘도 서준이는 점근적 표기 수업 조교를 하고 있다. 아빠가 수업한 내용을 학생들이 잘 이해했는지 문제를 통해서 확인해보자.

알고리즘의 소요 시간을 나타내는 O-표기법(빅-오)을 다음과 같이 정의하자.

O(g(n)) = {f(n) | 모든 n ≥ n0에 대하여 f(n) ≤ c × g(n)인 양의 상수 c와 n0가 존재한다}

이 정의는 실제 O-표기법(https://en.wikipedia.org/wiki/Big_O_notation)과 다를 수 있다.

함수 f(n) = a1n + a0, 양의 정수 c, n0가 주어질 경우 O(n) 정의를 만족하는지 알아보자.

---

## 입력
첫째 줄에 함수 f(n)을 나타내는 정수 a1, a0가 주어진다. (0 ≤ |ai| ≤ 100)

다음 줄에 양의 정수 c가 주어진다. (1 ≤ c ≤ 100)

다음 줄에 양의 정수 n0가 주어진다. (1 ≤ n0 ≤ 100)

---

## 출력
f(n), c, n0가 O(n) 정의를 만족하면 1, 아니면 0을 출력한다.

---

## 성능 요약

## 풀이

### 풀이 핵심 로직
주어진 함수 \( f(n) = a_1 n + a_0 \)가 O(n) 정의를 만족하는지 검사하기 위해, 모든 \( n \geq n_0 \)에 대해 \( f(n) \leq c \times n \)를 만족해야 한다. 이를 위해 \( f(n) \)과 \( g(n) = c \times n \)을 비교하고, \( a_1 \)이 \( c \)보다 작거나 같아야 함을 확인한다.

### 동작 과정
1. 입력으로 \( a_1 \), \( a_0 \), \( c \), \( n_0 \)을 받는다.
2. \( f(n) \)을 계산한다: \( f(n) = a_1 n + a_0 \)
3. \( g(n) \)을 계산한다: \( g(n) = c \times n \)
4. \( n \)이 \( n_0 \) 이상일 때 \( f(n) \)과 \( g(n) \)를 비교한다.
5. \( f(n) \leq g(n) \)와 \( a_1 \leq c \)를 모두 만족하면 1을 출력, 그렇지 않으면 0을 출력한다.

예를 들어, 입력이 다음과 같다고 가정하자:
```
2 3
4
1
```
- \( a_1 = 2 \), \( a_0 = 3 \), \( c = 4 \), \( n_0 = 1 \)
- \( f(n) = 2n + 3 \)
- \( g(n) = 4n \)
- \( n = 1 \)일 때, \( f(1) = 2 \times 1 + 3 = 5 \)이고 \( g(1) = 4 \times 1 = 4 \)이다. \( 5 \)는 \( 4 \)보다 크므로 0을 출력한다.

### 시간 복잡도
이 알고리즘의 시간 복잡도는 \( O(1) \)이다. 입력 값을 읽고 간단한 산술 연산과 비교를 수행하는 데 상수 시간이 소요되기 때문이다.

## 코드
```python
a1, a0 = map(int, input().split())
c = int(input())
n = int(input())

f = a1 * n + a0
g = c * n

if f <= g and  a1 <= c:
    print(1)

else:
    print(0)
```
