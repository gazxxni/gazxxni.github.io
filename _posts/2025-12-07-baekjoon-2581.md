## 문제
자연수 M과 N이 주어질 때 M이상 N이하의 자연수 중 소수인 것을 모두 골라 이들 소수의 합과 최솟값을 찾는 프로그램을 작성하시오.

예를 들어 M=60, N=100인 경우 60이상 100이하의 자연수 중 소수는 61, 67, 71, 73, 79, 83, 89, 97 총 8개가 있으므로, 이들 소수의 합은 620이고, 최솟값은 61이 된다.

---

## 입력
입력의 첫째 줄에 M이, 둘째 줄에 N이 주어진다.

M과 N은 10,000이하의 자연수이며, M은 N보다 작거나 같다.

---

## 출력
M이상 N이하의 자연수 중 소수인 것을 모두 찾아 첫째 줄에 그 합을, 둘째 줄에 그 중 최솟값을 출력한다. 

단, M이상 N이하의 자연수 중 소수가 없을 경우는 첫째 줄에 -1을 출력한다.

## 풀이

### 풀이 핵심 로직
이 문제를 해결하기 위해 에라토스테네스의 체를 사용하여 M 이상 N 이하의 소수를 효율적으로 판별합니다. 먼저, 2부터 N까지의 모든 수에 대해 소수 여부를 판별한 후, 해당 범위 내의 소수를 찾아 그 합과 최솟값을 계산합니다.

### 동작 과정
1. 입력으로 M과 N을 받습니다. 예를 들어 M=60, N=100이라고 가정합니다.
2. 0부터 10000까지의 자연수를 범위로 하는 리스트 `arr`를 생성하며, 초기값을 True로 설정합니다. 단, 0과 1은 소수가 아니므로 False로 설정합니다.
3. 에라토스테네스의 체를 이용하여 리스트에서 소수를 판별합니다. 2부터 N의 제곱근까지 반복하며, 해당 수가 소수인 경우 그 배수를 False로 설정합니다.
4. M부터 N까지의 범위에서 소수를 탐색하여 그 합(`cnt`)과 최솟값(`ans`)을 계산합니다.
5. 소수가 발견되지 않으면 -1을 출력하고, 소수가 있다면 그 합과 최솟값을 차례로 출력합니다.

예시 입력:
```
60
100
```
단계별 설명:
- 에라토스테네스의 체를 통해 60 이상 100 이하의 소수로 61, 67, 71, 73, 79, 83, 89, 97을 찾습니다.
- 이들 소수의 합은 620이고, 최솟값은 61입니다.
- 따라서 출력은 다음과 같습니다.
```
620
61
```

### 시간 복잡도
이 알고리즘의 시간 복잡도는 O(N log log N)입니다. 이는 에라토스테네스의 체를 사용하여 소수를 판별하는 데 소요되는 시간입니다. 이후 M에서 N까지의 소수를 탐색하는 과정은 O(N-M)이며, 이 때문에 전체 시간 복잡도는 O(N log log N)로 평가할 수 있습니다.

## 코드
```python
m = int(input())
n = int(input())

arr = [True] * 10001
arr[0], arr[1] = False, False

for i in range(2, int(n ** 0.5) + 1): 
    if arr[i]:
        for j in range(i * i, n + 1, i):
            arr[j] = False
             
cnt = 0
ans = 0
for i in range(m, n + 1):
    if arr[i]:
        cnt += i
        if ans == 0:
            ans = i

if ans == 0:
    print(-1)
else:
    print(cnt)
    print(ans)
```
