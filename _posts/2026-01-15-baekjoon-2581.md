[https://www.acmicpc.net/problem/2581](https://www.acmicpc.net/problem/2581)

## 문제
자연수 M과 N이 주어질 때 M이상 N이하의 자연수 중 소수인 것을 모두 골라 이들 소수의 합과 최솟값을 찾는 프로그램을 작성하시오.

예를 들어 M=60, N=100인 경우 60이상 100이하의 자연수 중 소수는 61, 67, 71, 73, 79, 83, 89, 97 총 8개가 있으므로, 이들 소수의 합은 620이고, 최솟값은 61이 된다.

---

## 입력
입력의 첫째 줄에 M이, 둘째 줄에 N이 주어진다.

M과 N은 10,000이하의 자연수이며, M은 N보다 작거나 같다.

---

## 출력
M이상 N이하의 자연수 중 소수인 것을 모두 찾아 첫째 줄에 그 합을, 둘째 줄에 그 중 최솟값을 출력한다. 

단, M이상 N이하의 자연수 중 소수가 없을 경우는 첫째 줄에 -1을 출력한다.

## 풀이

### 풀이 핵심 로직
주어진 범위 M 이상 N 이하의 자연수에서 소수를 찾기 위해 에라토스테네스의 체를 사용하여 1부터 N까지의 소수를 미리 계산합니다. 이후 M부터 N까지의 소수를 확인하여 그 합과 최솟값을 구합니다. 소수가 존재하지 않을 경우 -1을 출력합니다.

### 동작 과정
1. 입력값으로 M과 N을 받습니다. 예를 들어, M=60, N=100이라면 이 범위 내에서 소수를 찾습니다.
2. 에라토스테네스의 체를 사용하여 2부터 N까지의 소수 여부를 판별합니다. 배열 `arr`을 사용하여 소수인 경우 True, 아닌 경우 False로 표시합니다.
3. M부터 N까지의 수를 순회하면서 `arr`에서 True인 경우를 찾아 합계와 최솟값을 계산합니다.
   - M=60, N=100의 경우 소수는 61, 67, 71, 73, 79, 83, 89, 97입니다.
   - 이들 소수의 합은 620이며, 최솟값은 61입니다.
4. 만약 소수가 없다면 합계와 최솟값을 계산하는 과정에서 최솟값 변수가 초기값인 0을 유지하게 되고, 이 경우 -1을 출력합니다.

### 시간 복잡도
최악의 경우, 에라토스테네스의 체를 사용하여 소수를 찾는 데 O(N log log N) 시간이 걸립니다. 이후 M에서 N까지의 수를 확인하는 과정은 O(N-M)으로, 전체 시간 복잡도는 O(N log log N)입니다.

## 코드
```python
m = int(input())
n = int(input())

arr = [True] * 10001
arr[0], arr[1] = False, False

for i in range(2, int(n ** 0.5) + 1): 
    if arr[i]:
        for j in range(i * i, n + 1, i):
            arr[j] = False
             
cnt = 0
ans = 0
for i in range(m, n + 1):
    if arr[i]:
        cnt += i
        if ans == 0:
            ans = i

if ans == 0:
    print(-1)
else:
    print(cnt)
    print(ans)
```
