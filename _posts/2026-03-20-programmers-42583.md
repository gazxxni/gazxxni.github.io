## 풀이
### 풀이 핵심 로직
이 문제는 다리를 건너는 트럭의 흐름을 시뮬레이션하는 문제입니다. 다리를 건너는 트럭의 상태를 관리하기 위해 큐를 사용하여 현재 다리에 올라가 있는 트럭의 무게와 다리에서의 경과 시간을 추적합니다. 매 초마다 트럭을 다리에 올리거나 내리는 과정을 반복하며, 모든 트럭이 다리를 건너는 데 걸리는 최소 시간을 계산합니다.

### 동작 과정
1. **초기 설정**: 현재 시간을 나타내는 변수 `ans`를 0으로 초기화하고, 다리에 올라가 있는 트럭을 관리할 큐 `running`을 생성합니다.
2. **시뮬레이션 반복**: `truck_weights`에 대기 중인 트럭이 있거나 `running` 큐에 트럭이 존재할 때까지 반복합니다.
3. **시간 증가**: 매 반복마다 시간을 1초 증가시킵니다.
4. **다리에서 트럭 이동**: 현재 `running` 큐에 있는 트럭의 경과 시간을 1초씩 증가시킵니다.
5. **다리에서 트럭 내리기**: 만약 다리에 올라간 트럭의 경과 시간이 `bridge_length`를 초과하면 그 트럭을 `running` 큐에서 제거합니다.
6. **새 트럭 추가**: 대기 중인 `truck_weights` 리스트에서 트럭을 꺼내어 추가할 수 있는 조건을 체크합니다. 
   - 새로운 트럭을 추가하기 위해 현재 `running`에서의 총 무게와 대기 중인 트럭의 무게를 비교하여, 다리가 견딜 수 있는 무게 이하이고, 현재 다리에 올라간 트럭의 수가 최대 `bridge_length` 이하인 경우에만 트럭을 다리에 추가합니다.
7. **결과 반환**: 모든 트럭이 다리를 지나면 최종적으로 걸린 시간을 반환합니다.

### 시간 복잡도
이 알고리즘의 시간 복잡도는 O(n), 여기서 n은 트럭의 수입니다. 매 초마다 트럭의 상태를 업데이트하고, 다리에 트럭을 추가하는 과정이 각 트럭에 대해 최대 한 번씩 진행되기 때문입니다.

## 코드
```python
from collections import deque
def solution(bridge_length, weight, truck_weights):
    ans = 0
    running = deque()
    
    while truck_weights or running:
        ans += 1
        
        for i in range(len(running)):
            running[i][1] += 1
        
        if running and running[0][1] > bridge_length:
            running.popleft()
            
        if truck_weights:
            new = sum(i[0] for i in running)
            
            if new + truck_weights[0] <= weight and len(running) <= bridge_length:
                running.append([truck_weights[0], 1])
                truck_weights.pop(0)
    
    return ans
```
