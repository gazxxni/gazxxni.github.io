## 문제
XX게임에는 피로도 시스템(0 이상의 정수로 표현합니다)이 있으며, 일정 피로도를 사용해서 던전을 탐험할 수 있습니다. 이때, 각 던전마다 탐험을 시작하기 위해 필요한 "최소 필요 피로도"와 던전 탐험을 마쳤을 때 소모되는 "소모 피로도"가 있습니다. "최소 필요 피로도"는 해당 던전을 탐험하기 위해 가지고 있어야 하는 최소한의 피로도를 나타내며, "소모 피로도"는 던전을 탐험한 후 소모되는 피로도를 나타냅니다. 예를 들어 "최소 필요 피로도"가 80, "소모 피로도"가 20인 던전을 탐험하기 위해서는 유저의 현재 남은 피로도는 80 이상 이어야 하며, 던전을 탐험한 후에는 피로도 20이 소모됩니다.

이 게임에는 하루에 한 번씩 탐험할 수 있는 던전이 여러개 있는데, 한 유저가 오늘 이 던전들을 최대한 많이 탐험하려 합니다. 유저의 현재 피로도 k와 각 던전별 "최소 필요 피로도", "소모 피로도"가 담긴 2차원 배열 dungeons 가 매개변수로 주어질 때, 유저가 탐험할수 있는 최대 던전 수를 return 하도록 solution 함수를 완성해주세요.

## 풀이
### 풀이 핵심 로직
이 문제는 주어진 피로도를 이용해 던전을 탐험할 수 있는 순서를 찾는 문제입니다. 모든 가능한 던전 탐험 순서를 고려하기 위해 `itertools.permutations`를 사용하여 던전의 모든 순열을 생성한 후, 각 순열에 대해 탐험 가능한 던전 수를 계산합니다. 이 과정에서 최대 탐험 가능한 던전 수를 업데이트하여 최종적으로 반환합니다.

### 동작 과정
1. **입력 예시**: `k = 80`, `dungeons = [[80, 20], [50, 40], [30, 10]]`
2. 던전의 인덱스를 생성하여 모든 순열을 만듭니다:
   - 가능한 순열: `(0, 1, 2)`, `(0, 2, 1)`, `(1, 0, 2)`, `(1, 2, 0)`, `(2, 0, 1)`, `(2, 1, 0)`
3. 각 순열에 대해 던전 탐험을 시작합니다.
   - 예를 들어 순열 `(0, 1, 2)`에서:
     - 첫 번째 던전 (0): 현재 피로도 80 ≥ 80 → 탐험 후 남은 피로도는 60
     - 두 번째 던전 (1): 현재 피로도 60 ≥ 50 → 탐험 후 남은 피로도는 20
     - 세 번째 던전 (2): 현재 피로도 20 < 30 → 탐험 불가
     - 총 탐험한 던전 수: 2
   - (0, 2, 1) 순열을 시도할 경우:
     - 첫 번째 던전 (0): 탐험 후 남은 피로도 60
     - 두 번째 던전 (2): 탐험 후 남은 피로도 50
     - 세 번째 던전 (1): 탐험 후 남은 피로도 10
     - 총 탐험한 던전 수: 3
4. 모든 순열을 탐험한 후 최대 탐험 수인 3을 반환합니다.

### 시간 복잡도
이 알고리즘의 시간 복잡도는 O(n!)입니다. 여기서 n은 던전의 개수입니다. 각 순열에 대해 탐험하는 데 O(n)의 연산이 필요하므로, 전체 시간 복잡도는 O(n * n!)입니다. 던전의 개수가 최대 8개이므로 이 범위 내에서 처리할 수 있습니다.

## 코드
```python
from itertools import permutations

def solution(k, dungeons):
    answer = -1
    n = len(dungeons)
    arr = [i for i in range(n)]
    per = list(permutations(arr, n))
    
    for p in per:
        cnt = 0
        num = k
        for i in range(len(p)):
            if num >= dungeons[p[i]][0]:
                num -= dungeons[p[i]][1]
                cnt += 1
                
            else:
                break
        
        answer = max(answer, cnt)

    
    return answer
```
