## 문제
민식이와 준영이는 자기 방에서 문자열을 공부하고 있다. 민식이가 말하길 인접해 있는 모든 문자가 같지 않은 문자열을 행운의 문자열이라고 한다고 한다. 준영이는 문자열 S를 분석하기 시작했다. 준영이는 문자열 S에 나오는 문자를 재배치하면 서로 다른 행운의 문자열이 몇 개 나오는지 궁금해졌다. 만약 원래 문자열 S도 행운의 문자열이라면 그것도 개수에 포함한다.

---

## 입력
첫째 줄에 문자열 S가 주어진다. S의 길이는 최대 10이고, 알파벳 소문자로만 이루어져 있다.

---

## 출력
첫째 줄에 위치를 재배치해서 얻은 서로 다른 행운의 문자열의 개수를 출력한다.

## 풀이

### 풀이 핵심 로직
이 문제는 주어진 문자열을 재배치하여 인접한 문자가 같지 않은 행운의 문자열을 만드는 경우의 수를 구하는 문제이다. 이를 위해 백트래킹을 사용하여 가능한 모든 조합을 탐색하되, 이전에 사용한 문자와 같은 문자를 연속으로 사용하지 않도록 조건을 설정한다.

### 동작 과정
1. 입력 문자열을 리스트 형태로 변환하고, 각 문자의 개수를 세어 사전(`dic`)에 저장한다.
   - 예를 들어, 입력 문자열이 "aabb"라면 `dic`은 `{'a': 2, 'b': 2}`가 된다.
   
2. `back` 함수는 현재 사용된 문자(`before`)와 사용한 문자 수(`use`)를 인자로 받아 재귀적으로 모든 가능한 조합을 생성한다.
   - 초기 호출은 `back(dic, None, 0)`로 시작한다.

3. 각 단계에서 사용 가능한 문자들 중에서 `before`와 다른 문자를 선택하여 그 문자의 개수를 하나 줄이고, `back` 함수를 재귀 호출한다.
   - 예를 들어, "aabb"에서 첫 번째 호출이 'a'를 선택하면 `back(dic, 'a', 1)`로 호출된다.
   
4. 모든 문자를 사용한 경우(`use == l`)에 카운트를 증가시킨다.

5. 모든 조합을 탐색한 후, 최종적으로 카운트를 출력한다.

### 시간 복잡도
이 알고리즘의 시간 복잡도는 O(N!)이며, 여기서 N은 문자열의 길이이다. 문자열 길이가 최대 10이므로, 이 복잡도는 허용 가능한 범위 내에 있다.

## 코드
```python
s = list(input().rstrip())
l = len(s)
dic = {}

for i in s:
    dic[i] = dic.get(i, 0) + 1

cnt = 0
def back(dic, before, use):
    global cnt
    if use == l:
        cnt += 1
        return
    
    for k, v in dic.items():
        if v > 0 and k != before:
            dic[k] -= 1
            back(dic, k, use + 1)
            dic[k] += 1

back(dic, None, 0)
print(cnt)

```
