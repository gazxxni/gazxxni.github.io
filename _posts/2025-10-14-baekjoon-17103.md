## 문제
골드바흐의 추측: 2보다 큰 짝수는 두 소수의 합으로 나타낼 수 있다.


짝수 N을 두 소수의 합으로 나타내는 표현을 골드바흐 파티션이라고 한다. 짝수 N이 주어졌을 때, 골드바흐 파티션의 개수를 구해보자. 두 소수의 순서만 다른 것은 같은 파티션이다.

---

## 입력
첫째 줄에 테스트 케이스의 개수 T (1 ≤ T ≤ 100)가 주어진다. 각 테스트 케이스는 한 줄로 이루어져 있고, 정수 N은 짝수이고, 2

---

## 출력
각각의 테스트 케이스마다 골드바흐 파티션의 수를 출력한다.

## 풀이

### 풀이 핵심 로직
골드바흐 파티션을 찾기 위해 먼저 주어진 최대값까지의 모든 소수를 구합니다. 그런 다음, 각 테스트 케이스에서 짝수 N을 입력받고, N의 절반까지의 소수와 그 소수의 차가 또 다른 소수인지 확인하여, 두 소수의 조합으로 N을 만들 수 있는 경우의 수를 센다.

### 동작 과정
1. **소수 구하기**: `primenum` 함수를 통해 2부터 1,000,000까지의 모든 소수를 구합니다. 이 함수는 에라토스테네스의 체 알고리즘을 사용하여 소수를 효율적으로 찾습니다.
   
2. **테스트 케이스 처리**: 입력으로 주어진 T(테스트 케이스의 개수) 만큼 반복하여, 각 테스트 케이스에서 짝수 N을 읽어옵니다.

3. **골드바흐 파티션 계산**:
   - 소수 리스트에서 N의 절반까지의 소수를 순회합니다.
   - 각 소수 i에 대해, N - i가 소수 집합에 존재하는지 확인합니다.
   - 존재할 경우, 카운트를 증가시킵니다.

4. **결과 출력**: 각 테스트 케이스마다 계산된 골드바흐 파티션의 수를 출력합니다.

예시 입력:
```
3
8
10
16
```
단계별 설명:
1. T = 3으로 3개의 테스트 케이스가 입력됩니다.
2. 첫 번째 테스트 케이스인 N = 8에 대해:
   - 소수 리스트: [2, 3, 5, 7]
   - 2 + 6 (6은 소수가 아님)
   - 3 + 5 (성공)
   - 파티션 수: 1
3. 두 번째 테스트 케이스인 N = 10에 대해:
   - 3 + 7 (성공)
   - 5 + 5 (성공)
   - 파티션 수: 2
4. 세 번째 테스트 케이스인 N = 16에 대해:
   - 3 + 13 (성공)
   - 5 + 11 (성공)
   - 7 + 9 (9는 소수가 아님)
   - 11 + 5 (중복)
   - 13 + 3 (중복)
   - 파티션 수: 3
5. 출력: `1`, `2`, `3`

### 시간 복잡도
소수를 찾는 `primenum` 함수의 시간 복잡도는 O(n log log n)이며, 각 테스트 케이스에서 소수를 확인하는 과정의 시간 복잡도는 O(n/2)입니다. 최악의 경우 T = 100이므로 전체 시간 복잡도는 O(n log log n + T * n/2)로 표현할 수 있습니다.

## 코드
```python
def primenum(n):
    arr = [True] * (n + 1)
    arr[0] = arr[1] = False
    
    for i in range(2, int(n ** 0.5) + 1):
        if arr[i]:
            for j in range(i * i, n + 1, i):
                arr[j] = False
                
    brr = []
    for i in range(2, n + 1):
        if arr[i]:
            brr.append(i)
            
    return brr

a = primenum(1000000)
b = set(a)
t = int(input())

for _ in range(t):
    n = int(input())

    cnt = 0

    for i in a:
        if i > n // 2:
            break
        
        if (n - i) in b:
            cnt += 1
            
    print(cnt)
```
