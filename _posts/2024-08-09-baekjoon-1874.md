## 풀이

### 풀이 핵심 로직
이 문제는 스택을 이용하여 주어진 수열을 생성할 수 있는지를 판단하는 문제이다. 입력으로 주어진 수를 순서대로 처리하면서, 현재 수(cnt)를 스택에 push하고, 원하는 수를 pop하여 수열을 만들기 위해 `+`와 `-` 연산을 기록한다. 원하는 수가 스택의 최상단에 있을 경우 pop하고, 그렇지 않으면 불가능하므로 "NO"를 출력한다.

### 동작 과정
1. 첫 번째 입력으로 `n`을 받는다. (예: `5`)
2. 반복문을 통해 아래의 과정을 `n`번 수행한다.
   - 첫 번째로 `1`을 입력받는다.
     - `cnt`가 `1`이므로 스택에 `1`을 push하고 `+`를 기록한다. (`stack = [1]`, `op = ['+']`)
     - `1`이 스택의 최상단에 있으므로 pop하고 `-`를 기록한다. (`stack = []`, `op = ['+', '-']`)
   - 두 번째로 `2`를 입력받는다.
     - `cnt`가 `2`이므로 스택에 `2`를 push하고 `+`를 기록한다. (`stack = [2]`, `op = ['+', '-', '+']`)
     - `2`가 스택의 최상단에 있으므로 pop하고 `-`를 기록한다. (`stack = []`, `op = ['+', '-', '+', '-']`)
   - 세 번째로 `3`을 입력받는다.
     - `cnt`가 `3`이므로 스택에 `3`을 push하고 `+`를 기록한다. (`stack = [3]`, `op = ['+', '-', '+', '-', '+']`)
     - `3`가 스택의 최상단에 있으므로 pop하고 `-`를 기록한다. (`stack = []`, `op = ['+', '-', '+', '-', '+', '-']`)
   - 네 번째로 `4`를 입력받는다.
     - `cnt`가 `4`이므로 스택에 `4`를 push하고 `+`를 기록한다. (`stack = [4]`, `op = ['+', '-', '+', '-', '+', '-', '+']`)
     - `4`가 스택의 최상단에 있으므로 pop하고 `-`를 기록한다. (`stack = []`, `op = ['+', '-', '+', '-', '+', '-', '+', '-']`)
   - 마지막으로 `5`를 입력받는다.
     - `cnt`가 `5`이므로 스택에 `5`를 push하고 `+`를 기록한다. (`stack = [5]`, `op = ['+', '-', '+', '-', '+', '-', '+', '-', '+']`)
     - `5`가 스택의 최상단에 있으므로 pop하고 `-`를 기록한다. (`stack = []`, `op = ['+', '-', '+', '-', '+', '-', '+', '-', '+', '-']`)
3. 모든 입력을 처리한 후, `tmp`가 `False`가 아닐 경우(즉, 수열 생성이 가능할 경우) 기록된 연산을 출력한다.

### 시간 복잡도
이 알고리즘의 시간 복잡도는 O(n)이다. 각 수에 대해 스택에 push하거나 pop하는 연산은 상수 시간에 수행되므로 전체적으로 n개의 수를 처리하는 데 O(n) 시간이 소요된다.

## 코드
```python
import sys

cnt=1
tmp=True
stack=[]
op=[]

n=int(input())

for i in range(n):
    num=int(input())

    while cnt <= num:
        stack.append(cnt)
        op.append('+')
        cnt+=1

    if stack[-1]==num:
        stack.pop()
        op.append('-')
    else:
        tmp=False
        break

if tmp==False:
    print("NO")
else:
    for i in op:
        print(i)



```
