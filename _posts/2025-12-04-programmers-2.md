## 문제
당신은 온라인 게임을 운영하고 있습니다. 같은 시간대에 게임을 이용하는 사람이 m명 늘어날 때마다 서버 1대가 추가로 필요합니다. 어느 시간대의 이용자가 m명 미만이라면, 서버 증설이 필요하지 않습니다. 어느 시간대의 이용자가 n x m명 이상 (n + 1) x m명 미만이라면 최소 n대의 증설된 서버가 운영 중이어야 합니다. 한 번 증설한 서버는 k시간 동안 운영하고 그 이후에는 반납합니다. 예를 들어, k = 5 일 때 10시에 증설한 서버는 10 ~ 15시에만 운영됩니다.

## 풀이
### 풀이 핵심 로직
서버 증설을 관리하기 위해 각 시간대에 필요한 서버 수를 계산하고, 현재 운영 중인 서버 수와 비교하여 부족한 서버 수만큼 증설합니다. 증설된 서버는 k시간 동안 유지되므로, 이를 반영하여 서버 수를 업데이트합니다.

### 동작 과정
1. `players` 배열을 순회하며 각 시간대의 게임 이용자 수를 확인합니다.
2. 필요한 서버 수는 `players[i] // m`으로 계산합니다.
3. 현재 시간대에 필요한 서버 수가 운영 중인 서버 수보다 많다면, 부족한 서버 수만큼 증설합니다. 이때 증설 횟수를 증가시킵니다.
4. 증설된 서버는 k시간 동안 유지되므로, 해당 시간대부터 k시간 동안 필요한 서버 수를 업데이트합니다.
5. 모든 시간대를 순회한 후, 총 증설 횟수를 반환합니다.

예시 입력 `[0, 2, 3, 3, 1, 2, 0, 0, 0, 0, 4, 2, 0, 6, 0, 4, 2, 13, 3, 5, 10, 0, 1, 5]`, `m = 3`, `k = 5`의 경우:
- 2시: 1대 증설 (총 1회)
- 10시: 1대 증설 (총 2회)
- 13시: 2대 증설 (총 3회)
- 17시: 4대 증설 (총 7회)
최종적으로 7번의 서버 증설이 필요함을 알 수 있습니다.

### 시간 복잡도
O(24) = O(1) (24시간 동안의 반복이므로 상수 시간 복잡도)

## 코드
```python
def solution(players, m, k):
    servers = [0] * 24 # 증설된 서버의 수
    cnt = 0 # 증설 횟수
    
    for i in range(24):
        p = players[i] // m # 필요한 서버의 수
        
        if p > servers[i]: # 필요한 서버의 수가 증설된 서버의 수보다 크면
            diff = p - servers[i] # 증설 횟수
            cnt += diff
            
            if i + k < 24:
                for j in range(i, i+k):
                    servers[j] += diff
            else:
                for j in range(i, 24):
                    servers[j] += diff

    return cnt
```
