## 문제
0과 1로 이루어진 어떤 문자열 x에 대한 이진 변환을 다음과 같이 정의합니다.


x의 모든 0을 제거합니다.
x의 길이를 c라고 하면, x를 "c를 2진법으로 표현한 문자열"로 바꿉니다.


예를 들어, x = "0111010"이라면, x에 이진 변환을 가하면 x = "0111010" -&gt; "1111" -&gt; "100" 이 됩니다.

s의 길이는 1 이상 150,000 이하입니다.

## 풀이
### 풀이 핵심 로직
주어진 문자열에서 '0'을 제거하고, 남은 '1'의 개수를 2진수로 변환하는 과정을 반복합니다. 이때, 이진 변환의 횟수와 제거된 '0'의 개수를 각각 카운트하여 최종적으로 리스트에 담아 반환합니다.

### 동작 과정
1. **초기화**: 변환 횟수를 저장할 변수 `a`와 제거된 '0'의 개수를 저장할 변수 `b`를 0으로 초기화합니다.
2. **루프 시작**: 문자열 `s`가 '1'이 아닐 때까지 반복합니다.
   - 변환 횟수 `a`를 1 증가시킵니다.
   - 현재 문자열에서 '1'의 개수를 세어 `num`에 저장합니다.
   - 전체 문자열의 길이에서 '1'의 개수를 빼서 제거된 '0'의 개수를 계산하고 `b`에 더합니다.
   - `num`을 이진수 문자열로 변환하여 `s`를 업데이트합니다.
3. **종료**: 문자열 `s`가 '1'이 되면 루프를 종료하고, `[a, b]`를 반환합니다.

**예시 입력 설명**:
- 입력: `"110010101001"`
  - 1회: `s = "110010101001"` → '0' 제거 후 `s = "110"` (6개 '0' 제거)
  - 2회: `s = "110"` → '0' 제거 후 `s = "10"` (1개 '0' 제거)
  - 3회: `s = "10"` → '0' 제거 후 `s = "1"` (1개 '0' 제거)
  - 최종 결과: `[3, 8]`

### 시간 복잡도
변환 과정에서 문자열을 한 번 순회하면서 '1'의 개수를 세고, 이진수로 변환하는 작업이 포함됩니다. 최악의 경우 변환 과정이 문자열의 길이에 비례하므로, 전체 시간 복잡도는 O(N log N)입니다. 여기서 N은 처음 문자열의 길이이며, log N은 이진수 변환에 필요한 시간입니다.

## 코드
```python
def solution(s):
    a, b = 0, 0
    while s != '1':
        a += 1
        num = s.count('1')
        b += len(s) - num

        binary = ""
        while num > 0:
            binary = str(num % 2) + binary
            num //= 2
        s = binary
        
    return [a, b]
```
