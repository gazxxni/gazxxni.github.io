## 풀이

### 풀이 핵심 로직
이 문제는 '윌콕스 함수'로 알려진 함수 \( w(a, b, c) \)를 계산하는 문제로, 재귀적 정의를 가진다. 이 함수는 입력 값 \( a, b, c \)의 조합에 따라 서로 다른 값을 반환하며, 메모이제이션을 통해 이미 계산된 값을 저장하여 중복 계산을 방지한다.

### 동작 과정
1. **입력**: 사용자가 세 개의 정수 \( a, b, c \)를 입력한다. 입력이 -1 -1 -1일 경우 프로그램을 종료한다.
2. **기저 사례**: 
   - \( a, b, c \) 중 하나라도 0 이하일 경우 \( w(a, b, c) = 1 \)을 반환한다.
   - \( a, b, c \) 중 하나라도 20을 초과하면 \( w(a, b, c) = w(20, 20, 20) \)로 변환하여 호출한다.
3. **메모이제이션 확인**: 이미 계산된 값이 존재하면 해당 값을 반환한다.
4. **재귀 호출**:
   - 만약 \( a < b < c \)인 경우, 특정한 방식으로 \( w \)를 재귀적으로 호출하며 값을 계산한다.
   - 그렇지 않은 경우에는 다른 방식으로 \( w \)를 호출하며 값을 계산한다.
5. **출력**: 최종적으로 계산된 \( w(a, b, c) \) 값을 출력한다.

#### 예시 입력
- 입력: `1 1 1`
  - \( w(1, 1, 1) \) 호출
  - 메모이제이션이 없으므로 재귀 호출 진행
  - 최종적으로 \( w(1, 1, 1) = 2 \) 반환
- 입력: `20 20 20`
  - \( w(20, 20, 20) \) 호출
  - 이 경우는 모든 값이 20을 초과하지 않으므로 직접 계산
  - 최종적으로 \( w(20, 20, 20) = 1048576 \) 반환

### 시간 복잡도
시간 복잡도는 \( O(1) \)로, 배열의 크기가 고정되어 있기 때문에 모든 가능한 \( (a, b, c) \) 조합에 대한 값을 미리 계산하고 저장함으로써 빠르게 결과를 찾아낼 수 있다.

## 코드
```python
def w(a, b, c):
    if a <= 0 or b <= 0 or c <= 0:
        return 1
    
    if a > 20 or b > 20 or c > 20:
        return w(20, 20, 20)
    
    if dp[a][b][c]:
        return dp[a][b][c]
    
    if a < b < c:
        dp[a][b][c] = w(a, b, c - 1) + w(a, b - 1, c- 1) - w(a, b - 1, c)
        return dp[a][b][c]
    
    dp[a][b][c] = w(a - 1, b, c) + w(a - 1, b - 1, c) + w(a - 1, b, c - 1) - w(a - 1, b - 1, c - 1)
    return dp[a][b][c]

dp = [[[0] * 21 for _ in range(21)] for _ in range(21)]

while True:
    a, b, c = map(int, input().split())

    if a == -1 and b == -1 and c == -1:
        exit()
    
    print(f'w({a}, {b}, {c}) = {w(a, b, c)}')
```
