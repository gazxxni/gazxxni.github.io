## 문제
수 N개 A1, A2, ..., AN이 주어진다. 이때, 연속된 부분 구간의 합이 M으로 나누어 떨어지는 구간의 개수를 구하는 프로그램을 작성하시오.

즉, Ai + ... + Aj (i ≤ j) 의 합이 M으로 나누어 떨어지는 (i, j) 쌍의 개수를 구해야 한다.

---

## 입력
첫째 줄에 N과 M이 주어진다. (1 ≤ N ≤ 106, 2 ≤ M ≤ 103)

둘째 줄에 N개의 수 A1, A2, ..., AN이 주어진다. (0 ≤ Ai ≤ 109)

---

## 출력
첫째 줄에 연속된 부분 구간의 합이 M으로 나누어 떨어지는 구간의 개수를 출력한다.

## 풀이

### 풀이 핵심 로직
문제는 연속된 부분 구간의 합이 M으로 나누어 떨어지는 구간의 개수를 구하는 것입니다. 이를 위해 누적합을 사용하고, 누적합을 M으로 나눈 나머지를 활용하여 같은 나머지를 가지는 누적합의 개수를 세는 방식으로 접근합니다. 즉, 두 누적합의 나머지가 같다면 그 두 구간 사이의 합은 M으로 나누어 떨어지므로, 같은 나머지를 가진 인덱스의 쌍을 찾아 카운트합니다.

### 동작 과정
1. **입력 예시**: N=5, M=3, A=[1, 2, 3, 1, 2]
   
2. **누적합 계산**:
   - 첫 번째 수: current_sum = 1 → current_sum % 3 = 1
   - 두 번째 수: current_sum = 3 → current_sum % 3 = 0
   - 세 번째 수: current_sum = 6 → current_sum % 3 = 0
   - 네 번째 수: current_sum = 7 → current_sum % 3 = 1
   - 다섯 번째 수: current_sum = 9 → current_sum % 3 = 0

3. **나머지 카운트**:
   - 나머지 배열: [3, 0, 2] (0은 3번, 1은 2번, 2는 0번 발생)

4. **쌍 계산**:
   - 나머지 0: cnt = 3 → 쌍 수 = 3 * (3 - 1) / 2 = 3
   - 나머지 1: cnt = 2 → 쌍 수 = 2 * (2 - 1) / 2 = 1
   - 나머지 2: cnt = 0 → 쌍 수 = 0

5. **총 쌍 수**: 3 + 1 = 4

### 시간 복잡도
이 알고리즘의 시간 복잡도는 O(N)입니다. N개의 수를 한 번 순회하며 누적합을 계산하고 나머지 카운트를 갱신하기 때문입니다.

## 코드
```python
import sys
input = sys.stdin.readline

'''
1. 구간 합이 M으로 나누어 떨어진다는 것은 
(prefix_sum[j] - prefix_sum[i-1]) % M == 0이라는 뜻

2. 이 식을 이항하면 prefix_sum[j] % M == prefix_sum[i-1] % M

3. 즉, 누적합을 M으로 나눈 나머지가 같은 인덱스들을 찾으면 됨
'''
n, m = map(int, input().split())
arr = list(map(int, input().split()))

# 1. 나머지를 카운트할 배열 또는 딕셔너리 준비
remainder_count = [0] * m

# 2. 누적합 변수 초기화
current_sum = 0

# 3. 나머지가 0인 경우를 미리 카운트
remainder_count[0] = 1 

# 4. 수열을 순회하면서 누적합과 나머지 계산
for x in arr:
    current_sum += x
    r = current_sum % m
    remainder_count[r] += 1

ans = 0
for cnt in remainder_count:
    # 해당 나머지를 가진 누적합의 개수가 2개 이상일 때만 쌍을 만들 수 있음
    if cnt > 1:
        ans += (cnt * (cnt - 1)) // 2

print(ans)
```
