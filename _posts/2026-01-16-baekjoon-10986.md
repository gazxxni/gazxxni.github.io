## 문제
수 N개 A1, A2, ..., AN이 주어진다. 이때, 연속된 부분 구간의 합이 M으로 나누어 떨어지는 구간의 개수를 구하는 프로그램을 작성하시오.

즉, Ai + ... + Aj (i ≤ j) 의 합이 M으로 나누어 떨어지는 (i, j) 쌍의 개수를 구해야 한다.

---

## 입력
첫째 줄에 N과 M이 주어진다. (1 ≤ N ≤ 106, 2 ≤ M ≤ 103)

둘째 줄에 N개의 수 A1, A2, ..., AN이 주어진다. (0 ≤ Ai ≤ 109)

---

## 출력
첫째 줄에 연속된 부분 구간의 합이 M으로 나누어 떨어지는 구간의 개수를 출력한다.

## 풀이

### 풀이 핵심 로직
주어진 수열의 연속된 부분 구간의 합이 M으로 나누어 떨어지려면, 해당 구간의 누적합 두 개의 나머지가 같아야 합니다. 즉, `prefix_sum[j] % M == prefix_sum[i-1] % M`이 성립하는 경우에 해당합니다. 이를 이용해 각 나머지의 빈도를 기록하고, 각 나머지별로 쌍을 계산하여 결과를 도출합니다.

### 동작 과정
1. **입력 데이터 설정**: N(수의 개수)과 M(나누는 수)을 입력받고, N개의 정수로 이루어진 배열을 읽습니다.
   - 예시 입력: 
     ```
     5 3
     1 2 3 1 2
     ```

2. **나머지 카운트 배열 초기화**: 나머지가 0부터 M-1까지의 빈도를 기록하기 위한 배열을 초기화합니다. `remainder_count[0]`는 1로 초기화하여, 누적합이 M으로 나누어 떨어지는 경우를 포함합니다.

3. **누적합 계산**:
   - 배열을 순회하며 현재까지의 누적합을 계산하고, M으로 나눈 나머지를 구합니다.
   - 각 나머지의 빈도를 증가시킵니다.
   - 예를 들어, 배열을 순회하며 각 단계에서 누적합과 나머지를 다음과 같이 계산합니다:
     - 누적합: 1 → 나머지: 1
     - 누적합: 3 → 나머지: 0
     - 누적합: 6 → 나머지: 0
     - 누적합: 7 → 나머지: 1
     - 누적합: 9 → 나머지: 0

4. **결과 계산**: 각 나머지별로 카운트된 빈도를 사용하여, 쌍을 계산합니다. 각 나머지의 빈도가 2 이상인 경우, `cnt * (cnt - 1) / 2`로 해당 나머지를 가진 쌍의 개수를 구합니다.

5. **출력**: 최종적으로 구한 쌍의 개수를 출력합니다.

### 시간 복잡도
이 알고리즘은 O(N)입니다. N은 입력 배열의 크기이며, 각 원소에 대해 한 번씩만 순회하며 누적합과 나머지를 계산하기 때문에 선형 시간 복잡도를 가집니다.

## 코드
```python
import sys
input = sys.stdin.readline

'''
1. 구간 합이 M으로 나누어 떨어진다는 것은 
(prefix_sum[j] - prefix_sum[i-1]) % M == 0이라는 뜻

2. 이 식을 이항하면 prefix_sum[j] % M == prefix_sum[i-1] % M

3. 즉, 누적합을 M으로 나눈 나머지가 같은 인덱스들을 찾으면 됨
'''
n, m = map(int, input().split())
arr = list(map(int, input().split()))

# 1. 나머지를 카운트할 배열 또는 딕셔너리 준비
remainder_count = [0] * m

# 2. 누적합 변수 초기화
current_sum = 0

# 3. 나머지가 0인 경우를 미리 카운트
remainder_count[0] = 1 

# 4. 수열을 순회하면서 누적합과 나머지 계산
for x in arr:
    current_sum += x
    r = current_sum % m
    remainder_count[r] += 1

ans = 0
for cnt in remainder_count:
    # 해당 나머지를 가진 누적합의 개수가 2개 이상일 때만 쌍을 만들 수 있음
    if cnt > 1:
        ans += (cnt * (cnt - 1)) // 2

print(ans)
```
