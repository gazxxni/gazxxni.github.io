## 문제
지원이에게 2진 수열을 가르쳐 주기 위해, 지원이 아버지는 그에게 타일들을 선물해주셨다. 그리고 이 각각의 타일들은 0 또는 1이 쓰여 있는 낱장의 타일들이다.

어느 날 짓궂은 동주가 지원이의 공부를 방해하기 위해 0이 쓰여진 낱장의 타일들을 붙여서 한 쌍으로 이루어진 00 타일들을 만들었다. 결국 현재 1 하나만으로 이루어진 타일 또는 0타일을 두 개 붙인 한 쌍의 00타일들만이 남게 되었다.

그러므로 지원이는 타일로 더 이상 크기가 N인 모든 2진 수열을 만들 수 없게 되었다. 예를 들어, N=1일 때 1만 만들 수 있고, N=2일 때는 00, 11을 만들 수 있다. (01, 10은 만들 수 없게 되었다.) 또한 N=4일 때는 0011, 0000, 1001, 1100, 1111 등 총 5개의 2진 수열을 만들 수 있다.

우리의 목표는 N이 주어졌을 때 지원이가 만들 수 있는 모든 가짓수를 세는 것이다. 단 타일들은 무한히 많은 것으로 가정하자.

---

## 입력
첫 번째 줄에 자연수 N이 주어진다. (1 ≤ N ≤ 1,000,000)

---

## 출력
첫 번째 줄에 지원이가 만들 수 있는 길이가 N인 모든 2진 수열의 개수를 15746으로 나눈 나머지를 출력한다.

## 풀이

### 풀이 핵심 로직
문제는 길이 N의 이진 수열을 만들 때, '0'은 두 개가 붙어 있는 '00' 형태로만 사용할 수 있으며, '1'은 하나만 사용할 수 있다는 조건을 다룹니다. 이러한 조건을 바탕으로 동적 프로그래밍을 통해 수열의 가짓수를 계산할 수 있습니다. 길이 N의 수열은 길이 N-1의 수열에 '1'을 붙이거나, 길이 N-2의 수열에 '00'을 붙여서 만들 수 있으므로, 점화식 `dp[n] = dp[n-1] + dp[n-2]`을 세울 수 있습니다.

### 동작 과정
1. **초기화**: `dp[1]`은 1 (수열: "1"), `dp[2]`은 2 (수열: "00", "11")로 설정합니다.
2. **점화식 적용**: N이 3 이상일 때, 각 dp[k]는 dp[k-1]과 dp[k-2]의 합으로 정의됩니다.
   - 예를 들어, N=3일 경우: 
     - dp[2] = 2 ("00", "11")
     - dp[1] = 1 ("1")
     - dp[3] = dp[2] + dp[1] = 2 + 1 = 3 ("001", "111", "100")
3. **모듈로 연산**: 결과는 15746으로 나눈 나머지를 출력합니다. 이는 큰 수에 대한 오버플로우를 방지하기 위한 조치입니다.
4. **출력**: dp[n]의 값을 출력합니다.

예시 입력 N=4의 경우:
- dp[3] = 3
- dp[4] = dp[3] + dp[2] = 3 + 2 = 5
- 가능한 수열: "0000", "0011", "1001", "1100", "1111"

### 시간 복잡도
시간 복잡도는 O(N)입니다. N에 대해 한 번의 반복문을 수행하여 dp 배열을 계산하기 때문입니다.

## 코드
```python
import sys
input = sys.stdin.readline

n = int(input())
dp = [0] * 1000001
dp[1] = 1
dp[2] = 2

for k in range(3,n+1):
    dp[k] = (dp[k-1]+ dp[k-2])%15746
print(dp[n])
```
