## 문제
총 N개의 문자열로 이루어진 집합 S가 주어진다.

입력으로 주어지는 M개의 문자열 중에서 집합 S에 포함되어 있는 것이 총 몇 개인지 구하는 프로그램을 작성하시오.

---

## 입력
첫째 줄에 문자열의 개수 N과 M (1 ≤ N ≤ 10,000, 1 ≤ M ≤ 10,000)이 주어진다. 

다음 N개의 줄에는 집합 S에 포함되어 있는 문자열들이 주어진다.

다음 M개의 줄에는 검사해야 하는 문자열들이 주어진다.

입력으로 주어지는 문자열은 알파벳 소문자로만 이루어져 있으며, 길이는 500을 넘지 않는다. 집합 S에 같은 문자열이 여러 번 주어지는 경우는 없다.

---

## 출력
첫째 줄에 M개의 문자열 중에 총 몇 개가 집합 S에 포함되어 있는지 출력한다.

## 풀이

### 풀이 핵심 로직
이 문제의 핵심 아이디어는 문자열 집합 S를 해시 세트(set)로 저장하여, 검사할 문자열이 집합 S에 포함되어 있는지를 빠르게 확인하는 것이다. Python의 set 자료형은 평균적으로 O(1) 시간 복잡도로 원소 존재 여부를 확인할 수 있어, 효율적인 탐색이 가능하다.

### 동작 과정
1. 첫째 줄에서 문자열의 개수 N과 M을 입력받는다.
2. N개의 문자열을 입력받아 set 자료형인 `arr`에 저장한다. 이때 중복된 문자열은 자동으로 제거된다.
3. M개의 문자열을 차례대로 입력받으며, 각 문자열이 `arr`에 포함되어 있는지를 확인한다.
4. 포함되어 있는 문자열의 개수를 세어 최종적으로 출력한다.

예를 들어, 입력이 다음과 같다고 가정해보자:
```
5 4
apple
banana
grape
orange
peach
banana
grape
kiwi
apple
```
- 첫째 줄에서 N=5, M=4를 읽는다.
- 다음 5줄에서 각각 "apple", "banana", "grape", "orange", "peach"를 집합 S에 추가한다.
- 이어서 4줄에서 "banana", "grape", "kiwi", "apple"을 검사한다.
- "banana", "grape", "apple"은 집합 S에 포함되어 있으므로, 최종적으로 3을 출력한다.

### 시간 복잡도
이 알고리즘의 시간 복잡도는 O(N + M)이다. N개의 문자열을 집합 S에 추가하는 데 O(N) 시간이 소요되고, M개의 문자열을 검사하는 데 O(M) 시간이 소요된다. 각 검사에서 set의 평균 탐색 시간은 O(1)이다.

## 코드
```python
n, m = map(int, input().split())

arr = set()
for _ in range(n):
    arr.add(input().rstrip())

cnt = 0
for _ in range(m):
    s = input().rstrip()
    if s in arr:
        cnt += 1

print(cnt)
```
