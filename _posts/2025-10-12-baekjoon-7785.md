## 풀이

### 풀이 핵심 로직
주어진 문제는 특정 인원이 'enter'와 'leave' 이벤트를 통해 출입하는 관리 시스템입니다. 이 코드에서는 각 사람의 출입 여부를 관리하기 위해 `defaultdict`를 사용하여 각 사람의 출입 상태를 저장하고, 최종적으로 출입 중인 사람의 목록을 내림차순으로 정렬하여 출력합니다.

### 동작 과정
1. **입력 받기**: 첫 줄에서 총 `n`개의 이벤트 수를 입력받습니다.
2. **사전 초기화**: `defaultdict(int)`를 사용하여 `dic`라는 사전을 초기화합니다. 이는 인원의 출입 상태를 저장하는 데 사용됩니다. 기본값은 0입니다.
3. **이벤트 처리**:
   - `for` 루프를 통해 `n`개의 이벤트를 반복 처리합니다.
   - 각 이벤트를 공백으로 분리하여 `a`(이름)와 `b`('enter' 또는 'leave')를 가져옵니다.
   - 만약 `b`가 'enter'라면 해당 인원의 출입 상태를 1 증가시키고, 'leave'라면 1 감소시킵니다.
4. **출입 중인 인원 확인**: `dic`에서 값이 1인 사람(즉, 현재 출입 중인 인원)을 확인하여 `ans` 리스트에 추가합니다.
5. **정렬 및 출력**: `ans` 리스트를 내림차순으로 정렬한 후, 각 인원을 출력합니다.

**예시 입력**:
```
5
Alice enter
Bob enter
Alice leave
Bob enter
Charlie enter
```
**단계별 설명**:
- `dic`의 상태는 다음과 같습니다:
  - 입력 1: Alice enter → `{'Alice': 1}`
  - 입력 2: Bob enter → `{'Alice': 1, 'Bob': 1}`
  - 입력 3: Alice leave → `{'Alice': 0, 'Bob': 1}`
  - 입력 4: Bob enter → `{'Alice': 0, 'Bob': 2}`
  - 입력 5: Charlie enter → `{'Alice': 0, 'Bob': 2, 'Charlie': 1}`
- 최종적으로 `ans`에 추가되는 인원은 Charlie입니다.
- `ans`를 내림차순 정렬 후 출력 → `Charlie`

### 시간 복잡도
이 알고리즘의 시간 복잡도는 O(n log n)입니다. 여기서 n은 입력 이벤트의 수입니다. 각 이벤트를 처리하는 데 O(1)의 시간이 소요되고, 최종적으로 `ans` 리스트를 정렬하는 데 O(k log k)의 시간이 소요되는데, k는 현재 출입 중인 인원의 수입니다. 최악의 경우 k는 n과 같을 수 있으므로, 전체 복잡도는 O(n log n)입니다.

## 코드
```python
from collections import defaultdict

n = int(input())
dic = defaultdict(int)

for _ in range(n):
    a, b = map(str, input().rstrip().split())
    
    if b == 'enter':
        dic[a] += 1
        
    else:
        dic[a] -= 1
        
ans = []    
for k, v in dic.items():
    if v == 1:
        ans.append(k)
        
ans.sort(reverse=True)
for i in ans:
    print(i)
```
