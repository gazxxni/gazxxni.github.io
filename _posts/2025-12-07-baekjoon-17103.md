## 문제
골드바흐의 추측: 2보다 큰 짝수는 두 소수의 합으로 나타낼 수 있다.


짝수 N을 두 소수의 합으로 나타내는 표현을 골드바흐 파티션이라고 한다. 짝수 N이 주어졌을 때, 골드바흐 파티션의 개수를 구해보자. 두 소수의 순서만 다른 것은 같은 파티션이다.

---

## 입력
첫째 줄에 테스트 케이스의 개수 T (1 ≤ T ≤ 100)가 주어진다. 각 테스트 케이스는 한 줄로 이루어져 있고, 정수 N은 짝수이고, 2

---

## 출력
각각의 테스트 케이스마다 골드바흐 파티션의 수를 출력한다.

## 풀이

### 풀이 핵심 로직
골드바흐의 추측에 따르면, 모든 짝수 N은 두 소수의 합으로 표현될 수 있습니다. 본 문제는 주어진 짝수 N을 두 소수의 합으로 나타내는 경우의 수를 구하는 것입니다. 이를 위해 우리는 에라토스테네스의 체를 사용하여 N 이하의 모든 소수를 미리 구한 후, 두 소수의 조합이 N이 되는 경우를 탐색하여 개수를 세는 방법을 사용합니다.

### 동작 과정
1. **소수 생성**: 먼저, 1,000,000 이하의 모든 소수를 찾기 위해 에라토스테네스의 체를 이용하여 소수 리스트를 만듭니다.
2. **입력 처리**: 테스트 케이스의 수 T를 입력받고, 각 테스트 케이스에 대해 짝수 N을 입력받습니다.
3. **골드바흐 파티션 계산**:
   - N을 반으로 나누어, N/2 이하의 소수만 고려합니다.
   - 각 소수 i에 대해 N-i가 소수인지 확인합니다.
   - N-i가 소수인 경우, cnt를 증가시킵니다.
4. **결과 출력**: 각 테스트 케이스에 대해 골드바흐 파티션의 개수를 출력합니다.

예를 들어, 입력이 N=8일 경우, 소수는 [2, 3, 5, 7]로, 8을 두 소수로 나타내는 경우는 (3, 5)와 (5, 3)로, 순서가 다르지 않으므로 cnt는 1이 됩니다.

### 시간 복잡도
소수를 구하는 과정은 O(n log log n)이며, 각 테스트 케이스에 대해 소수를 점검하는 과정은 O(n)입니다. 따라서 전체 시간 복잡도는 O(n log log n + T * n)입니다. 이 경우 T는 최대 100이므로, 효율적으로 처리할 수 있습니다.

## 코드
```python
def primenum(n):
    arr = [True] * (n + 1)
    arr[0] = arr[1] = False
    
    for i in range(2, int(n ** 0.5) + 1):
        if arr[i]:
            for j in range(i * i, n + 1, i):
                arr[j] = False
                
    brr = []
    for i in range(2, n + 1):
        if arr[i]:
            brr.append(i)
            
    return brr

a = primenum(1000000)
b = set(a)
t = int(input())

for _ in range(t):
    n = int(input())

    cnt = 0

    for i in a:
        if i > n // 2:
            break
        
        if (n - i) in b:
            cnt += 1
            
    print(cnt)
```
