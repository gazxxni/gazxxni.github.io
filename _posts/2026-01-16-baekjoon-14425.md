## 문제
총 N개의 문자열로 이루어진 집합 S가 주어진다.

입력으로 주어지는 M개의 문자열 중에서 집합 S에 포함되어 있는 것이 총 몇 개인지 구하는 프로그램을 작성하시오.

---

## 입력
첫째 줄에 문자열의 개수 N과 M (1 ≤ N ≤ 10,000, 1 ≤ M ≤ 10,000)이 주어진다. 

다음 N개의 줄에는 집합 S에 포함되어 있는 문자열들이 주어진다.

다음 M개의 줄에는 검사해야 하는 문자열들이 주어진다.

입력으로 주어지는 문자열은 알파벳 소문자로만 이루어져 있으며, 길이는 500을 넘지 않는다. 집합 S에 같은 문자열이 여러 번 주어지는 경우는 없다.

---

## 출력
첫째 줄에 M개의 문자열 중에 총 몇 개가 집합 S에 포함되어 있는지 출력한다.

## 풀이

### 풀이 핵심 로직
주어진 문제는 두 개의 문자열 집합이 주어지고, 하나의 집합에서 다른 집합에 포함된 문자열의 개수를 세는 문제입니다. 이때, 문자열 검색의 효율성을 위해 집합(S)을 Python의 `set` 자료형으로 구현하여 O(1) 평균 시간 복잡도로 문자열 포함 여부를 체크합니다.

### 동작 과정
1. 입력으로 문자열 개수 N과 M을 받습니다.
2. N개의 문자열을 읽어 집합 S에 저장합니다.
3. M개의 문자열을 차례로 읽으며, 각 문자열이 집합 S에 포함되어 있는지 확인합니다.
4. 포함되어 있는 경우 카운트를 증가시킵니다.
5. 최종적으로 카운트를 출력합니다.

예시 입력:
```
5 3
apple
banana
orange
grape
melon
banana
grape
kiwi
```
- 첫 번째 줄에서 N=5, M=3을 읽습니다.
- 다음 5개의 문자열(`apple`, `banana`, `orange`, `grape`, `melon`)을 집합 S에 추가합니다.
- 그 후 3개의 문자열(`banana`, `grape`, `kiwi`)을 차례로 검사합니다.
- `banana`와 `grape`는 집합 S에 포함되지만 `kiwi`는 포함되지 않으므로, 최종 카운트는 2가 됩니다.

### 시간 복잡도
O(N + M) - N개의 문자열을 집합에 추가하는 시간과 M개의 문자열을 검사하는 시간이 각각 O(N)과 O(M) 소요되므로, 전체적으로 두 가지 작업의 합으로 O(N + M)입니다.

## 코드
```python
n, m = map(int, input().split())

arr = set()
for _ in range(n):
    arr.add(input().rstrip())

cnt = 0
for _ in range(m):
    s = input().rstrip()
    if s in arr:
        cnt += 1

print(cnt)
```
