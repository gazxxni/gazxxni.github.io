## 문제
철수는 롤케이크를 두 조각으로 잘라서 동생과 한 조각씩 나눠 먹으려고 합니다. 이 롤케이크에는 여러가지 토핑들이 일렬로 올려져 있습니다. 철수와 동생은 롤케이크를 공평하게 나눠먹으려 하는데, 그들은 롤케이크의 크기보다 롤케이크 위에 올려진 토핑들의 종류에 더 관심이 많습니다. 그래서 잘린 조각들의 크기와 올려진 토핑의 개수에 상관없이 각 조각에 동일한 가짓수의 토핑이 올라가면 공평하게 롤케이크가 나누어진 것으로 생각합니다.

예를 들어, 롤케이크에 4가지 종류의 토핑이 올려져 있다고 합시다. 토핑들을 1, 2, 3, 4와 같이 번호로 표시했을 때, 케이크 위에 토핑들이 [1, 2, 1, 3, 1, 4, 1, 2] 순서로 올려져 있습니다. 만약 세 번째 토핑(1)과 네 번째 토핑(3) 사이를 자르면 롤케이크의 토핑은 [1, 2, 1], [3, 1, 4, 1, 2]로 나뉘게 됩니다. 철수가 [1, 2, 1]이 놓인 조각을, 동생이 [3, 1, 4, 1, 2]가 놓인 조각을 먹게 되면 철수는 두 가지 토핑(1, 2)을 맛볼 수 있지만, 동생은 네 가지 토핑(1, 2, 3, 4)을 맛볼 수 있으므로, 이는 공평하게 나누어진 것이 아닙니다. 만약 롤케이크의 네 번째 토핑(3)과 다섯 번째 토핑(1) 사이를 자르면 [1, 2, 1, 3], [1, 4, 1, 2]로 나뉘게 됩니다. 이 경우 철수는 세 가지 토핑(1, 2, 3)을, 동생도 세 가지 토핑(1, 2, 4)을 맛볼 수 있으므로, 이는 공평하게 나누어진 것입니다. 공평하게 롤케이크를 자르는 방법은 여러가지 일 수 있습니다. 위의 롤케이크를 [1, 2, 1, 3, 1], [4, 1, 2]으로 잘라도 공평하게 나뉩니다. 어떤 경우에는 롤케이크를 공평하게 나누지 못할 수도 있습니다.

롤케이크에 올려진 토핑들의 번호를 저장한 정수 배열 topping이 매개변수로 주어질 때, 롤케이크를 공평하게 자르는 방법의 수를 return 하도록 solution 함수를 완성해주세요.



제한사항


1 ≤ topping의 길이 ≤ 1,000,000


1 ≤ topping의 원소 ≤ 10,000





입출력 예

        
topping
result


        
[1, 2, 1, 3, 1, 4, 1, 2]
2


[1, 2, 3, 1, 4]
0


      


입출력 예 설명

입출력 예 #1


롤케이크를 [1, 2, 1, 3], [1, 4, 1, 2] 또는 [1, 2, 1, 3, 1], [4, 1, 2]와 같이 자르면 철수와 동생은 각각 세 가지 토핑을 맛볼 수 있습니다. 이 경우 공평하게 롤케이크를 나누는 방법은 위의 두 가지만 존재합니다.


입출력 예 #2


롤케이크를 공평하게 나눌 수 없습니다.

## 풀이
### 풀이 핵심 로직
롤케이크를 공평하게 자르기 위해서는, 자른 조각의 왼쪽과 오른쪽에 있는 토핑의 종류 수가 동일해야 합니다. 이를 위해 각 조각의 토핑 종류 수를 카운트하여 비교합니다. 왼쪽 조각에 있는 토핑 종류의 수가 증가할 때마다 오른쪽 조각에 있는 토핑 종류의 수를 감소시키며, 두 수가 같을 때마다 카운트를 증가시킵니다.

### 동작 과정
주어진 입력 `topping`이 [1, 2, 1, 3, 1, 4, 1, 2]일 때, 아래와 같이 진행됩니다.

1. **초기화**: 
   - `right`: 전체 토핑의 카운트를 저장합니다. 초기 값은 `Counter({1: 4, 2: 2, 3: 1, 4: 1})`입니다.
   - `left`: 현재 왼쪽 조각에 있는 토핑의 종류를 저장하는 집합입니다. 초기 값은 빈 집합입니다.
   - `r_cnt`: 오른쪽 조각의 토핑 종류 수 (초기값 4).
   - `l_cnt`: 왼쪽 조각의 토핑 종류 수 (초기값 0).
   - `answer`: 공평하게 나눌 수 있는 방법의 수 (초기값 0).

2. **반복문 시작**: `topping` 배열을 순회하면서 각 토핑을 처리합니다.
   - 첫 번째 토핑(1): `left`에 추가, `l_cnt` 증가 (1), `right`에서 1 감소, `r_cnt` 감소 (3). 두 수가 다르므로 `answer`는 증가하지 않음.
   - 두 번째 토핑(2): `left`에 추가, `l_cnt` 증가 (2), `right`에서 2 감소, `r_cnt` 감소 (2). 두 수가 같으므로 `answer` 증가 (1).
   - 세 번째 토핑(1): `left`에 추가, `right`에서 1 감소 (2). 두 수가 다르므로 `answer`는 증가하지 않음.
   - 네 번째 토핑(3): `left`에 추가, `l_cnt` 증가 (3), `right`에서 3 감소, `r_cnt` 감소 (1). 두 수가 다르므로 `answer`는 증가하지 않음.
   - 다섯 번째 토핑(1): `right`에서 1 감소 (1). 두 수가 다르므로 `answer`는 증가하지 않음.
   - 여섯 번째 토핑(4): `left`에 추가, `l_cnt` 증가 (4), `right`에서 4 감소 (0). 두 수가 같지 않으므로 `answer`는 증가하지 않음.
   - 일곱 번째 토핑(1): `right`에서 1 감소 (0). 두 수가 같지 않으므로 `answer`는 증가하지 않음.
   - 여덟 번째 토핑(2): `right`에서 2 감소 (0). 두 수가 같지 않으므로 `answer`는 증가하지 않음.

3. **결과**: 최종적으로 `answer`는 2가 되며, 이는 롤케이크를 공평하게 나눌 수 있는 방법의 수입니다.

### 시간 복잡도
이 알고리즘의 시간 복잡도는 O(N)입니다. 여기서 N은 `topping`의 길이입니다. 각 토핑을 한 번씩만 순회하면서 카운트를 업데이트하고 비교하므로 효율적입니다.

## 코드
```python
from collections import Counter
def solution(topping):
    answer = 0
    right = Counter(topping)
    left = set()
    
    r_cnt = len(right)
    l_cnt = 0
    for t in topping:
        if t not in left:
            left.add(t)
            l_cnt += 1
            
        right[t] -= 1
        if right[t] == 0:
            r_cnt -= 1
            
        if r_cnt == l_cnt:
            answer += 1
        
    return answer
```
