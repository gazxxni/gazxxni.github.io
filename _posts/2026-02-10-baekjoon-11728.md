## 문제 링크
[https://www.acmicpc.net/problem/11728](https://www.acmicpc.net/problem/11728)

## 문제
정렬되어있는 두 배열 A와 B가 주어진다. 두 배열을 합친 다음 정렬해서 출력하는 프로그램을 작성하시오.

## 입력
첫째 줄에 배열 A의 크기 N, 배열 B의 크기 M이 주어진다. (1 ≤ N, M ≤ 1,000,000)

둘째 줄에는 배열 A의 내용이, 셋째 줄에는 배열 B의 내용이 주어진다. 배열에 들어있는 수는 절댓값이 109보다 작거나 같은 정수이다.

## 출력
첫째 줄에 두 배열을 합친 후 정렬한 결과를 출력한다.

## 풀이
### 풀이 핵심 로직
주어진 두 배열 A와 B는 이미 정렬되어 있으므로, 두 포인터 기법을 사용하여 O(N + M) 시간 복잡도로 두 배열을 합칠 수 있다. 이를 통해 메모리와 시간 효율성을 높일 수 있으며, 마지막으로 합쳐진 배열을 출력하면 된다.

### 동작 과정
1. 입력값을 읽어들이고, 두 배열 A와 B를 각각 정수 리스트로 변환한다.
2. 두 포인터를 각각 배열 A와 B의 시작 위치에 두고, 두 배열을 비교하면서 작은 값을 결과 리스트에 추가한다.
3. 한 배열의 요소를 모두 사용한 후, 남은 배열의 요소를 결과 리스트에 추가한다.
4. 최종 결과 리스트를 출력한다.

예시 입력:
```
5 3
1 3 5 7 9
2 4 6
```
단계별 설명:
- 초기 포인터 위치: A 포인터(0), B 포인터(0)
- A[0] (1)과 B[0] (2) 비교: 1이 더 작으므로 결과 리스트에 1 추가, A 포인터(1)로 이동
- A[1] (3)과 B[0] (2) 비교: 2가 더 작으므로 결과 리스트에 2 추가, B 포인터(1)로 이동
- A[1] (3)과 B[1] (4) 비교: 3이 더 작으므로 결과 리스트에 3 추가, A 포인터(2)로 이동
- A[2] (5)와 B[1] (4) 비교: 4가 더 작으므로 결과 리스트에 4 추가, B 포인터(2)로 이동
- A[2] (5)와 B[2] (6) 비교: 5가 더 작으므로 결과 리스트에 5 추가, A 포인터(3)로 이동
- A[3] (7)와 B[2] (6) 비교: 6가 더 작으므로 결과 리스트에 6 추가, B 포인터(3)로 이동 (B 배열의 끝)
- 나머지 A 배열 요소(7, 9)를 결과 리스트에 추가
- 최종 결과: [1, 2, 3, 4, 5, 6, 7, 9]

### 시간 복잡도
O(N + M) - 두 배열의 모든 요소를 한 번씩만 방문하므로, 최악의 경우에도 두 배열의 크기 N과 M에 비례하는 선형 시간 복잡도를 가진다.

## 코드
```python
n, m = map(int, input().split())
arr = list(map(int, input().split()))
brr = list(map(int, input().split()))

crr = arr + brr
crr.sort()
print(*crr)
```
