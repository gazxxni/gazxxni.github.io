## 풀이

### 풀이 핵심 로직
이 문제는 세 개의 주기(15, 28, 19)로 이루어진 날짜를 계산하는 문제이다. 주기마다 각각의 연도는 1부터 시작하므로, 각 주기마다의 연도를 계산하여 세 주기가 모두 주어진 `e`, `s`, `m`과 일치하는 최소의 연도를 찾는다. 이를 위해 1부터 시작하여 각 주기별 조건을 만족할 때까지 반복한다.

### 동작 과정
예를 들어, 입력으로 `e = 1, s = 1, m = 1`이 주어진 경우:
1. `i = 1`: (1 - 1) % 15 + 1 = 1, (1 - 1) % 28 + 1 = 1, (1 - 1) % 19 + 1 = 1 → 모두 일치하므로 출력: `1`.
   
또 다른 예로, 입력으로 `e = 2, s = 3, m = 4`가 주어진 경우:
1. `i = 1`: (1 - 1) % 15 + 1 = 1, (1 - 1) % 28 + 1 = 1, (1 - 1) % 19 + 1 = 1 → 불일치
2. `i = 2`: (2 - 1) % 15 + 1 = 2, (2 - 1) % 28 + 1 = 2, (2 - 1) % 19 + 1 = 2 → 불일치
3. `i = 3`: (3 - 1) % 15 + 1 = 3, (3 - 1) % 28 + 1 = 3, (3 - 1) % 19 + 1 = 3 → 불일치
4. ...
5. `i = 28`: (28 - 1) % 15 + 1 = 13, (28 - 1) % 28 + 1 = 1, (28 - 1) % 19 + 1 = 9 → 불일치
6. ...
7. `i = 19`: (19 - 1) % 15 + 1 = 4, (19 - 1) % 28 + 1 = 19, (19 - 1) % 19 + 1 = 1 → 불일치
8. ...
9. `i = 69`: (69 - 1) % 15 + 1 = 9, (69 - 1) % 28 + 1 = 17, (69 - 1) % 19 + 1 = 5 → 불일치
10. `i = 107`: (107 - 1) % 15 + 1 = 14, (107 - 1) % 28 + 1 = 2, (107 - 1) % 19 + 1 = 11 → 불일치
11. `i = 123`: (123 - 1) % 15 + 1 = 1, (123 - 1) % 28 + 1 = 1, (123 - 1) % 19 + 1 = 1 → 모두 일치하므로 출력: `123`.

### 시간 복잡도
이 알고리즘의 시간 복잡도는 O(N)이다. 여기서 N은 세 주기(15, 28, 19)의 최소공배수(LCM)에 해당하는 최대 반복 횟수를 의미한다. 따라서 최악의 경우 약 7980회의 반복이 발생할 수 있다.

## 코드
```python
e, s, m = map(int, input().split())
i = 1

if e == s and s == m:
    print(e)

else:
    i = 1
    while True:
        if ((i - 1) % 15 + 1 == e and
            (i - 1) % 28 + 1 == s and
            (i - 1) % 19 + 1 == m):
            print(i)

            break
        i += 1

        
```
