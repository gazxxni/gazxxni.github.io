## 문제
총 N개의 문자열로 이루어진 집합 S가 주어진다.

입력으로 주어지는 M개의 문자열 중에서 집합 S에 포함되어 있는 것이 총 몇 개인지 구하는 프로그램을 작성하시오.

---

## 입력
첫째 줄에 문자열의 개수 N과 M (1 ≤ N ≤ 10,000, 1 ≤ M ≤ 10,000)이 주어진다. 

다음 N개의 줄에는 집합 S에 포함되어 있는 문자열들이 주어진다.

다음 M개의 줄에는 검사해야 하는 문자열들이 주어진다.

입력으로 주어지는 문자열은 알파벳 소문자로만 이루어져 있으며, 길이는 500을 넘지 않는다. 집합 S에 같은 문자열이 여러 번 주어지는 경우는 없다.

---

## 출력
첫째 줄에 M개의 문자열 중에 총 몇 개가 집합 S에 포함되어 있는지 출력한다.

## 풀이

### 풀이 핵심 로직
문제는 두 개의 문자열 집합이 주어졌을 때, 두 번째 집합의 문자열이 첫 번째 집합에 몇 개나 포함되어 있는지를 세는 것입니다. 이를 위해 첫 번째 집합을 파이썬의 `set` 자료형으로 구현하여 O(1) 평균 시간 복잡도로 문자열 포함 여부를 검사할 수 있습니다.

### 동작 과정
1. 첫 번째 줄에서 N과 M을 입력받습니다. N은 집합 S의 문자열 개수, M은 검사할 문자열 개수입니다.
2. N개의 문자열을 입력받아 집합 S를 생성합니다. 이때 `set`을 사용하여 중복을 자동으로 제거합니다.
3. 이후 M개의 문자열을 입력받으며, 각각의 문자열이 집합 S에 존재하는지 확인합니다.
4. 존재한다면 카운트를 증가시키고, 최종적으로 카운트를 출력합니다.

예시 입력:
```
5 3
apple
banana
orange
grape
peach
apple
grape
kiwi
```
단계별 설명:
- N=5, M=3으로 설정.
- 집합 S는 `{'apple', 'banana', 'orange', 'grape', 'peach'}`로 구성됩니다.
- 검사할 문자열은 `['apple', 'grape', 'kiwi']`입니다.
- 'apple'은 집합 S에 존재하므로 카운트 1.
- 'grape'도 집합 S에 존재하므로 카운트 2.
- 'kiwi'는 집합 S에 존재하지 않으므로 카운트 유지.
- 최종적으로 카운트는 2가 되어 출력됩니다.

### 시간 복잡도
이 알고리즘의 시간 복잡도는 O(N + M)입니다. 여기서 N은 집합 S의 문자열 개수, M은 검사할 문자열 개수입니다. 집합 S를 만드는 데 O(N) 시간이 걸리고, 각 문자열 포함 여부를 확인하는 데 평균 O(1) 시간이 걸리므로 총 O(M) 시간이 추가됩니다.

## 코드
```python
n, m = map(int, input().split())

arr = set()
for _ in range(n):
    arr.add(input().rstrip())

cnt = 0
for _ in range(m):
    s = input().rstrip()
    if s in arr:
        cnt += 1

print(cnt)
```
