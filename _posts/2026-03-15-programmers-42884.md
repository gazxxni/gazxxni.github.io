## 문제
고속도로를 이동하는 모든 차량이 고속도로를 이용하면서 단속용 카메라를 한 번은 만나도록 카메라를 설치하려고 합니다.

고속도로를 이동하는 차량의 경로 routes가 매개변수로 주어질 때, 모든 차량이 한 번은 단속용 카메라를 만나도록 하려면 최소 몇 대의 카메라를 설치해야 하는지를 return 하도록 solution 함수를 완성하세요.

제한사항


차량의 대수는 1대 이상 10,000대 이하입니다.
routes에는 차량의 이동 경로가 포함되어 있으며 routes[i][0]에는 i번째 차량이 고속도로에 진입한 지점, routes[i][1]에는 i번째 차량이 고속도로에서 나간 지점이 적혀 있습니다.
차량의 진입/진출 지점에 카메라가 설치되어 있어도 카메라를 만난것으로 간주합니다.
차량의 진입 지점, 진출 지점은 -30,000 이상 30,000 이하입니다.


입출력 예

        
routes
return


        
[[-20,-15], [-14,-5], [-18,-13], [-5,-3]]
2


      
입출력 예 설명

-5 지점에 카메라를 설치하면 두 번째, 네 번째 차량이 카메라를 만납니다.

-15 지점에 카메라를 설치하면 첫 번째, 세 번째 차량이 카메라를 만납니다.

## 풀이
### 풀이 핵심 로직
이 문제의 핵심 아이디어는 차량의 경로를 종료 지점 기준으로 정렬한 후, 각 차량의 진입 지점과 현재 카메라 설치 지점을 비교하여 카메라를 효율적으로 설치하는 것입니다. 차량의 종료 지점을 기준으로 카메라를 설치하면 최대한 많은 차량을 한 번의 카메라로 감지할 수 있습니다.

### 동작 과정
1. 주어진 `routes` 리스트를 차량이 도로를 떠나는 지점(종료 지점) 기준으로 정렬합니다. 이렇게 하면 종료 지점이 가까운 차량부터 처리할 수 있습니다.
   
   예를 들어, `routes = [[-20,-15], [-14,-5], [-18,-13], [-5,-3]]`일 경우, 정렬 후 결과는 `[[-20, -15], [-18, -13], [-14, -5], [-5, -3]]`가 됩니다.

2. 카메라를 설치할 위치를 초기값으로 매우 작은 숫자(`-99999999`)로 설정하고, 카메라 개수를 0으로 초기화합니다.

3. 정렬된 각 차량의 경로를 순회하면서:
   - 현재 차량의 진입 지점이 마지막으로 설치된 카메라 위치보다 크다면, 이는 현재 차량이 카메라의 범위를 벗어난 것이므로 새로운 카메라를 설치합니다.
   - 카메라 위치는 현재 차량의 종료 지점으로 업데이트하고, 카메라 개수를 1 증가시킵니다.

4. 모든 차량을 순회한 후 카메라 개수를 반환합니다.

위 과정을 통해 모든 차량이 최소한 하나의 카메라를 만날 수 있도록 설치할 수 있습니다.

### 시간 복잡도
이 알고리즘의 시간 복잡도는 O(n log n)입니다. 차량 경로를 정렬하는 데 O(n log n)이 소요되고, 이후 차량 경로를 한 번 순회하는 데 O(n)이 소요되므로 전체 시간 복잡도는 O(n log n)입니다.

## 코드
```python
def solution(routes):
    routes.sort(key=lambda x:x[1])
    camera = -99999999
    cnt = 0
    for route in routes:
        if route[0] > camera:
            camera = route[1]
            cnt +=1
    
    
    return cnt
```
