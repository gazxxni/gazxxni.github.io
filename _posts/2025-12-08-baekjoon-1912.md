## 문제
n개의 정수로 이루어진 임의의 수열이 주어진다. 우리는 이 중 연속된 몇 개의 수를 선택해서 구할 수 있는 합 중 가장 큰 합을 구하려고 한다. 단, 수는 한 개 이상 선택해야 한다.

예를 들어서 10, -4, 3, 1, 5, 6, -35, 12, 21, -1 이라는 수열이 주어졌다고 하자. 여기서 정답은 12+21인 33이 정답이 된다.

---

## 입력
첫째 줄에 정수 n(1 ≤ n ≤ 100,000)이 주어지고 둘째 줄에는 n개의 정수로 이루어진 수열이 주어진다. 수는 -1,000보다 크거나 같고, 1,000보다 작거나 같은 정수이다.

---

## 출력
첫째 줄에 답을 출력한다.

## 풀이

### 풀이 핵심 로직
이 문제는 연속된 부분 배열의 최대 합을 찾는 문제로, 다이나믹 프로그래밍을 이용하여 해결할 수 있다. 현재 원소를 포함한 연속 부분 배열의 최대 합을 구하기 위해, 이전 원소까지의 최대 합과 현재 원소를 비교하여 최대값을 업데이트한다.

### 동작 과정
예를 들어, 주어진 수열이 `[10, -4, 3, 1, 5, 6, -35, 12, 21, -1]`라고 하자. 이 경우의 단계별 설명은 다음과 같다:

1. `dp[0]`은 첫 번째 원소인 `10`으로 초기화한다. 현재 최대 합 `ans`도 `10`으로 설정한다.
2. `i = 1`: `arr[1] = -4`
   - `dp[1] = max(-4, 10 - 4) = max(-4, 6) = 6`
   - `ans = max(10, 6) = 10`
3. `i = 2`: `arr[2] = 3`
   - `dp[2] = max(3, 6 + 3) = max(3, 9) = 9`
   - `ans = max(10, 9) = 10`
4. `i = 3`: `arr[3] = 1`
   - `dp[3] = max(1, 9 + 1) = max(1, 10) = 10`
   - `ans = max(10, 10) = 10`
5. `i = 4`: `arr[4] = 5`
   - `dp[4] = max(5, 10 + 5) = max(5, 15) = 15`
   - `ans = max(10, 15) = 15`
6. `i = 5`: `arr[5] = 6`
   - `dp[5] = max(6, 15 + 6) = max(6, 21) = 21`
   - `ans = max(15, 21) = 21`
7. `i = 6`: `arr[6] = -35`
   - `dp[6] = max(-35, 21 - 35) = max(-35, -14) = -14`
   - `ans = max(21, -14) = 21`
8. `i = 7`: `arr[7] = 12`
   - `dp[7] = max(12, -14 + 12) = max(12, -2) = 12`
   - `ans = max(21, 12) = 21`
9. `i = 8`: `arr[8] = 21`
   - `dp[8] = max(21, 12 + 21) = max(21, 33) = 33`
   - `ans = max(21, 33) = 33`
10. `i = 9`: `arr[9] = -1`
    - `dp[9] = max(-1, 33 - 1) = max(-1, 32) = 32`
    - `ans = max(33, 32) = 33`

최종적으로, 연속 부분 배열의 최대 합은 `33`이다.

### 시간 복잡도
이 알고리즘의 시간 복잡도는 O(n)이다. 각 원소를 한 번씩만 방문하여 최대 합을 계산하므로, 입력 수열의 길이에 비례한 시간만 소요된다.

## 코드
```python
import sys
input = sys.stdin.readline

n = int(input())
arr = list(map(int, input().split()))

dp = [0] * (n + 1)
dp[0] = arr[0]
ans = arr[0]

for i in range(1, n):
    dp[i] = max(arr[i], dp[i - 1] + arr[i])
    ans = max(ans, dp[i])


print(ans)
```
