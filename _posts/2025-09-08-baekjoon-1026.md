## 문제 링크
[https://www.acmicpc.net/problem/1026](https://www.acmicpc.net/problem/1026)

## 풀이

### 풀이 핵심 로직
주어진 두 리스트 `arr`와 `brr`에서 각 요소의 곱의 합을 최소화하기 위해, `arr`은 오름차순으로 정렬하고 `brr`은 내림차순으로 정렬합니다. 이를 통해 가장 작은 수와 가장 큰 수의 곱을 곱하게 되어 결과적으로 곱의 합이 최소화됩니다.

### 동작 과정
1. 입력으로 받은 정수 `n`을 통해 두 개의 리스트를 읽어옵니다. 예를 들어, `n = 3`, `arr = [1, 2, 3]`, `brr = [4, 5, 6]`이라고 가정합니다.
2. `arr` 리스트는 오름차순으로 정렬되어 `[1, 2, 3]`가 되고, `brr` 리스트는 내림차순으로 정렬되어 `[6, 5, 4]`가 됩니다.
3. 각 인덱스에 대해 두 리스트의 요소를 곱하여 합산합니다:
   - `1 * 6 = 6`
   - `2 * 5 = 10`
   - `3 * 4 = 12`
4. 이 값들을 모두 더하면 `6 + 10 + 12 = 28`이 됩니다.
5. 최종적으로 `28`을 출력합니다.

### 시간 복잡도
정렬을 수행하는 데 O(n log n) 시간이 소요되고, 합산 과정은 O(n) 시간이 소요됩니다. 따라서 전체 시간 복잡도는 O(n log n)입니다.

---

## 코드
```python
n = int(input())

arr = list(map(int, input().split()))
brr = list(map(int, input().split()))

arr.sort()
brr.sort(reverse=True)

a = 0
for i in range(n):
    a += arr[i] * brr[i]
    
print(a)
```
