## 문제
은민이는 4와 7을 좋아하고, 나머지 숫자는 싫어한다. 금민수는 어떤 수가 4와 7로만 이루어진 수를 말한다.

A와 B가 주어졌을 때, A보다 크거나 같고, B보다 작거나 같은 자연수 중에 금민수인 것의 개수를 출력하는 프로그램을 작성하시오.

---

## 입력
첫째 줄에 A와 B가 주어진다. A는 1보다 크거나 같고, 1,000,000,000보다 작거나 같은 자연수이다. B는 A보다 크거나 같고, 1,000,000,000보다 작거나 같은 자연수이다.

---

## 출력
첫째 줄에 A보다 크거나 같고, B보다 작거나 같은 자연수 중에 금민수인 것의 개수를 출력한다.

## 풀이

### 풀이 핵심 로직
이 문제는 4와 7로만 이루어진 수, 즉 '금민수'를 생성하여 주어진 범위 A와 B 사이에 있는 금민수를 세는 문제입니다. 브루트포스 방식으로 4와 7의 조합으로 가능한 모든 금민수를 생성하여 범위에 맞는 수를 카운트합니다.

### 동작 과정
1. 입력으로 A와 B가 주어집니다. 예를 들어 A = 1, B = 100.
2. 금민수를 생성하기 위해, 각 자리수에 대해 4와 7을 추가하는 방식으로 모든 조합을 만듭니다. 즉, 1자리 수는 4, 7, 2자리 수는 44, 47, 74, 77과 같이 생성합니다.
3. 각 자리수에 대한 금민수를 리스트에 저장하고, 생성한 금민수 중 A 이상 B 이하인 숫자를 세기 위해 각각을 확인합니다.
4. 최종적으로 범위에 포함되는 금민수의 개수를 출력합니다.

예를 들어, A = 1, B = 100인 경우:
- 생성된 금민수: 4, 7, 44, 47, 74, 77
- 이 중 A와 B 사이에 있는 수: 4, 7, 44, 47, 74, 77 (모두 포함)
- 출력 결과: 6

### 시간 복잡도
이 알고리즘의 시간 복잡도는 O(2^n)입니다. 이는 n이 금민수의 최대 자리수(10자리 수를 고려할 경우 n=10)일 때 가능한 조합의 수에 해당합니다. 4와 7로 이루어진 모든 조합을 생성하기 때문에, 최악의 경우 2^10 = 1024개의 금민수를 생성하게 됩니다.

## 코드
```python
import sys
input = sys.stdin.readline

a, b = map(int, input().split())
arr = [[] for _ in range(10)]
brr = [[] for _ in range(10)]
arr[0].append('4')
brr[0].append('7')

for i in range(1,10):
    for j in arr[i - 1]:
        arr[i].append('4' + j)
        arr[i].append('7' + j)
    for k in brr[i - 1]:
        brr[i].append('4' + k)
        brr[i].append('7' + k)

aa = len(str(a))
bb = len(str(b))
cnt = 0
for i in range(10):
    for j in arr[i]:
        if a <= int(j) <= b:
            cnt += 1
    
    for k in brr[i]:
        if a <= int(k) <= b:
            cnt += 1
            
print(cnt)
```
