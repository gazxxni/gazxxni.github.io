## 문제
은민이는 4와 7을 좋아하고, 나머지 숫자는 싫어한다. 금민수는 어떤 수가 4와 7로만 이루어진 수를 말한다.

A와 B가 주어졌을 때, A보다 크거나 같고, B보다 작거나 같은 자연수 중에 금민수인 것의 개수를 출력하는 프로그램을 작성하시오.

---

## 입력
첫째 줄에 A와 B가 주어진다. A는 1보다 크거나 같고, 1,000,000,000보다 작거나 같은 자연수이다. B는 A보다 크거나 같고, 1,000,000,000보다 작거나 같은 자연수이다.

---

## 출력
첫째 줄에 A보다 크거나 같고, B보다 작거나 같은 자연수 중에 금민수인 것의 개수를 출력한다.

## 풀이

### 풀이 핵심 로직
문제는 A와 B 사이의 금민수(4와 7로만 이루어진 수)의 개수를 구하는 것이다. 금민수를 생성하기 위해 브루트포스 방법을 사용하여 4와 7의 조합으로 가능한 모든 수를 만들어내고, 그 수가 주어진 범위 내에 있는지를 확인하여 개수를 세는 방식으로 해결한다.

### 동작 과정
1. 입력으로 A와 B가 주어진다. 예를 들어, A = 10, B = 100이라고 가정하자.
2. 금민수를 생성하기 위해 최대 10자리 수까지 고려하여 4와 7의 조합으로 가능한 모든 수를 만든다. 
   - 1자리: 4, 7
   - 2자리: 44, 47, 74, 77
   - 3자리: 444, 447, 474, 477, 744, 747, 774, 777
   - ... 이와 같이 계속 확장한다.
3. 생성된 금민수 중에서 A(10) 이상 B(100) 이하인 수를 확인하고, 조건을 만족하는 수의 개수를 센다.
4. 최종적으로 금민수의 개수를 출력한다.

예시 입력: `10 100`
- 생성된 금민수: 4, 7, 44, 47, 74, 77, 444, 447, 474, 477, 744, 747, 774, 777
- 범위 내 금민수: 44, 47, 74, 77 (총 4개)
- 출력: `4`

### 시간 복잡도
이 알고리즘의 시간 복잡도는 O(2^N)이다. 여기서 N은 금민수의 자리수이며, 최대 10자리까지 고려하므로 실제로는 2^10 = 1024번 정도의 연산을 수행하게 된다. 각 금민수에 대해 A와 B 범위 내에 있는지를 확인하는 연산이 포함되어 있지만, 금민수의 개수가 상대적으로 적기 때문에 효율적으로 처리된다.

## 코드
```python
import sys
input = sys.stdin.readline

a, b = map(int, input().split())
arr = [[] for _ in range(10)]
brr = [[] for _ in range(10)]
arr[0].append('4')
brr[0].append('7')

for i in range(1,10):
    for j in arr[i - 1]:
        arr[i].append('4' + j)
        arr[i].append('7' + j)
    for k in brr[i - 1]:
        brr[i].append('4' + k)
        brr[i].append('7' + k)

aa = len(str(a))
bb = len(str(b))
cnt = 0
for i in range(10):
    for j in arr[i]:
        if a <= int(j) <= b:
            cnt += 1
    
    for k in brr[i]:
        if a <= int(k) <= b:
            cnt += 1
            
print(cnt)
```
