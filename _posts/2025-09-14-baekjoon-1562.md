## 문제
45656이란 수를 보자.

이 수는 인접한 모든 자리의 차이가 1이다. 이런 수를 계단 수라고 한다.

N이 주어질 때, 길이가 N이면서 0부터 9까지 숫자가 모두 등장하는 계단 수가 총 몇 개 있는지 구하는 프로그램을 작성하시오. 0으로 시작하는 수는 계단수가 아니다.

---

## 입력
첫째 줄에 N이 주어진다. N은 1보다 크거나 같고, 100보다 작거나 같은 자연수이다.

---

## 출력
첫째 줄에 정답을 1,000,000,000으로 나눈 나머지를 출력한다.

## 풀이

### 풀이 핵심 로직
이 문제는 계단 수의 길이와 숫자가 모두 등장하는 조건을 만족하기 위해 다이나믹 프로그래밍과 비트마스킹을 사용합니다. 각 자릿수에서 가능한 숫자와 이미 사용된 숫자의 상태를 비트마스크로 표현하여, 각 길이와 마지막 숫자에 대해 가능한 조합을 저장합니다. 최종적으로 길이가 N이고 모든 숫자가 포함된 경우의 수를 구합니다.

### 동작 과정
1. **초기화**: 길이가 1인 계단 수를 초기화합니다. 1부터 9까지의 숫자로 시작하는 경우를 설정하고, 이때의 비트마스크는 해당 숫자만 포함된 상태로 설정합니다.
   
2. **상태 전이**: 각 길이 i에 대해, 마지막 숫자 j와 현재 상태 k를 순회합니다.
   - **Case 1**: 다음 숫자가 j-1인 경우, j가 0보다 클 때만 가능하므로 새로운 숫자와 비트마스크를 업데이트합니다.
   - **Case 2**: 다음 숫자가 j+1인 경우, j가 9보다 작을 때만 가능하므로 마찬가지로 업데이트합니다.

3. **결과 집계**: 길이가 N이고 모든 숫자가 사용된 경우의 수를 집계합니다. 모든 숫자가 사용되었다는 것은 비트마스크가 `(1 << 10) - 1`인 경우입니다.

4. **출력**: 최종 결과를 1,000,000,000으로 나눈 나머지를 출력합니다.

예시 입력으로 N=3일 때:
- 초기화 후 (예: j=1) 상태는 `dp[1][1][2] = 1`이 됩니다.
- j가 1일 때 j-1로 가면 0, j+1로 가면 2로 이어지며, 각각의 경우에 대해 비트마스크가 업데이트됩니다.
- 최종적으로 N=3에 도달했을 때 모든 비트가 1로 설정된 상태에서의 경우의 수를 집계합니다.

### 시간 복잡도
이 알고리즘의 시간 복잡도는 O(N * 10 * 2^10)입니다. 여기서 N은 숫자의 길이, 10은 마지막 숫자(0-9)의 종류, 2^10은 비트마스크의 가능한 상태 수입니다. N이 최대 100이므로, 이 복잡도는 허용 가능한 수준입니다.

## 코드
```python
import sys
input = sys.stdin.readline

MOD = 1_000_000_000
n = int(input())
dp = [[[0] * 1024 for _ in range(10)] for _ in range(n + 1)]

# 길이 1의 계단 수 초기화
for j in range(1, 10):
    dp[1][j][1 << j] = 1

for i in range(1, n):
    for j in range(10):
        for k in range(1 << 10):
            if dp[i][j][k] == 0:
                continue
            
            # Case 1: 다음 숫자가 j-1일 경우
            if j > 0:
                new_digit = j - 1
                new_mask = k | (1 << new_digit)
                dp[i + 1][new_digit][new_mask] = (dp[i + 1][new_digit][new_mask] + dp[i][j][k]) % MOD
            
            # Case 2: 다음 숫자가 j+1일 경우
            if j < 9:
                new_digit = j + 1
                new_mask = k | (1 << new_digit)
                dp[i + 1][new_digit][new_mask] = (dp[i + 1][new_digit][new_mask] + dp[i][j][k]) % MOD

result = 0
for j in range(10):
    result = (result + dp[n][j][(1 << 10) - 1]) % MOD
    
print(result)
```
