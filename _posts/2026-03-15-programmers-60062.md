## 문제
레스토랑을 운영하고 있는 "스카피"는 레스토랑 내부가 너무 낡아 친구들과 함께 직접 리모델링 하기로 했습니다. 레스토랑이 있는 곳은 스노우타운으로 매우 추운 지역이어서 내부 공사를 하는 도중에 주기적으로 외벽의 상태를 점검해야 할 필요가 있습니다.

레스토랑의 구조는 완전히 동그란 모양이고 외벽의 총 둘레는 n미터이며, 외벽의 몇몇 지점은 추위가 심할 경우 손상될  수도 있는 취약한 지점들이 있습니다. 따라서 내부 공사 도중에도 외벽의 취약 지점들이 손상되지 않았는 지, 주기적으로 친구들을 보내서 점검을 하기로 했습니다. 다만, 빠른 공사 진행을 위해 점검 시간을 1시간으로 제한했습니다. 친구들이 1시간 동안 이동할 수 있는 거리는 제각각이기 때문에, 최소한의 친구들을 투입해 취약 지점을 점검하고 나머지 친구들은 내부 공사를 돕도록 하려고 합니다. 편의 상 레스토랑의 정북 방향 지점을 0으로 나타내며, 취약 지점의 위치는 정북 방향 지점으로부터 시계 방향으로 떨어진 거리로 나타냅니다. 또, 친구들은 출발 지점부터 시계, 혹은 반시계 방향으로 외벽을 따라서만 이동합니다.

외벽의 길이 n, 취약 지점의 위치가 담긴 배열 weak, 각 친구가 1시간 동안 이동할 수 있는 거리가 담긴 배열 dist가 매개변수로 주어질 때, 취약 지점을 점검하기 위해 보내야 하는 친구 수의 최소값을 return 하도록 solution 함수를 완성해주세요.

제한사항


n은 1 이상 200 이하인 자연수입니다.
weak의 길이는 1 이상 15 이하입니다.


서로 다른 두 취약점의 위치가 같은 경우는 주어지지 않습니다.
취약 지점의 위치는 오름차순으로 정렬되어 주어집니다.
weak의 원소는 0 이상 n - 1 이하인 정수입니다.

dist의 길이는 1 이상 8 이하입니다.


dist의 원소는 1 이상 100 이하인 자연수입니다.

친구들을 모두 투입해도 취약 지점을 전부 점검할 수 없는 경우에는 -1을 return 해주세요.

## 풀이
### 풀이 핵심 로직
이 문제는 외벽의 취약 지점을 점검하기 위해 최소한의 친구 수를 투입하는 문제로, 친구들이 이동할 수 있는 거리와 취약 지점의 위치를 고려하여 가능한 모든 조합을 탐색합니다. 이를 위해 각 친구의 이동을 시뮬레이션하고, 모든 취약 지점을 점검할 수 있는지 확인합니다. 최종적으로 가장 적은 수의 친구로 모든 취약 지점을 점검할 수 있는 경우를 찾아 반환합니다.

### 동작 과정
1. **입력 데이터 처리**: 
   - 외벽의 길이(n)와 취약 지점의 위치(weak), 각각의 친구가 이동할 수 있는 거리(dist)를 입력받습니다.
   - 원형 구조이므로 취약 지점의 위치를 원형으로 고려하기 위해, `weak` 리스트에 원형 구조를 반영하여 `weak` 리스트를 두 배로 확장합니다. (즉, `weak`의 각 요소에 `n`을 더한 값을 추가)

2. **친구 배치와 점검 시뮬레이션**:
   - 각 취약 지점(start)에서 시작하여 모든 친구의 경우의 수를 고려합니다. 친구들은 이동 거리(dist)의 순열로 배치됩니다.
   - 각 친구가 점검할 수 있는 범위를 계산하여, 해당 친구가 점검한 후 다음 취약 지점까지 도달할 수 있는지 확인합니다. 
   - 점검이 필요한 취약 지점이 있다면 카운트(`cnt`)를 증가시키고, 추가적인 친구가 필요한 경우를 처리합니다.

3. **최소 친구 수 계산**:
   - 모든 경우의 수를 탐색하여 점검할 수 있는 취약 지점의 수에 따른 최소 친구 수를 업데이트합니다.
   - 만약 모든 취약 지점을 점검할 수 있는 경우가 없다면 `-1`을 반환하고, 최소 친구 수를 반환합니다.

예시 입력을 통한 단계별 설명:
- 예를 들어, `n = 12`, `weak = [1, 5, 6, 10]`, `dist = [1, 2, 3, 4]`인 경우, 
  - 취약 지점은 [1, 5, 6, 10, 13, 17, 18, 22]로 확장됩니다.
  - 각 취약 지점에서 시작하여 친구들을 배치하고 점검한 후, 모든 취약 지점을 커버할 수 있는 최소 친구 수를 찾습니다. 이 경우 최종적으로 2명의 친구가 필요하다는 결과를 도출합니다.

### 시간 복잡도
이 알고리즘의 시간 복잡도는 O(W! * W), 여기서 W는 취약 지점의 수입니다. 모든 친구의 순열을 탐색하기 때문에 W!의 복잡도를 가지며, 각 순열에 대해 최대 W개의 취약 지점을 확인하므로 최종 복잡도는 O(W! * W)입니다.

## 코드
```python
from itertools import permutations 
def solution(n, weak, dist):
    len_weak = len(weak)
    for i in range(len_weak):
        weak.append(weak[i] + n)
        
    ans = len(dist) + 1
    
    for start in range(len_weak):
        for friend in list(permutations(dist, len(dist))):
            cnt = 1
            position = weak[start] + friend[cnt-1]
            
            for idx in range(start, start + len_weak):
                if position < weak[idx]:
                    cnt += 1
                    
                    if cnt > len(dist):
                        break
                        
                    position = weak[idx] + friend[cnt-1]
                    
            ans = min(ans, cnt)
        
    if ans > len(dist):
        return -1
    
    return ans
```
