## 문제
오늘도 서준이는 점근적 표기 수업 조교를 하고 있다. 아빠가 수업한 내용을 학생들이 잘 이해했는지 문제를 통해서 확인해보자.

알고리즘의 소요 시간을 나타내는 O-표기법(빅-오)을 다음과 같이 정의하자.

O(g(n)) = {f(n) | 모든 n ≥ n0에 대하여 f(n) ≤ c × g(n)인 양의 상수 c와 n0가 존재한다}

이 정의는 실제 O-표기법(https://en.wikipedia.org/wiki/Big_O_notation)과 다를 수 있다.

함수 f(n) = a1n + a0, 양의 정수 c, n0가 주어질 경우 O(n) 정의를 만족하는지 알아보자.

---

## 입력
첫째 줄에 함수 f(n)을 나타내는 정수 a1, a0가 주어진다. (0 ≤ |ai| ≤ 100)

다음 줄에 양의 정수 c가 주어진다. (1 ≤ c ≤ 100)

다음 줄에 양의 정수 n0가 주어진다. (1 ≤ n0 ≤ 100)

---

## 출력
f(n), c, n0가 O(n) 정의를 만족하면 1, 아니면 0을 출력한다.

---

## 성능 요약

## 풀이

### 풀이 핵심 로직
주어진 함수 \( f(n) = a1 \cdot n + a0 \)가 O(n) 정의를 만족하기 위해서는, 모든 \( n \geq n0 \)에 대해 \( f(n) \leq c \cdot n \)이 성립해야 합니다. 이를 확인하기 위해 \( f(n) \)과 \( g(n) = c \cdot n \)을 비교하고, \( a1 \)이 \( c \)보다 작거나 같은지를 확인합니다.

### 동작 과정
1. **입력 받기**: 첫 줄에서 \( a1 \)과 \( a0 \)를 입력받고, 다음 줄에서 양의 정수 \( c \)와 \( n0 \)를 입력받습니다.
2. **함수 계산**: \( f(n) \)을 계산합니다. \( f(n) = a1 \cdot n + a0 \)에서 \( n \)은 \( n0 \)로 설정합니다.
3. **비교**:
   - \( f(n0) \)과 \( g(n0) = c \cdot n0 \)을 비교하여 \( f(n0) \leq g(n0) \)인지 확인합니다.
   - 또한 \( a1 \)이 \( c \)보다 작거나 같아야 합니다.
4. **결과 출력**: 두 조건이 모두 만족하면 1을 출력하고, 그렇지 않으면 0을 출력합니다.

예시 입력:
```
2 3
5
1
```
- \( a1 = 2, a0 = 3, c = 5, n0 = 1 \)일 때,
- \( f(1) = 2 \cdot 1 + 3 = 5 \)
- \( g(1) = 5 \cdot 1 = 5 \)
- \( 5 \leq 5 \)이고 \( 2 \leq 5 \)이므로 출력은 1입니다.

### 시간 복잡도
이 알고리즘은 입력을 받고, 간단한 산술 연산 및 비교를 수행하므로 시간 복잡도는 \( O(1) \)입니다.

## 코드
```python
a1, a0 = map(int, input().split())
c = int(input())
n = int(input())

f = a1 * n + a0
g = c * n

if f <= g and  a1 <= c:
    print(1)

else:
    print(0)
```
