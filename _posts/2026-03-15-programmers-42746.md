## 문제
0 또는 양의 정수가 주어졌을 때, 정수를 이어 붙여 만들 수 있는 가장 큰 수를 알아내 주세요.

예를 들어, 주어진 정수가 [6, 10, 2]라면 [6102, 6210, 1062, 1026, 2610, 2106]를 만들 수 있고, 이중 가장 큰 수는 6210입니다.

0 또는 양의 정수가 담긴 배열 numbers가 매개변수로 주어질 때, 순서를 재배치하여 만들 수 있는 가장 큰 수를 문자열로 바꾸어 return 하도록 solution 함수를 작성해주세요.

제한 사항


numbers의 길이는 1 이상 100,000 이하입니다.
numbers의 원소는 0 이상 1,000 이하입니다.
정답이 너무 클 수 있으니 문자열로 바꾸어 return 합니다.


입출력 예

        
numbers
return


        
[6, 10, 2]
"6210"


[3, 30, 34, 5, 9]
"9534330"

## 풀이
### 풀이 핵심 로직
주어진 정수를 이어 붙여 만들 수 있는 가장 큰 수를 찾기 위해, 각 숫자를 문자열로 변환한 후, 조합 가능한 순서를 정렬합니다. 정렬 기준은 숫자를 문자열로 반복하여 비교하는 방식으로 설정하여, 두 숫자 a와 b의 조합(a+b와 b+a)을 비교하며 큰 값을 우선적으로 배치합니다. 최종적으로 정렬된 숫자들을 이어 붙여 결과를 생성합니다.

### 동작 과정
1. **입력 변환**: 입력된 정수 리스트 `numbers`를 문자열 리스트로 변환합니다.
   - 예시: `[6, 10, 2]` → `["6", "10", "2"]`
   
2. **정렬**: 문자열 리스트를 정렬합니다. 각 문자열을 3번 반복하여 비교를 수행합니다. 이때, 반복 횟수는 각 요소의 길이에 따라 조정할 수 있지만, 최대 3자리 수이므로 3회 반복하는 것은 안정적인 방법입니다.
   - 예시: `["6", "10", "2"]`에서 `"6"*3`은 `"666"`이 되고, `"10"*3`은 `"101010"`가 되므로 `"666"`이 더 크기 때문에 `"6"`이 앞에 오게 됩니다.
   - 정렬 후: `["6", "2", "10"]`
   
3. **결과 조합**: 정렬된 리스트를 문자열로 연결하여 최종 결과를 생성합니다.
   - 예시: `''.join(["6", "2", "10"])` → `"6210"`
   
4. **예외 처리**: 만약 결과의 첫 번째 문자가 `'0'`일 경우 (모든 수가 0일 경우), 결과는 `'0'`로 반환합니다.
   - 예시: 결과가 `['0', '0', '0']`인 경우 → `'0'`

### 시간 복잡도
이 알고리즘의 시간 복잡도는 O(N log N)입니다. 여기서 N은 numbers 리스트의 길이입니다. 정렬 과정이 가장 큰 비중을 차지하며, 각 숫자를 문자열로 변환하는 과정은 O(N)입니다. 따라서 전체 시간 복잡도는 O(N log N)으로 평가할 수 있습니다.

## 코드
```python
def solution(numbers):
    numbers = list(map(str, numbers))
    
    numbers.sort(key=lambda x:x*3, reverse=True)
    
    answer = ''.join(numbers)
    
    return '0' if answer[0] == '0' else answer
```
