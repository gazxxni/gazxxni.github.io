## 문제
수많은 마라톤 선수들이 마라톤에 참여하였습니다. 단 한 명의 선수를 제외하고는 모든 선수가 마라톤을 완주하였습니다.

마라톤에 참여한 선수들의 이름이 담긴 배열 participant와 완주한 선수들의 이름이 담긴 배열 completion이 주어질 때, 완주하지 못한 선수의 이름을 return 하도록 solution 함수를 작성해주세요.

제한사항


마라톤 경기에 참여한 선수의 수는 1명 이상 100,000명 이하입니다.
completion의 길이는 participant의 길이보다 1 작습니다.
참가자의 이름은 1개 이상 20개 이하의 알파벳 소문자로 이루어져 있습니다.
참가자 중에는 동명이인이 있을 수 있습니다.


입출력 예

        
participant
completion
return


        
["leo", "kiki", "eden"]
["eden", "kiki"]
"leo"


["marina", "josipa", "nikola", "vinko", "filipa"]
["josipa", "filipa", "marina", "nikola"]
"vinko"


["mislav", "stanko", "mislav", "ana"]
["stanko", "ana", "mislav"]
"mislav"


      
입출력 예 설명

예제 #1
"leo"는 참여자 명단에는 있지만, 완주자 명단에는 없기 때문에 완주하지 못했습니다.

예제 #2
"vinko"는 참여자 명단에는 있지만, 완주자 명단에는 없기 때문에 완주하지 못했습니다.

예제 #3
"mislav"는 참여자 명단에는 두 명이 있지만, 완주자 명단에는 한 명밖에 없기 때문에 한명은 완주하지 못했습니다.

## 풀이
### 풀이 핵심 로직
문제의 핵심 아이디어는 참가자 목록과 완주자 목록을 정렬한 후, 두 목록을 비교하여 가장 먼저 차이가 나는 참가자를 찾는 것입니다. 만약 모든 완주자가 일치하면, 마지막 참가자가 완주하지 못한 선수로 판별됩니다.

### 동작 과정
1. 참가자 목록(`participant`)과 완주자 목록(`completion`)을 각각 정렬합니다.
2. 정렬된 두 목록을 인덱스별로 비교합니다.
   - 만약 두 목록의 i번째 요소가 다르면, `participant[i]`가 완주하지 못한 선수입니다.
3. 모든 요소가 일치하는 경우, 마지막 참가자(`participant[-1]`)가 완주하지 못한 선수입니다.

예시 입력:
- `participant = ["leo", "kiki", "eden"]`
- `completion = ["eden", "kiki"]`

단계별 설명:
1. 정렬된 `participant`: `["eden", "kiki", "leo"]`
2. 정렬된 `completion`: `["eden", "kiki"]`
3. 비교:
   - i=0: `participant[0]`와 `completion[0]`가 같음 (둘 다 "eden")
   - i=1: `participant[1]`와 `completion[1]`가 같음 (둘 다 "kiki")
   - i=2: `participant[2]` ("leo")는 비교할 `completion`이 없으므로, "leo"가 완주하지 못한 선수임을 리턴.

### 시간 복잡도
이 알고리즘의 시간 복잡도는 O(n log n)입니다. 이는 정렬 과정에 의해 결정됩니다. 이후의 비교는 O(n) 시간이 소요됩니다.

## 코드
```python
def solution(participant, completion):
    answer = ''
    participant.sort()
    completion.sort()
    
    for i in range(len(completion)):
        if participant[i] != completion[i]:
            return participant[i]
            
        
    return participant[-1]
```
