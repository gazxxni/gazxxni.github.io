## 문제
n개의 섬 사이에 다리를 건설하는 비용(costs)이 주어질 때, 최소의 비용으로 모든 섬이 서로 통행 가능하도록 만들 때 필요한 최소 비용을 return 하도록 solution을 완성하세요.

다리를 여러 번 건너더라도, 도달할 수만 있으면 통행 가능하다고 봅니다. 예를 들어 A 섬과 B 섬 사이에 다리가 있고, B 섬과 C 섬 사이에 다리가 있으면 A 섬과 C 섬은 서로 통행 가능합니다.

제한사항


섬의 개수 n은 1 이상 100 이하입니다.
costs의 길이는 ((n-1) * n) / 2이하입니다.
임의의 i에 대해, costs[i][0] 와 costs[i] [1]에는 다리가 연결되는 두 섬의 번호가 들어있고, costs[i] [2]에는 이 두 섬을 연결하는 다리를 건설할 때 드는 비용입니다.
같은 연결은 두 번 주어지지 않습니다. 또한 순서가 바뀌더라도 같은 연결로 봅니다. 즉 0과 1 사이를 연결하는 비용이 주어졌을 때, 1과 0의 비용이 주어지지 않습니다.
모든 섬 사이의 다리 건설 비용이 주어지지 않습니다. 이 경우, 두 섬 사이의 건설이 불가능한 것으로 봅니다.
연결할 수 없는 섬은 주어지지 않습니다.


입출력 예

        
n
costs
return


        
4
[[0,1,1],[0,2,2],[1,2,5],[1,3,1],[2,3,8]]
4


      
입출력 예 설명

costs를 그림으로 표현하면 다음과 같으며, 이때 초록색 경로로 연결하는 것이 가장 적은 비용으로 모두를 통행할 수 있도록 만드는 방법입니다.

## 풀이
### 풀이 핵심 로직
이 문제는 최소 신장 트리(MST) 문제로, 크루스칼 알고리즘을 사용하여 해결합니다. 주어진 섬과 다리의 비용을 기반으로 가장 낮은 비용의 다리들을 선택하여 모든 섬을 연결하는 것이 목표입니다. 이를 위해 다리의 비용을 오름차순으로 정렬한 후, 서로 연결되지 않은 섬들을 union-find 구조를 통해 연결합니다.

### 동작 과정
1. **입력 예시**: `n = 4`, `costs = [[0,1,1],[0,2,2],[1,2,5],[1,3,1],[2,3,8]]`
2. **초기화**: parent 배열을 `[0, 1, 2, 3]`로 초기화합니다. 각 섬은 자기 자신을 부모로 가집니다.
3. **비용 정렬**: costs를 비용을 기준으로 오름차순 정렬합니다. 정렬된 costs는 `[[0, 1, 1], [1, 3, 1], [0, 2, 2], [1, 2, 5], [2, 3, 8]]`입니다.
4. **다리 연결 과정**:
   - `(0, 1, 1)`을 선택하여 섬 0과 1을 연결합니다. 총 비용 = 1.
   - `(1, 3, 1)`을 선택하여 섬 1과 3을 연결합니다. 총 비용 = 2.
   - `(0, 2, 2)`을 선택하여 섬 0과 2를 연결합니다. 총 비용 = 4.
   - `(1, 2, 5)`와 `(2, 3, 8)`은 이미 연결된 섬들이므로 건너뜁니다.
5. **결과**: 모든 섬이 연결되었으므로, 최소 비용은 4로 반환됩니다.

### 시간 복잡도
- 정렬 과정: O(E log E), 여기서 E는 costs의 길이입니다.
- union-find 연산: 각 union 및 find 연산은 거의 O(1)입니다(경로 압축을 통해). 따라서 전체적으로 O(E log E)입니다. 
- 최종 시간 복잡도는 O(E log E)입니다.

## 코드
```python
def find_parent(parent, x):
    if parent[x] != x:
        parent[x] = find_parent(parent, parent[x])
        
    return parent[x]

def union(parent, a, b):
    a = find_parent(parent, a)
    b = find_parent(parent, b)
    
    if a < b:
        parent[b] = a
    else:
        parent[a] = b

def solution(n, costs):
    total = 0
    parent = [i for i in range(n)]
    costs.sort(key=lambda x:x[2])
    
    for a, b, cost in costs:
        if find_parent(parent, a) != find_parent(parent, b):
            union(parent, a, b)
            total += cost

        
    return total
```
