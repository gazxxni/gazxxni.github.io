밤늦게 귀가할 때 안전을 위해 항상 택시를 이용하던 무지는 최근 야근이 잦아져 택시를 더 많이 이용하게 되어 택시비를 아낄 수 있는 방법을 고민하고 있습니다. "무지"는 자신이 택시를 이용할 때 동료인 어피치 역시 자신과 비슷한 방향으로 가는 택시를 종종 이용하는 것을 알게 되었습니다. "무지"는 "어피치"와 귀가 방향이 비슷하여 택시 합승을 적절히 이용하면 택시요금을 얼마나 아낄 수 있을 지 계산해 보고 "어피치"에게 합승을 제안해 보려고 합니다.



위 예시 그림은 택시가 이동 가능한 반경에 있는 6개 지점 사이의 이동 가능한 택시노선과 예상요금을 보여주고 있습니다.
그림에서 A와 B 두 사람은 출발지점인 4번 지점에서 출발해서 택시를 타고 귀가하려고 합니다. A의 집은 6번 지점에 있으며 B의 집은 2번 지점에 있고 두 사람이 모두 귀가하는 데 소요되는 예상 최저 택시요금이 얼마인 지 계산하려고 합니다.


그림의 원은 지점을 나타내며 원 안의 숫자는 지점 번호를 나타냅니다.


지점이 n개일 때, 지점 번호는 1부터 n까지 사용됩니다.

지점 간에 택시가 이동할 수 있는 경로를 간선이라 하며, 간선에 표시된 숫자는 두 지점 사이의 예상 택시요금을 나타냅니다.


간선은 편의 상 직선으로 표시되어 있습니다.
위 그림 예시에서, 4번 지점에서 1번 지점으로(4→1) 가거나, 1번 지점에서 4번 지점으로(1→4) 갈 때 예상 택시요금은 10원으로 동일하며 이동 방향에 따라 달라지지 않습니다.

예상되는 최저 택시요금은 다음과 같이 계산됩니다.


4→1→5 : A, B가 합승하여 택시를 이용합니다. 예상 택시요금은 10 + 24 = 34원 입니다.
5→6 : A가 혼자 택시를 이용합니다. 예상 택시요금은 2원 입니다.
5→3→2 : B가 혼자 택시를 이용합니다. 예상 택시요금은 24 + 22 = 46원 입니다.
A, B 모두 귀가 완료까지 예상되는 최저 택시요금은 34 + 2 + 46 = 82원 입니다.

## 풀이
### 풀이 핵심 로직
이 문제는 주어진 도시의 지점들 간의 최단 경로를 계산하여, 택시 합승을 통해 요금을 최소화하는 방법을 찾는 것입니다. 이를 위해 다익스트라 알고리즘을 사용하여 각 지점에서 출발하는 최단 거리 배열을 계산하고, A와 B가 합승할 수 있는 지점을 찾아 그 경로를 통해 요금을 계산합니다.

### 동작 과정
1. **그래프 초기화**: 주어진 `fares`를 이용하여 각 지점 간의 요금을 저장하는 그래프를 초기화합니다. 이때 사전(`dict`)을 사용하여 인접 리스트 형식으로 저장합니다.
   
2. **다익스트라 알고리즘 구현**: `dijk` 함수를 통해 특정 출발 지점에서 다른 모든 지점까지의 최단 거리를 구하는 함수를 구현합니다. 이 함수는 우선순위 큐(`heapq`)를 사용하여 최단 경로를 효율적으로 계산합니다.

3. **최단 거리 계산**:
   - `dist_s`: 출발지점 s에서 모든 지점까지의 최소 요금
   - `dist_a`: A의 도착지점 a에서 모든 지점까지의 최소 요금
   - `dist_b`: B의 도착지점 b에서 모든 지점까지의 최소 요금

4. **합승 요금 계산**: 각 지점 i에 대해 A와 B가 그 지점에서 합승한다고 가정하고, s에서 i로 가는 요금과 A와 B가 각각 i에서 그들의 목적지로 가는 요금을 합산하여 최소 요금을 찾습니다.

5. **결과 반환**: 합승을 하지 않는 경우와 합승을 하는 경우 중 더 저렴한 요금을 선택하여 반환합니다.

#### 예시 입력
> n = 6, s = 4, a = 6, b = 2, fares = [[4, 1, 10], [3, 5, 24], [5, 6, 2], [3, 1, 41], [5, 1, 24], [4, 6, 50], [2, 4, 66], [2, 3, 22], [1, 6, 25]]

1. **그래프 생성**: 
   - 1과 4: 10
   - 3과 5: 24
   - 5와 6: 2
   - ...
   
2. **다익스트라 실행**:
   - `dist_s` = [inf, 10, 66, inf, 0, 34, 50]
   - `dist_a` = [inf, 25, 0, inf, 66, 2, 0]
   - `dist_b` = [inf, 25, 0, inf, 66, 2, 0]

3. **합승 요금 계산**:
   - i = 1: `dist_s[1] + dist_a[1] + dist_b[1] = 10 + 25 + 25 = 60`
   - i = 2: `dist_s[2] + dist_a[2] + dist_b[2] = 66 + 0 + 0 = 66`
   - i = 3: `dist_s[3] + dist_a[3] + dist_b[3] = inf + inf + inf`
   - i = 4: `dist_s[4] + dist_a[4] + dist_b[4] = 0 + 66 + 66 = 132`
   - i = 5: `dist_s[5] + dist_a[5] + dist_b[5] = 34 + 2 + 24 = 60`
   - i = 6: `dist_s[6] + dist_a[6] + dist_b[6] = 50 + 0 + 0 = 50`
   
4. **최소값 선택**: 82가 합승을 통한 최소 요금으로 결정되고 반환됩니다.

### 시간 복잡도
다익스트라 알고리즘은 O((V + E) log V)입니다. 여기서 V는 정점(지점)의 수, E는 간선(택시 요금 정보)의 수입니다. 따라서 이 문제의 시간 복잡도는 O((n + m) log n)입니다. n은 최대 200이므로, 200개의 정점과 최악의 경우 15개의 간선이 주어질 수 있습니다.

## 코드
```python
import heapq

def solution(n, s, a, b, fares):
    answer = 0
    
    graph = {i:{} for i in range(1, n+1)}
    for u, v, w in fares:
        graph[u][v] = w
        graph[v][u] = w
        
    def dijk(graph, start):
        distance = {i:float('inf') for i in range(1, n+1)}
        distance[start] = 0
        
        q = [(0, start)]
        while q:
            cur_dist, cur_node = heapq.heappop(q)
            
            if distance[cur_node] < cur_dist:
                continue
                
            for adj, weight in graph[cur_node].items():
                dist = cur_dist + weight
                
                if dist < distance[adj]:
                    distance[adj] = dist
                    heapq.heappush(q, (dist, adj))
        
        return distance
    
    dist_s = dijk(graph, s)
    dist_a = dijk(graph, a)
    dist_b = dijk(graph, b)
    
    min_fare = float('inf')
    
    for i in range(1, n+1):
        cur_fare = dist_s[i] + dist_a[i] + dist_b[i]
        min_fare = cur_fare if cur_fare < min_fare else min_fare
        
    return min_fare
```
