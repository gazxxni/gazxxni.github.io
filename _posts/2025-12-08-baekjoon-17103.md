## 문제
골드바흐의 추측: 2보다 큰 짝수는 두 소수의 합으로 나타낼 수 있다.


짝수 N을 두 소수의 합으로 나타내는 표현을 골드바흐 파티션이라고 한다. 짝수 N이 주어졌을 때, 골드바흐 파티션의 개수를 구해보자. 두 소수의 순서만 다른 것은 같은 파티션이다.

---

## 입력
첫째 줄에 테스트 케이스의 개수 T (1 ≤ T ≤ 100)가 주어진다. 각 테스트 케이스는 한 줄로 이루어져 있고, 정수 N은 짝수이고, 2

---

## 출력
각각의 테스트 케이스마다 골드바흐 파티션의 수를 출력한다.

## 풀이

### 풀이 핵심 로직
골드바흐의 추측에 따르면, 짝수 N은 두 소수의 합으로 표현될 수 있습니다. 이 문제에서는 주어진 짝수 N을 두 소수의 합으로 나타내는 경우의 수를 세기 위해, 먼저 에라토스테네스의 체를 사용하여 1,000,000 이하의 모든 소수를 미리 구한 후, 각 짝수 N에 대해 소수 목록을 이용해 가능한 소수 쌍을 찾아 카운트합니다.

### 동작 과정
1. **소수 목록 생성**: `primenum` 함수를 통해 1,000,000 이하의 소수를 구합니다. 이때, 에라토스테네스의 체를 사용하여 효율적으로 소수를 탐색합니다.
2. **입력 처리**: 테스트 케이스의 수 T를 입력받고, 각 짝수 N을 차례대로 입력받습니다.
3. **골드바흐 파티션 계산**:
   - 각 N에 대해, 소수 목록에서 N의 절반 이하의 소수를 순회합니다.
   - 현재 소수 i에 대해 N - i가 소수 목록에 존재하는지 확인합니다.
   - 존재하면 카운트를 증가시킵니다.
4. **결과 출력**: 각 짝수 N에 대한 골드바흐 파티션의 수를 출력합니다.

예시 입력:
```
3
8
10
12
```

단계별 설명:
- 첫 번째 테스트 케이스 N = 8:
  - 소수 목록: [2, 3, 5, 7]
  - 소수 쌍: (3, 5)
  - 카운트: 1
- 두 번째 테스트 케이스 N = 10:
  - 소수 쌍: (3, 7), (5, 5)
  - 카운트: 2
- 세 번째 테스트 케이스 N = 12:
  - 소수 쌍: (5, 7)
  - 카운트: 1

결과는:
```
1
2
1
```

### 시간 복잡도
소수 목록을 생성하는 데 O(n log log n)이 걸리며, 각 N에 대해 최대 O(n/2) 소수를 확인하므로 전체 시간 복잡도는 O(T * n)입니다. 여기서 T는 테스트 케이스의 수, n은 주어진 최대 짝수입니다.

## 코드
```python
def primenum(n):
    arr = [True] * (n + 1)
    arr[0] = arr[1] = False
    
    for i in range(2, int(n ** 0.5) + 1):
        if arr[i]:
            for j in range(i * i, n + 1, i):
                arr[j] = False
                
    brr = []
    for i in range(2, n + 1):
        if arr[i]:
            brr.append(i)
            
    return brr

a = primenum(1000000)
b = set(a)
t = int(input())

for _ in range(t):
    n = int(input())

    cnt = 0

    for i in a:
        if i > n // 2:
            break
        
        if (n - i) in b:
            cnt += 1
            
    print(cnt)
```
