## 풀이

### 풀이 핵심 로직
이 문제는 덱(deque)을 사용하여 다양한 명령을 수행하는 방식으로 해결된다. 명령에 따라 덱의 양쪽 끝에서 요소를 추가하거나 제거하고, 현재 덱의 상태에 대한 정보를 출력하는 방식으로 구현된다.

### 동작 과정
1. 먼저, 입력으로 주어지는 명령의 수 `n`을 읽어온다.
2. `deque`를 초기화하고, `n`번의 반복을 통해 각 명령을 처리한다.
   - 명령이 `1`일 경우, 주어진 값을 덱의 앞에 추가한다.
   - 명령이 `2`일 경우, 주어진 값을 덱의 뒤에 추가한다.
   - 명령이 `3`일 경우, 덱의 앞에서 요소를 제거하고 출력한다. 덱이 비어있으면 `-1` 출력.
   - 명령이 `4`일 경우, 덱의 뒤에서 요소를 제거하고 출력한다. 덱이 비어있으면 `-1` 출력.
   - 명령이 `5`일 경우, 덱의 크기를 출력한다.
   - 명령이 `6`일 경우, 덱이 비어있으면 `1`, 그렇지 않으면 `0`을 출력한다.
   - 명령이 `7`일 경우, 덱의 앞 요소를 출력하고 비어있으면 `-1`을 출력한다.
   - 명령이 `8`일 경우, 덱의 뒤 요소를 출력하고 비어있으면 `-1`을 출력한다.

예시 입력:
```
8
1 10
2 20
3
4
5
6
7
8
```
단계별 설명:
- `1 10`: 덱에 `10`을 앞에 추가 → 덱: `[10]`
- `2 20`: 덱에 `20`을 뒤에 추가 → 덱: `[10, 20]`
- `3`: 덱의 앞에서 `10`을 제거하고 출력 → 출력: `10`, 덱: `[20]`
- `4`: 덱의 뒤에서 `20`을 제거하고 출력 → 출력: `20`, 덱: `[]`
- `5`: 덱의 크기 출력 → 출력: `0`
- `6`: 덱이 비어있으므로 `1` 출력 → 출력: `1`
- `7`: 덱이 비어 있으므로 `-1` 출력 → 출력: `-1`
- `8`: 덱이 비어 있으므로 `-1` 출력 → 출력: `-1`

### 시간 복잡도
모든 명령은 `O(1)`의 시간 복잡도를 가지므로 전체 시간 복잡도는 `O(n)`이다.

## 코드
```python
import sys
input = sys.stdin.readline
from collections import deque

n = int(input())
q = deque()

for _ in range(n):
    command = input().split()
    
    a = int(command[0])
    
    if a == 1:
        q.appendleft(command[1])
        
    elif a == 2:
        q.append(command[1])
        
    elif a == 3:
        if q:
            print(q.popleft())
        else:
            print(-1)
            
    elif a == 4:
        if q:
            print(q.pop())
        else:
            print(-1)
            
    elif a == 5:
        print(len(q))
        
    elif a == 6:
        if q:
            print(0)
        else:
            print(1)
            
    elif a == 7:
        if q:
            print(q[0])
        else:
            print(-1)
            
    elif a == 8:
        if q:
            print(q[-1])
        else:
            print(-1)
```
