## 문제
재귀 호출만 생각하면 신이 난다! 아닌가요?

다음과 같은 재귀함수 w(a, b, c)가 있다.

if a  20 or b > 20 or c > 20, then w(a, b, c) returns:
    w(20, 20, 20)

if a 

위의 함수를 구현하는 것은 매우 쉽다. 하지만, 그대로 구현하면 값을 구하는데 매우 오랜 시간이 걸린다. (예를 들면, a=15, b=15, c=15)

a, b, c가 주어졌을 때, w(a, b, c)를 출력하는 프로그램을 작성하시오.

---

## 입력
입력은 세 정수 a, b, c로 이루어져 있으며, 한 줄에 하나씩 주어진다. 입력의 마지막은 -1 -1 -1로 나타내며, 세 정수가 모두 -1인 경우는 입력의 마지막을 제외하면 없다.

---

## 출력
입력으로 주어진 각각의 a, b, c에 대해서, w(a, b, c)를 출력한다.

## 풀이

### 풀이 핵심 로직
주어진 문제는 재귀 함수 `w(a, b, c)`의 값을 구하는 것으로, 입력값 `a`, `b`, `c`가 특정 조건을 만족할 때 함수의 결과를 결정합니다. 이 함수는 동적 프로그래밍을 사용하여 이미 계산된 값을 저장함으로써 중복 계산을 피하고, 입력값이 특정 범위를 초과할 경우 `w(20, 20, 20)`을 반환합니다.

### 동작 과정
1. **입력 처리:** 세 정수 `a`, `b`, `c`를 입력받고, `-1 -1 -1`이 입력될 때까지 반복합니다.
2. **함수 호출:** `w(a, b, c)`를 호출하여 결과를 계산합니다.
3. **조건 검사:** 
   - 만약 `a`, `b`, `c` 중 하나라도 0 이하이면 1을 반환합니다.
   - 만약 `a`, `b`, `c` 중 하나라도 20을 초과하면 `w(20, 20, 20)`을 호출합니다.
4. **메모이제이션:** 이미 계산된 결과가 있다면 그 값을 반환합니다.
5. **재귀 호출:** 
   - `a < b < c`인 경우와 그렇지 않은 경우를 나누어 결과를 계산합니다.
6. **출력:** 각 입력에 대한 결과를 포맷에 맞추어 출력합니다.

예시 입력: 
```
1 2 3
2 2 2
10 20 30
-1 -1 -1
```

출력:
```
w(1, 2, 3) = 5
w(2, 2, 2) = 4
w(10, 20, 30) = 1048576
```

### 시간 복잡도
이 알고리즘의 시간 복잡도는 O(1)입니다. 이는 `a`, `b`, `c`의 범위가 0부터 20까지로 고정되어 있기 때문에, 모든 가능한 조합을 미리 계산하여 저장해두기 때문에 상수 시간에 값을 조회할 수 있습니다.

## 코드
```python
def w(a, b, c):
    if a <= 0 or b <= 0 or c <= 0:
        return 1
    
    if a > 20 or b > 20 or c > 20:
        return w(20, 20, 20)
    
    if dp[a][b][c]:
        return dp[a][b][c]
    
    if a < b < c:
        dp[a][b][c] = w(a, b, c - 1) + w(a, b - 1, c- 1) - w(a, b - 1, c)
        return dp[a][b][c]
    
    dp[a][b][c] = w(a - 1, b, c) + w(a - 1, b - 1, c) + w(a - 1, b, c - 1) - w(a - 1, b - 1, c - 1)
    return dp[a][b][c]

dp = [[[0] * 21 for _ in range(21)] for _ in range(21)]

while True:
    a, b, c = map(int, input().split())

    if a == -1 and b == -1 and c == -1:
        exit()
    
    print(f'w({a}, {b}, {c}) = {w(a, b, c)}')
```
