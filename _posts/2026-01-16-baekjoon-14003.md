## 문제
수열 A가 주어졌을 때, 가장 긴 증가하는 부분 수열을 구하는 프로그램을 작성하시오.

예를 들어, 수열 A = {10, 20, 10, 30, 20, 50} 인 경우에 가장 긴 증가하는 부분 수열은 A = {10, 20, 10, 30, 20, 50} 이고, 길이는 4이다.

---

## 입력
첫째 줄에 수열 A의 크기 N (1 ≤ N ≤ 1,000,000)이 주어진다.

둘째 줄에는 수열 A를 이루고 있는 Ai가 주어진다. (-1,000,000,000 ≤ Ai ≤ 1,000,000,000)

---

## 출력
첫째 줄에 수열 A의 가장 긴 증가하는 부분 수열의 길이를 출력한다.

둘째 줄에는 정답이 될 수 있는 가장 긴 증가하는 부분 수열을 출력한다.

## 풀이

### 풀이 핵심 로직
문제의 핵심은 가장 긴 증가하는 부분 수열(LIS)을 효율적으로 찾는 것입니다. 이를 위해 이분 탐색을 활용하여 현재 수열의 가장 긴 증가하는 부분 수열의 끝 값을 저장하는 `tails` 리스트를 유지합니다. 이를 통해 새로운 요소를 추가할 때마다 `tails` 리스트에서 적절한 위치를 찾아 업데이트하여 LIS의 길이를 계산하고, 역추적을 통해 실제 LIS를 구성합니다.

### 동작 과정
예시 입력으로 수열 A = {10, 20, 10, 30, 20, 50}를 고려해보겠습니다.

1. **초기화**: `tails`와 `tails_pos`는 비어있고, `prev_idx`는 -1로 초기화됩니다.
   
2. **첫 번째 요소 (10)**:
   - `search` 함수로 `tails`에서 10의 위치를 찾습니다(0).
   - `tails`에 10을 추가합니다: `tails = [10]`, `tails_pos = [0]`, `prev_idx`는 [ -1 ].

3. **두 번째 요소 (20)**:
   - `search`로 20의 위치를 찾습니다(1).
   - `tails`에 20을 추가합니다: `tails = [10, 20]`, `tails_pos = [0, 1]`, `prev_idx`는 [ -1, 0 ].

4. **세 번째 요소 (10)**:
   - `search`로 10의 위치를 찾습니다(0).
   - `tails`를 갱신하지 않고, `prev_idx`에 현재 인덱스를 기록합니다: `prev_idx`는 [ -1, 0, -1 ].

5. **네 번째 요소 (30)**:
   - `search`로 30의 위치를 찾습니다(2).
   - `tails`에 30을 추가합니다: `tails = [10, 20, 30]`, `tails_pos = [0, 1, 3]`, `prev_idx`는 [ -1, 0, -1, 1 ].

6. **다섯 번째 요소 (20)**:
   - `search`로 20의 위치를 찾습니다(1).
   - `tails`를 갱신하지 않고, `prev_idx`에 현재 인덱스를 기록합니다: `prev_idx`는 [ -1, 0, -1, 1, 1 ].

7. **여섯 번째 요소 (50)**:
   - `search`로 50의 위치를 찾습니다(3).
   - `tails`에 50을 추가합니다: `tails = [10, 20, 30, 50]`, `tails_pos = [0, 1, 3, 5]`, `prev_idx`는 [ -1, 0, -1, 1, 1, 3 ].

마지막으로 `tails`의 길이를 통해 LIS의 길이를 얻고, `prev_idx`를 사용하여 실제 LIS를 역추적하여 출력합니다. 최종 결과는 길이 4와 부분 수열 {10, 20, 30, 50}입니다.

### 시간 복잡도
이 알고리즘의 시간 복잡도는 O(N log N)입니다. 여기서 N은 수열의 크기입니다. 이분 탐색을 통해 `tails` 리스트를 업데이트하는 과정이 log N의 시간복잡도를 가지며, 전체 N개의 요소에 대해 반복하므로 O(N log N)이 됩니다.

## 코드
```python
import sys
input = sys.stdin.readline

n = int(input())
arr = list(map(int, input().split()))

# tails[k] = 길이 (k+1)인 증가 부분수열의 최소 끝값
# tails_pos[k] = 위 tails[k]를 만든 인덱스
tails = []
tails_pos = []

# prev_idx[i] = arr[i]가 LIS에 포함될 때 바로 이전 원소의 인덱스
prev_idx = [-1] * n

def search(a, x):
    lo, hi = 0, len(a)
    while lo < hi:
        mid = (lo + hi) // 2
        if a[mid] < x:
            lo = mid + 1
        else:
            hi = mid
    return lo

for i, x in enumerate(arr):
    j = search(tails, x)

    # j-1 길이의 꼬리에 이어 붙이므로, 경로의 마지막 인덱스를 prev로 기록
    prev_idx[i] = tails_pos[j - 1] if j > 0 else -1

    if j == len(tails):  # 길이 확장
        tails.append(x)
        tails_pos.append(i)
    else:    # 꼬리값 갱신
        tails[j] = x
        tails_pos[j] = i

# 실가장 긴 길이의 마지막 인덱스부터 prev_idx를 따라 역추적
lis_len = len(tails)
k = tails_pos[-1] if tails_pos else -1

seq = []
while k != -1:
    seq.append(arr[k])
    k = prev_idx[k]
seq.reverse()

print(lis_len)
print(*seq)
```
