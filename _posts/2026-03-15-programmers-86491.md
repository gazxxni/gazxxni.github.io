## 문제
명함 지갑을 만드는 회사에서 지갑의 크기를 정하려고 합니다. 다양한 모양과 크기의 명함들을 모두 수납할 수 있으면서, 작아서 들고 다니기 편한 지갑을 만들어야 합니다. 이러한 요건을 만족하는 지갑을 만들기 위해 디자인팀은 모든 명함의 가로 길이와 세로 길이를 조사했습니다.

가장 긴 가로 길이와 세로 길이가 각각 80, 70이기 때문에 80(가로) x 70(세로) 크기의 지갑을 만들면 모든 명함들을 수납할 수 있습니다. 하지만 2번 명함을 가로로 눕혀 수납한다면 80(가로) x 50(세로) 크기의 지갑으로 모든 명함들을 수납할 수 있습니다. 이때의 지갑 크기는 4000(=80 x 50)입니다.

모든 명함의 가로 길이와 세로 길이를 나타내는 2차원 배열 sizes가 매개변수로 주어집니다. 모든 명함을 수납할 수 있는 가장 작은 지갑을 만들 때, 지갑의 크기를 return 하도록 solution 함수를 완성해주세요.

## 풀이
### 풀이 핵심 로직
이 문제의 핵심 아이디어는 각 명함의 가로와 세로 길이를 비교하여, 모든 명함을 가로로 정렬하거나 세로로 정렬했을 때의 최대 길이를 계산하는 것입니다. 각 명함의 가로와 세로를 비교하여 더 큰 값을 가로로, 더 작은 값을 세로로 설정하고, 이들 중 최대값을 이용해 최종 지갑의 크기를 계산합니다.

### 동작 과정
1. **입력 예시**: `sizes = [[60, 50], [30, 70], [60, 30], [80, 40]]`
2. 각 명함의 가로와 세로 길이를 비교하여, 가로가 더 크면 가로, 세로를 그대로 유지하고, 세로가 더 크면 두 값을 스왑하여 항상 가로가 더 크도록 만듭니다.
   - 명함 1: (60, 50) → (60, 50)
   - 명함 2: (30, 70) → (70, 30)
   - 명함 3: (60, 30) → (60, 30)
   - 명함 4: (80, 40) → (80, 40)
3. 최종적으로 가로 길이의 최대값과 세로 길이의 최대값을 구합니다:
   - 최대 가로: max(60, 70, 60, 80) = 80
   - 최대 세로: max(50, 30, 30, 40) = 50
4. 지갑의 크기를 계산합니다: `80 * 50 = 4000`
5. 결과는 `4000`입니다.

### 시간 복잡도
이 알고리즘의 시간 복잡도는 O(n)입니다. 여기서 n은 sizes 배열의 길이입니다. 각 명함에 대해 한 번씩만 순회하면서 최대값을 찾기 때문에 선형 시간 복잡도를 가지게 됩니다.

## 코드
```python
def solution(sizes):
    w = []
    h = []
    
    for i in range(len(sizes)):
        if sizes[i][0] >= sizes[i][1]:
            w.append(sizes[i][0])
            h.append(sizes[i][1])
        else:
            w.append(sizes[i][1])
            h.append(sizes[i][0])
            
    return max(w) * max(h)
```
