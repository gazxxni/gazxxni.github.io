## 문제
총 N개의 문자열로 이루어진 집합 S가 주어진다.

입력으로 주어지는 M개의 문자열 중에서 집합 S에 포함되어 있는 것이 총 몇 개인지 구하는 프로그램을 작성하시오.

---

## 입력
첫째 줄에 문자열의 개수 N과 M (1 ≤ N ≤ 10,000, 1 ≤ M ≤ 10,000)이 주어진다. 

다음 N개의 줄에는 집합 S에 포함되어 있는 문자열들이 주어진다.

다음 M개의 줄에는 검사해야 하는 문자열들이 주어진다.

입력으로 주어지는 문자열은 알파벳 소문자로만 이루어져 있으며, 길이는 500을 넘지 않는다. 집합 S에 같은 문자열이 여러 번 주어지는 경우는 없다.

---

## 출력
첫째 줄에 M개의 문자열 중에 총 몇 개가 집합 S에 포함되어 있는지 출력한다.

## 풀이

### 풀이 핵심 로직
이 문제의 핵심 아이디어는 집합 S에 포함된 문자열들을 해시 세트(set)로 저장하여, 각 검사 문자열이 이 집합에 존재하는지를 효율적으로 확인하는 것입니다. 해시 세트를 사용하면 문자열의 포함 여부를 평균적으로 O(1)의 시간 복잡도로 확인할 수 있습니다.

### 동작 과정
1. 입력으로 문자열의 개수 N과 검사할 문자열의 개수 M을 받습니다.
2. N개의 문자열을 입력받아 집합 S에 저장합니다. 이때, 중복된 문자열은 없으므로 set 자료형을 사용합니다.
3. M개의 문자열을 입력받아 각 문자열이 집합 S에 존재하는지를 확인합니다. 존재하면 카운트를 증가시킵니다.
4. 최종적으로 카운트된 값을 출력합니다.

예시 입력:
```
5 5
apple
banana
orange
grape
melon
apple
grapefruit
banana
orange
kiwi
```
- 첫 번째 줄에서 5개의 문자열과 5개의 검사 문자열을 입력받습니다.
- 다음 5개의 줄에서 S에 포함된 문자열을 저장합니다: {'apple', 'banana', 'orange', 'grape', 'melon'}
- 검사 문자열을 하나씩 확인하며 'apple', 'banana', 'orange'는 집합 S에 포함되어 있으므로 카운트가 증가합니다. 'grapefruit'와 'kiwi'는 포함되어 있지 않으므로 카운트가 증가하지 않습니다.
- 최종 카운트는 3이므로 출력됩니다.

### 시간 복잡도
이 알고리즘의 시간 복잡도는 O(N + M)입니다. N은 집합 S에 저장하는 문자열의 개수, M은 검사하는 문자열의 개수입니다. 각 문자열을 집합에 추가하는 과정과 검사하는 과정이 각각 O(1)의 평균 시간 복잡도를 가지므로, 이 두 과정을 합치면 총 O(N + M)이 됩니다.

## 코드
```python
n, m = map(int, input().split())

arr = set()
for _ in range(n):
    arr.add(input().rstrip())

cnt = 0
for _ in range(m):
    s = input().rstrip()
    if s in arr:
        cnt += 1

print(cnt)
```
