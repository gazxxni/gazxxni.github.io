## 문제
골드바흐의 추측: 2보다 큰 짝수는 두 소수의 합으로 나타낼 수 있다.


짝수 N을 두 소수의 합으로 나타내는 표현을 골드바흐 파티션이라고 한다. 짝수 N이 주어졌을 때, 골드바흐 파티션의 개수를 구해보자. 두 소수의 순서만 다른 것은 같은 파티션이다.

---

## 입력
첫째 줄에 테스트 케이스의 개수 T (1 ≤ T ≤ 100)가 주어진다. 각 테스트 케이스는 한 줄로 이루어져 있고, 정수 N은 짝수이고, 2

---

## 출력
각각의 테스트 케이스마다 골드바흐 파티션의 수를 출력한다.

## 풀이

### 풀이 핵심 로직
골드바흐 파티션의 개수를 구하기 위해, 먼저 주어진 최대값까지의 소수를 미리 구한 후, 각 테스트 케이스에서 주어진 짝수 N을 두 소수의 합으로 나타낼 수 있는 방법을 세는 방식이다. 각 소수 p에 대해 n - p가 소수인지 확인하여, 두 소수의 조합을 카운트한다.

### 동작 과정
1. `primenum` 함수에서 에라토스테네스의 체를 이용해 1,000,000까지의 소수를 구한다.
   - 소수를 구하기 위해 True 배열을 만들고, 각 숫자에 대해 배수를 False로 설정하여 소수를 판별한다.
   - 최종적으로 소수 리스트 `brr`를 반환한다.

2. `t`개의 테스트 케이스를 입력받는다.

3. 각 짝수 N에 대해, 다음 과정을 반복한다:
   - 소수 리스트 `a`를 순회하며, N의 절반보다 작은 소수 p를 선택한다.
   - N - p가 소수 리스트 `b`에 존재하는지 확인한다.
   - 존재하면 카운트를 증가시킨다.

4. 최종적으로 각 테스트 케이스에 대해 카운트된 골드바흐 파티션의 개수를 출력한다.

예시 입력:
```
3
8
10
12
```
- 첫 번째 케이스 N=8: (3, 5) → 1개의 파티션
- 두 번째 케이스 N=10: (3, 7), (5, 5) → 2개의 파티션
- 세 번째 케이스 N=12: (5, 7) → 1개의 파티션

출력:
```
1
2
1
```

### 시간 복잡도
소수를 구하는 데 O(n log log n)의 시간이 걸리고, 각 테스트 케이스에서 O(n/2)로 소수를 체크하기 때문에 전체 시간 복잡도는 O(T * n)으로, 여기서 T는 테스트 케이스의 수, n은 최대 짝수의 크기이다.

## 코드
```python
def primenum(n):
    arr = [True] * (n + 1)
    arr[0] = arr[1] = False
    
    for i in range(2, int(n ** 0.5) + 1):
        if arr[i]:
            for j in range(i * i, n + 1, i):
                arr[j] = False
                
    brr = []
    for i in range(2, n + 1):
        if arr[i]:
            brr.append(i)
            
    return brr

a = primenum(1000000)
b = set(a)
t = int(input())

for _ in range(t):
    n = int(input())

    cnt = 0

    for i in a:
        if i > n // 2:
            break
        
        if (n - i) in b:
            cnt += 1
            
    print(cnt)
```
