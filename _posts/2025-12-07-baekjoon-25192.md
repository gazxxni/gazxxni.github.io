## 문제
알고리즘 입문방 오픈 채팅방에서는 새로운 분들이 입장을 할 때마다 곰곰티콘을 사용해 인사를 한다. 이를 본 문자열 킬러 임스는 채팅방의 기록을 수집해 그 중 곰곰티콘이 사용된 횟수를 구해 보기로 했다.

ENTER는 새로운 사람이 채팅방에 입장했음을 나타낸다. 그 외는 채팅을 입력한 유저의 닉네임을 나타낸다. 닉네임은 숫자 또는 영문 대소문자로 구성되어 있다.

새로운 사람이 입장한 이후 처음 채팅을 입력하는 사람은 반드시 곰곰티콘으로 인사를 한다. 그 외의 기록은 곰곰티콘을 쓰지 않은 평범한 채팅 기록이다.

채팅 기록 중 곰곰티콘이 사용된 횟수를 구해보자!

---

## 입력
첫 번째 줄에는 채팅방의 기록 수를 나타내는 정수 N$N$ 이 주어진다. (1≤N≤100000$1 \le N \le 100\,000$)

두 번째 줄부터 N$N$ 개의 줄에 걸쳐 새로운 사람의 입장을 나타내는 ENTER, 혹은 채팅을 입력한 유저의 닉네임이 문자열로 주어진다. (문자열길이1≤문자열 길이≤20$1 \le \texttt{문자열 길이} \le 20$)

첫 번째 주어지는 문자열은 무조건 ENTER이다.

---

## 출력
채팅 기록 중 곰곰티콘이 사용된 횟수를 출력하시오.

## 풀이

### 풀이 핵심 로직
이 문제의 핵심 아이디어는 사용자가 ENTER로 들어올 때마다 현재 채팅방에 있는 사용자들의 수를 세고, 새로운 사용자가 채팅을 시작할 때마다 곰곰티콘을 사용한 것으로 계산하는 것입니다. 각 사용자가 채팅을 입력하고 나면, 그 사용자는 다시 카운트하지 않기 위해 이전 사용자 목록을 초기화합니다.

### 동작 과정
1. 첫 번째 입력으로 N이 주어집니다. N은 채팅 기록의 수입니다.
2. 이후 N개의 기록을 읽어들입니다.
3. ENTER가 입력되면, 현재까지 채팅한 사용자 수를 카운트에 더하고, 사용자 목록을 초기화합니다.
4. 사용자의 닉네임이 입력되면, 해당 사용자를 현재 목록에 추가합니다.
5. 마지막으로 모든 기록이 처리된 후, 남아있는 사용자 수를 카운트에 더하고 결과를 출력합니다.

예시 입력:
```
7
ENTER
user1
user2
ENTER
user3
user1
user4
```

- 첫 번째 ENTER: `arr`이 비어있으므로 카운트는 0.
- `user1` 추가: `arr`이 ['user1'].
- `user2` 추가: `arr`이 ['user1', 'user2'].
- 두 번째 ENTER: 카운트에 2(현재 사용자 수) 추가. `cnt = 2`, `arr` 초기화.
- `user3` 추가: `arr`이 ['user3'].
- `user1` 추가: `arr`이 ['user3', 'user1'].
- `user4` 추가: `arr`이 ['user3', 'user1', 'user4'].
- 마지막 카운트 추가: 현재 사용자 수 3을 카운트에 더함. `cnt = 2 + 3 = 5`.

최종 출력은 5입니다.

### 시간 복잡도
이 알고리즘의 시간 복잡도는 O(N)입니다. 각 입력을 한 번씩만 처리하고, 집합에 사용자를 추가하는 연산은 평균적으로 O(1)입니다. 따라서 전체적으로 N개의 입력에 대해 O(N)의 시간 복잡도를 가집니다.

## 코드
```python
n = int(input())

cnt = 0
arr = set()
for _ in range(n):
    a = input().rstrip()
    if a == 'ENTER':
        cnt += len(arr)
        arr = set()
        continue
    
    arr.add(a)

cnt += len(arr)
print(cnt)
```
