## 문제
베르트랑 공준은 임의의 자연수 n에 대하여, n보다 크고, 2n보다 작거나 같은 소수는 적어도 하나 존재한다는 내용을 담고 있다.

이 명제는 조제프 베르트랑이 1845년에 추측했고, 파프누티 체비쇼프가 1850년에 증명했다.

예를 들어, 10보다 크고, 20보다 작거나 같은 소수는 4개가 있다. (11, 13, 17, 19) 또, 14보다 크고, 28보다 작거나 같은 소수는 3개가 있다. (17,19, 23)

자연수 n이 주어졌을 때, n보다 크고, 2n보다 작거나 같은 소수의 개수를 구하는 프로그램을 작성하시오.

---

## 입력
입력은 여러 개의 테스트 케이스로 이루어져 있다. 각 케이스는 n을 포함하는 한 줄로 이루어져 있다.

입력의 마지막에는 0이 주어진다.

---

## 출력
각 테스트 케이스에 대해서, n보다 크고, 2n보다 작거나 같은 소수의 개수를 출력한다.

## 풀이

### 풀이 핵심 로직
문제의 핵심은 주어진 n에 대해 n보다 크고 2n보다 작거나 같은 소수를 찾는 것이다. 이를 위해 먼저 주어진 범위 내의 모든 소수를 미리 계산한 후, 각 테스트 케이스에서 해당 범위 내 소수의 개수를 세는 방식으로 해결한다. 에라토스테네스의 체를 사용하여 효율적으로 소수를 판별한다.

### 동작 과정
1. **소수 배열 생성**: `prime_numbers` 함수를 통해 0부터 300,000까지의 소수를 미리 구한다. 이 배열에서는 소수인 경우 `True`, 소수가 아닌 경우 `False`로 표시한다.
2. **입력 처리**: 사용자로부터 n을 입력받고, n이 0일 경우 입력을 종료한다.
3. **소수 개수 세기**: n보다 크고 2n보다 작거나 같은 범위에서 소수를 세기 위해 `count` 변수를 초기화하고, n+1부터 2n까지의 수를 검사하여 해당 배열에서 `True`인 경우에만 `count`를 증가시킨다.
4. **결과 출력**: 각 테스트 케이스에 대해 소수의 개수를 출력한다.

**예시 입력**: 
```
10
13
16
0
```
- 입력 `10`: n=10, 범위는 (11, 20) -> 소수: 4개 (11, 13, 17, 19)
- 입력 `13`: n=13, 범위는 (14, 26) -> 소수: 3개 (17, 19, 23)
- 입력 `16`: n=16, 범위는 (17, 32) -> 소수: 7개 (17, 19, 23, 29, 31)

**출력**:
```
4
3
7
```

### 시간 복잡도
이 알고리즘의 시간 복잡도는 O(N log log N)으로, 에라토스테네스의 체를 통해 소수를 찾는 과정에서 발생하는 복잡도이다. 이후 각 테스트 케이스에 대해 소수를 세는 과정은 O(N)이다. 전체적으로 여러 테스트 케이스를 처리할 때는 입력된 n에 따라 다를 수 있지만, 소수를 미리 계산한 배열을 사용하기 때문에 효율적이다.

## 코드
```python
def prime_numbers(n):
    arr = [True] * (n + 1) 
    arr[0] = arr[1] = False

    for i in range(2, int(n ** 0.5) + 1):
        if arr[i]:
            for j in range(i * i, n + 1, i):
                arr[j] = False
    
    return arr

a = prime_numbers(300000)

while True:
    n = int(input())
    
    if n == 0:
        exit()
        
    cnt = 0
    for i in range(n + 1, 2 * n + 1):
        if a[i] == True:
            cnt += 1
    
    print(cnt)
```
