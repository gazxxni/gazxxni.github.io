## 문제
n개의 정수로 이루어진 임의의 수열이 주어진다. 우리는 이 중 연속된 몇 개의 수를 선택해서 구할 수 있는 합 중 가장 큰 합을 구하려고 한다. 단, 수는 한 개 이상 선택해야 한다.

예를 들어서 10, -4, 3, 1, 5, 6, -35, 12, 21, -1 이라는 수열이 주어졌다고 하자. 여기서 정답은 12+21인 33이 정답이 된다.

---

## 입력
첫째 줄에 정수 n(1 ≤ n ≤ 100,000)이 주어지고 둘째 줄에는 n개의 정수로 이루어진 수열이 주어진다. 수는 -1,000보다 크거나 같고, 1,000보다 작거나 같은 정수이다.

---

## 출력
첫째 줄에 답을 출력한다.

## 풀이

### 풀이 핵심 로직
이 문제는 연속된 수들의 합 중에서 최대값을 찾아야 하므로, 동적 프로그래밍을 활용하여 해결할 수 있다. 현재 위치까지의 최대 연속합을 저장하는 배열을 사용하고, 각 원소에서 현재 원소를 포함한 최대합과 현재 원소만을 선택한 최대합 중 큰 값을 선택하여 갱신한다.

### 동작 과정
입력 예시: `n = 10`, 수열: `10, -4, 3, 1, 5, 6, -35, 12, 21, -1`

1. 초기화:
   - `dp` 배열을 `0`으로 초기화하고, 첫 번째 원소인 `10`으로 설정한다.
   - `ans` 변수도 첫 번째 원소로 초기화한다.

2. 반복문을 통해 각 원소에 대해 최대 연속합을 계산한다:
   - `i = 1`: `dp[1] = max(-4, 10 - 4) = 6` → `ans = max(10, 6) = 10`
   - `i = 2`: `dp[2] = max(3, 6 + 3) = 9` → `ans = max(10, 9) = 10`
   - `i = 3`: `dp[3] = max(1, 9 + 1) = 10` → `ans = max(10, 10) = 10`
   - `i = 4`: `dp[4] = max(5, 10 + 5) = 15` → `ans = max(10, 15) = 15`
   - `i = 5`: `dp[5] = max(6, 15 + 6) = 21` → `ans = max(15, 21) = 21`
   - `i = 6`: `dp[6] = max(-35, 21 - 35) = -14` → `ans = max(21, -14) = 21`
   - `i = 7`: `dp[7] = max(12, -14 + 12) = 12` → `ans = max(21, 12) = 21`
   - `i = 8`: `dp[8] = max(21, 12 + 21) = 33` → `ans = max(21, 33) = 33`
   - `i = 9`: `dp[9] = max(-1, 33 - 1) = 32` → `ans = max(33, 32) = 33`

최종적으로 `ans`는 `33`이 된다.

### 시간 복잡도
이 알고리즘의 시간 복잡도는 O(n)이다. 입력 수열의 길이에 비례하여 반복문을 한 번 수행하기 때문이다.

## 코드
```python
import sys
input = sys.stdin.readline

n = int(input())
arr = list(map(int, input().split()))

dp = [0] * (n + 1)
dp[0] = arr[0]
ans = arr[0]

for i in range(1, n):
    dp[i] = max(arr[i], dp[i - 1] + arr[i])
    ans = max(ans, dp[i])


print(ans)
```
