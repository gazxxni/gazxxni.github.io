## 문제
두 자연수 A와 B에 대해서, A의 배수이면서 B의 배수인 자연수를 A와 B의 공배수라고 한다. 이런 공배수 중에서 가장 작은 수를 최소공배수라고 한다. 예를 들어, 6과 15의 공배수는 30, 60, 90등이 있으며, 최소 공배수는 30이다.

두 자연수 A와 B가 주어졌을 때, A와 B의 최소공배수를 구하는 프로그램을 작성하시오.

---

## 입력
첫째 줄에 테스트 케이스의 개수 T(1 ≤ T ≤ 1,000)가 주어진다. 둘째 줄부터 T개의 줄에 걸쳐서 A와 B가 주어진다. (1 ≤ A, B ≤ 45,000)

---

## 출력
첫째 줄부터 T개의 줄에 A와 B의 최소공배수를 입력받은 순서대로 한 줄에 하나씩 출력한다.

## 풀이

### 풀이 핵심 로직
두 자연수 A와 B의 최소공배수(LCM)는 두 수의 곱을 그들의 최대공약수(GCD)로 나눈 값으로 구할 수 있습니다. 즉, LCM(A, B) = (A * B) / GCD(A, B)입니다. GCD는 유클리드 호제법을 사용하여 효율적으로 계산합니다.

### 동작 과정
1. 입력으로 테스트 케이스의 개수 T를 받습니다.
2. 각 테스트 케이스마다 두 자연수 A와 B를 입력받습니다.
3. GCD를 계산하기 위해 유클리드 호제법을 사용합니다.
   - A와 B가 주어졌을 때, B가 0이 될 때까지 A와 B를 바꿔가며 나머지를 계속 계산합니다.
4. GCD가 계산되면, LCM을 (A * B) / GCD(A, B)로 계산합니다.
5. 결과를 출력합니다.

예시 입력:
```
3
6 15
8 12
9 28
```

- 첫 번째 케이스: 
  - A=6, B=15
  - GCD(6, 15) = 3
  - LCM(6, 15) = (6 * 15) / 3 = 30

- 두 번째 케이스:
  - A=8, B=12
  - GCD(8, 12) = 4
  - LCM(8, 12) = (8 * 12) / 4 = 24

- 세 번째 케이스:
  - A=9, B=28
  - GCD(9, 28) = 1
  - LCM(9, 28) = (9 * 28) / 1 = 252

결과 출력:
```
30
24
252
```

### 시간 복잡도
GCD를 계산하는 유클리드 호제법의 시간 복잡도는 O(log(min(A, B)))입니다. 전체 T개의 테스트 케이스에 대해, 최악의 경우는 O(T * log(max(A, B)))입니다. 주어진 문제의 조건에서 T는 최대 1,000이고 A, B는 최대 45,000이므로, 이 알고리즘은 효율적으로 동작합니다.

## 코드
```python
t = int(input())

def gcd(a, b):
    while b > 0:
        a, b = b, a % b
    return a

def lcm(a, b):
    return a * b / gcd(a, b)

for _ in range(t):
    a, b = map(int, input().split())
    
    print(int(lcm(a, b)))
```
