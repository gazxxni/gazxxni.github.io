## 풀이

### 풀이 핵심 로직
이 문제의 핵심 아이디어는 사이트와 비밀번호를 키-값 쌍으로 저장하는 딕셔너리를 활용하여, 주어진 사이트에 대한 비밀번호를 효율적으로 조회하는 것입니다. 사용자가 입력한 사이트에 대해 비밀번호를 빠르게 출력할 수 있도록 해줍니다.

### 동작 과정
1. 입력으로 주어지는 두 정수 `n`과 `m`은 각각 비밀번호를 저장할 사이트의 개수와 비밀번호를 조회할 사이트의 개수를 나타냅니다.
2. `n`번 만큼 반복하여, 각 사이트와 비밀번호를 입력받고 딕셔너리 `password`에 저장합니다.
   - 예를 들어, 입력이 다음과 같다고 가정합니다:
     ```
     3 2
     google 1234
     facebook 5678
     twitter 91011
     ```
   - 이 경우, `password` 딕셔너리는 다음과 같이 저장됩니다:
     ```python
     {
         'google': '1234',
         'facebook': '5678',
         'twitter': '91011'
     }
     ```
3. 이후 `m`번 만큼 반복하여 사용자로부터 사이트 이름을 입력받고, 저장된 딕셔너리에서 해당 사이트의 비밀번호를 출력합니다.
   - 예를 들어, 입력이 다음과 같다면:
     ```
     google
     twitter
     ```
   - 출력 결과는:
     ```
     1234
     91011
     ```

### 시간 복잡도
이 알고리즘의 시간 복잡도는 O(n + m)입니다. 입력받는 과정에서 `n`개의 사이트와 비밀번호를 저장하는 데 O(n) 시간이 소요되고, `m`개의 사이트의 비밀번호를 조회하는 데 O(m) 시간이 소요됩니다. 각 조회는 딕셔너리를 사용하기 때문에 평균적으로 O(1)의 시간 복잡도로 수행됩니다.

## 코드
```python
import sys

input = sys.stdin.readline

n, m = map(int, input().split())
password = {}

for _ in range(n):
    site, pw = input().split()
    password[site] = pw

for _ in range(m):
    print(password[input().rstrip()])
```
