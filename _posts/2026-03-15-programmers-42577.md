## 문제
전화번호부에 적힌 전화번호 중, 한 번호가 다른 번호의 접두어인 경우가 있는지 확인하려 합니다.
전화번호가 다음과 같을 경우, 구조대 전화번호는 영석이의 전화번호의 접두사입니다.


구조대 : 119
박준영 : 97 674 223
지영석 : 11 9552 4421


전화번호부에 적힌 전화번호를 담은 배열 phone_book 이 solution 함수의 매개변수로 주어질 때, 어떤 번호가 다른 번호의 접두어인 경우가 있으면 false를 그렇지 않으면 true를 return 하도록 solution 함수를 작성해주세요.

제한 사항


phone_book의 길이는 1 이상 1,000,000 이하입니다.


각 전화번호의 길이는 1 이상 20 이하입니다.
같은 전화번호가 중복해서 들어있지 않습니다.



입출력 예제

        
phone_book
return


        
["119", "97674223", "1195524421"]

["123","456","789"]
true


["12","123","1235","567","88"]

입출력 예 #1
앞에서 설명한 예와 같습니다.

입출력 예 #3
첫 번째 전화번호, “12”가 두 번째 전화번호 “123”의 접두사입니다. 따라서 답은 false입니다.

## 풀이
### 풀이 핵심 로직
전화번호 목록에서 어떤 번호가 다른 번호의 접두사인지 확인하기 위해, 전화번호 목록을 정렬한 후, 인접한 번호들을 비교합니다. 정렬을 통해 같은 접두사를 가진 번호들이 인접하게 되므로, 한 번호가 다른 번호의 접두사인지 빠르게 확인할 수 있습니다.

### 동작 과정
1. 전화번호 목록 `phone_book`을 정렬합니다. 예를 들어, `["119", "97674223", "1195524421"]`는 정렬 후 `["119", "1195524421", "97674223"]`가 됩니다.
2. 정렬된 리스트에서 인접한 두 번호를 비교합니다.
   - 첫 번째 번호가 두 번째 번호의 접두사인지 확인합니다. 접두사 확인은 `phone_book[i] == phone_book[i+1][:len(phone_book[i])]`로 수행합니다.
   - 예를 들어, `phone_book[0]` ("119")와 `phone_book[1]` ("1195524421")를 비교할 경우, "119"는 "1195524421"의 접두사이므로 `False`를 반환합니다.
3. 만약 모든 인접한 번호 쌍에 대해 접두사 관계가 없다면, 최종적으로 `True`를 반환합니다.

### 시간 복잡도
- 정렬하는데 O(n log n) 시간이 소요되고, 인접한 번호를 비교하는 과정은 O(n) 시간이 소요됩니다. 따라서 전체 시간 복잡도는 O(n log n)입니다.

## 코드
```python
def solution(phone_book):
    answer = True
    phone_book.sort()
    n = len(phone_book)
    for i in range(n - 1):
        if phone_book[i] == phone_book[i+1][:len(phone_book[i])]:
            answer = False
        
    return answer
```
