## 풀이

### 풀이 핵심 로직
이 문제의 핵심 아이디어는 동적 프로그래밍을 사용하여 각 층의 각 호실에 거주하는 사람 수를 누적적으로 계산하는 것입니다. 0층의 각 호실은 1명씩 거주하므로, 이후 각 층의 호실 사람 수는 바로 아래 층의 호실 사람 수와 그 왼쪽 호실 사람 수의 합으로 결정됩니다.

### 동작 과정
예를 들어, 입력이 다음과 같다고 가정해 봅시다:
```
2
1
3
2
3
```
- 첫 번째 테스트 케이스:
  - 층: 1, 호: 3
  - 0층: [1, 1, 1] (각 호실에 1명)
  - 1층: [1, 2, 3] (1층 1호: 1명, 1층 2호: 1+1=2명, 1층 3호: 1+2=3명)
  - 결과: 3 출력

- 두 번째 테스트 케이스:
  - 층: 2, 호: 3
  - 0층: [1, 1, 1]
  - 1층: [1, 2, 3]
  - 2층: [1, 3, 6] (2층 1호: 1명, 2층 2호: 1+2=3명, 2층 3호: 1+3=6명)
  - 결과: 6 출력

최종 출력은 `3`과 `6`이 됩니다.

### 시간 복잡도
이 알고리즘의 시간 복잡도는 O(k * n)입니다. 여기서 k는 층 수, n은 호실 수입니다. 각 층마다 n-1번의 반복문을 수행하므로, 전체 복잡도는 층 수와 호실 수의 곱에 비례합니다.

---

## 코드
```python
import sys

t=int(input())

for _ in range(t):  
    floor = int(input())  # 층
    num = int(input())  # 호
    f0 = [x for x in range(1, num+1)]  # 0층 리스트
    for k in range(floor):  # 층 수 만큼 반복
        for i in range(1, num):  # 1 ~ n-1까지 (인덱스로 사용)
            f0[i] += f0[i-1]  # 층별 각 호실의 사람 수를 변경
    print(f0[-1])  # 가장 마지막 수 출력

```
