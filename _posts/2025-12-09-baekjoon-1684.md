## 문제
정수 N을 정수 D로 나눴을 때의 몫을 Q, 나머지를 R이라고 하면 항등식 R = N - Q×D가 성립한다.

n개의 정수로 된 수열이 있을 때, 모든 정수를 한 정수 D로 나눴을 때 나머지가 같아지는 경우가 있다. 그리고 수열에 따라서는 이러한 정수 D가 여러 개 존재할 수 있다.

n개의 정수로 된 수열이 주어졌을 때, 가장 큰 D를 구하는 프로그램을 작성하시오.

---

## 입력
첫째 줄에 n(1 ≤ n ≤ 1,000)이 주어진다. 다음 줄에는 절댓값이 1,000,000을 넘지 않는 n개의 정수들이 주어진다.

---

## 출력
첫째 줄에 가장 큰 D를 출력한다. 항상 가장 큰 D가 존재하는 경우만 입력으로 주어진다.

## 풀이

### 풀이 핵심 로직
주어진 수열의 모든 정수에 대해 같은 나머지를 만들기 위한 가장 큰 정수 D를 찾는 문제입니다. 이를 위해 수열의 모든 정수 쌍의 차를 계산하고, 이 차들의 최대공약수(GCD)를 구하는 방식으로 D를 도출합니다. 두 수의 차가 D로 나누어 떨어져야 하므로, 모든 차의 GCD가 D가 됩니다.

### 동작 과정
1. **입력받기**: n과 n개의 정수를 입력받습니다.
2. **중복 제거**: 수열에서 중복된 값을 제거합니다. 이는 같은 값들 사이의 차는 0이 되어 GCD 계산에 불필요하므로, 유일한 값들만을 고려합니다.
3. **차 계산**: 각 서로 다른 값 쌍의 절대 차를 계산합니다.
   - 예시: 수열이 [3, 6, 9]인 경우, 서로 다른 값 쌍은 (3, 6), (3, 9), (6, 9)이며, 차는 |6 - 3| = 3, |9 - 3| = 6, |9 - 6| = 3입니다.
4. **최대공약수 계산**: 계산된 차들로부터 GCD를 구합니다. 이를 통해 가장 큰 D를 도출합니다.
   - 예시: 위에서 계산한 차들 3, 6, 3의 GCD는 3입니다.
5. **출력**: 구한 GCD 값을 출력합니다.

### 시간 복잡도
이 알고리즘의 시간 복잡도는 O(n^2 log M)입니다. 여기서 n은 수열의 길이이며, M은 차이의 최대값입니다. GCD 계산은 유클리드 호제법을 사용하므로 log M의 시간이 필요합니다. 수열의 수가 최대 1,000개일 경우, O(n^2) = O(1,000,000) 번의 차 계산이 필요합니다.

## 코드
```python
import sys
input = sys.stdin.readline

'''
두 수의 차가 D로 나누어 떨어진다
(aᵢ - aⱼ) % D = 0 -> 성립해야함

따라서 |a₂ - a₁|, |a₃ - a₁|, ..., |aₙ - a₁|
이 숫자들의 최대공약수를 구해야함
'''

n = int(input())
arr = list(map(int, input().split()))

def gcd(a, b):
    while b != 0:
        a, b = b, a % b
        
    return a

brr = list(set(arr))
crr = []

for i in range(len(brr) - 1):
    for j in range(i + 1, len(brr)):
        crr.append(abs(brr[j] - brr[i])) 
        
ans = crr[0]
for i in range(1, len(crr)): 
    ans = gcd(ans, crr[i])
    
print(ans)
```
