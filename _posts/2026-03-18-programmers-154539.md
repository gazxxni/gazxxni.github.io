## 문제
정수로 이루어진 배열 numbers가 있습니다. 배열 의 각 원소들에 대해 자신보다 뒤에 있는 숫자 중에서 자신보다 크면서 가장 가까이 있는 수를 뒷 큰수라고 합니다.
정수 배열 numbers가 매개변수로 주어질 때, 모든 원소에 대한 뒷 큰수들을 차례로 담은 배열을 return 하도록 solution 함수를 완성해주세요. 단, 뒷 큰수가 존재하지 않는 원소는 -1을 담습니다.



제한사항


4 ≤ numbers의 길이 ≤ 1,000,000


1 ≤ numbers[i] ≤ 1,000,000





입출력 예

        
numbers
result


        
[2, 3, 3, 5]
[3, 5, 5, -1]


[9, 1, 5, 3, 6, 2]
[-1, 5, 6, 6, -1, -1]


      


입출력 예 설명

입출력 예 #1
2의 뒷 큰수는 3입니다. 첫 번째 3의 뒷 큰수는 5입니다. 두 번째 3 또한 마찬가지입니다. 5는 뒷 큰수가 없으므로 -1입니다. 위 수들을 차례대로 배열에 담으면 [3, 5, 5, -1]이 됩니다.

입출력 예 #2
9는 뒷 큰수가 없으므로 -1입니다. 1의 뒷 큰수는 5이며, 5와 3의 뒷 큰수는 6입니다. 6과 2는 뒷 큰수가 없으므로 -1입니다. 위 수들을 차례대로 배열에 담으면 [-1, 5, 6, 6, -1, -1]이 됩니다.

## 풀이
### 풀이 핵심 로직
주어진 배열에서 각 원소에 대해 자신보다 뒤에 있는 큰 수를 찾기 위해 스택을 활용합니다. 스택에는 인덱스를 저장하고, 현재 원소가 스택의 최상위 원소가 가리키는 값보다 클 경우, 해당 인덱스의 원소에 현재 원소를 할당합니다. 이렇게 하면 각 원소에 대해 가장 가까운 뒤의 큰 수를 효율적으로 찾을 수 있습니다.

### 동작 과정
1. **초기화**: `answer` 배열을 -1로 초기화합니다. `stack`에는 첫 번째 원소의 인덱스(0)를 추가합니다.
2. **반복문 시작**: 배열의 두 번째 원소부터 끝까지 반복합니다.
   - 예를 들어, `numbers = [2, 3, 3, 5]`일 때:
     - i = 1: 현재 값이 3이고 스택의 최상위 인덱스(0)의 값 2보다 크므로, `answer[0]`을 3으로 업데이트하고 인덱스 0을 스택에서 제거합니다. 스택에 1 추가.
     - i = 2: 현재 값이 다시 3이므로 스택의 최상위 인덱스(1)의 값(3)과 같아 아무것도 하지 않고 인덱스 2를 스택에 추가.
     - i = 3: 현재 값이 5로 스택의 최상위 인덱스(2)의 값(3)보다 크므로, `answer[1]`과 `answer[2]`를 각각 5로 업데이트하고 인덱스 1과 2를 스택에서 제거. 스택에 3 추가.
3. **결과**: 모든 원소에 대해 반복이 끝나면, 스택에 남아있는 인덱스는 뒤에 큰 수가 없으므로 (-1) 유지.
4. 최종적으로 `answer` 배열을 반환합니다.

이 과정을 통해 `numbers = [2, 3, 3, 5]`로부터 `[3, 5, 5, -1]`을 얻을 수 있습니다.

### 시간 복잡도
이 알고리즘의 시간 복잡도는 O(n)입니다. 각 원소에 대해 스택에 한 번만 추가되고 한 번만 제거되므로, 전체적으로 n번의 작업을 수행하게 됩니다.

## 코드
```python
def solution(numbers):
    n = len(numbers)
    answer = [-1] * n
    stack = [0]
    for i in range(1, n):
        while stack and numbers[stack[-1]] < numbers[i]:
            idx = stack.pop(-1)
            answer[idx] = numbers[i]
        
        stack.append(i) 
            
    
            
    return answer
```
