## 문제
조이스틱으로 알파벳 이름을 완성하세요. 맨 처음엔 A로만 이루어져 있습니다.
ex) 완성해야 하는 이름이 세 글자면 AAA, 네 글자면 AAAA

조이스틱을 각 방향으로 움직이면 아래와 같습니다.
▲ - 다음 알파벳
▼ - 이전 알파벳 (A에서 아래쪽으로 이동하면 Z로)
◀ - 커서를 왼쪽으로 이동 (첫 번째 위치에서 왼쪽으로 이동하면 마지막 문자에 커서)
▶ - 커서를 오른쪽으로 이동 (마지막 위치에서 오른쪽으로 이동하면 첫 번째 문자에 커서)

예를 들어 아래의 방법으로 "JAZ"를 만들 수 있습니다.
- 첫 번째 위치에서 조이스틱을 위로 9번 조작하여 J를 완성합니다.
- 조이스틱을 왼쪽으로 1번 조작하여 커서를 마지막 문자 위치로 이동시킵니다.
- 마지막 위치에서 조이스틱을 아래로 1번 조작하여 Z를 완성합니다.
따라서 11번 이동시켜 "JAZ"를 만들 수 있고, 이때가 최소 이동입니다.

만들고자 하는 이름 name이 매개변수로 주어질 때, 이름에 대해 조이스틱 조작 횟수의 최솟값을 return 하도록 solution 함수를 만드세요.

제한 사항


name은 알파벳 대문자로만 이루어져 있습니다.
name의 길이는 1 이상 20 이하입니다.

return

## 풀이
### 풀이 핵심 로직
이 문제의 핵심 아이디어는 조이스틱을 위아래로 움직여서 각 문자를 변경하는 데 필요한 최소 횟수를 계산하는 것과, 커서를 좌우로 이동하는 데 드는 최소 횟수를 계산하는 것입니다. 이를 통해 전체 조작 횟수의 최솟값을 구합니다.

### 동작 과정
1. **문자 변경 횟수 계산**: 주어진 이름의 각 문자에 대해 A에서 해당 문자까지의 위아래 조작 횟수를 계산합니다. 이는 `min(정방향 조작 수, 역방향 조작 수)`를 통해 구합니다.
   - 예를 들어, J의 경우 A에서 J까지의 조작은 9번(▲)이고 A에서 Z를 거쳐 J로 가는 경우는 17번(▼)이므로, J를 만들기 위해 필요한 조작 횟수는 9입니다.

2. **커서 이동 횟수 계산**: 이름의 각 문자를 처리하면서, A로 이루어져 있는 연속된 부분을 찾아 커서를 이동하는 최소의 경로를 계산합니다.
   - 예를 들어, "JEROEN"의 경우 "J"에서 "E"로 이동 후 "R"로 이동하고, 다시 "O"로 이동하기 위해 커서를 이동해야 합니다. A가 연속된 경우는 커서를 왼쪽으로 이동하거나 오른쪽으로 이동하여 A를 건너뛰는 것이 유리합니다.
   - 이동 횟수는 `i * 2 + n - next_i` (현재 위치에서 반대 방향으로 이동 후 A를 건너뛰는 경우)와 `i + 2 * (n - next_i)` (현재 위치에서 A를 건너뛰고 반대 방향으로 이동하는 경우) 중에서 최소 값을 선택합니다.

3. **총 조작 횟수 반환**: 위에서 계산한 문자 변경 횟수와 커서 이동 횟수를 합쳐서 최종 결과를 반환합니다.

### 시간 복잡도
이 알고리즘은 이름의 길이 n에 대해 O(n) 시간 복잡도를 가집니다. 각각의 문자에 대해 한 번씩만 순회하므로 효율적입니다.

## 코드
```python
def solution(name):
    up_down = 0
    for c in name:
        up_down += min(ord(c) - ord('A'), ord('Z') - ord(c) + 1)
        
    n = len(name)
    move = n - 1
    
    for i in range(n):
        next_i = i + 1
        
        while next_i < n and name[next_i] == 'A':
            next_i += 1
            
        move = min(move, i * 2 + n - next_i, i + 2 * (n - next_i))
    
    return up_down + move
```
