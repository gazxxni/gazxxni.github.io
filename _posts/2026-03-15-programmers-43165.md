## 문제
n개의 음이 아닌 정수들이 있습니다. 이 정수들을 순서를 바꾸지 않고 적절히 더하거나 빼서 타겟 넘버를 만들려고 합니다. 예를 들어 [1, 1, 1, 1, 1]로 숫자 3을 만들려면 다음 다섯 방법을 쓸 수 있습니다.
-1+1+1+1+1 = 3
+1-1+1+1+1 = 3
+1+1-1+1+1 = 3
+1+1+1-1+1 = 3

## 풀이
### 풀이 핵심 로직
이 문제는 DFS(Depth First Search)를 사용하여 모든 가능한 경우의 수를 탐색하는 방식으로 해결할 수 있습니다. 각 숫자를 더하거나 빼는 두 가지 선택지를 모두 고려하여 재귀적으로 호출하고, 가장 마지막 단계에서 타겟 넘버와 현재의 합이 일치하는 경우를 카운트합니다.

### 동작 과정
1. **입력값**: `numbers = [1, 1, 1, 1, 1]`, `target = 3`
2. **초기 호출**: `dfs(0, 0)`으로 시작하여 현재 인덱스 `idx = 0`과 현재 합 `cur_sum = 0`을 설정합니다.
3. **재귀 호출**:
   - 인덱스가 `len(numbers)`에 도달하면 현재의 합과 타겟을 비교합니다.
   - 각각의 숫자에 대해 두 가지 선택지(더하기와 빼기)를 재귀적으로 호출합니다.
4. **예시 계산**:
   - 첫 번째 숫자 `1`을 더한 경우: `dfs(1, 1)`
   - 첫 번째 숫자 `1`을 뺀 경우: `dfs(1, -1)`
   - 이 과정을 반복하여 모든 조합을 탐색합니다.
5. **결과**: 모든 경우의 수를 탐색한 후, 타겟 넘버와 합이 일치하는 경우를 카운트하여 최종 결과를 반환합니다.

### 시간 복잡도
이 알고리즘의 시간 복잡도는 O(2^N)입니다. 각 숫자마다 두 가지 선택(더하기 또는 빼기)이 있으므로, N개의 숫자에 대해 총 2^N 개의 조합을 탐색해야 합니다. 이는 N이 클 경우 성능에 영향을 미칠 수 있습니다.

## 코드
```python
def solution(numbers, target):
    def dfs(idx, cur_sum):
        if idx == len(numbers):
            if cur_sum == target:
                return 1
            else:
                return 0
            
        return dfs(idx + 1, cur_sum + numbers[idx]) + dfs(idx + 1, cur_sum - numbers[idx])
        
        
        
    return dfs(0, 0)
```
